# 시각 instruction tuning의 표준 — LLaVA

> **Visual Instruction Tuning** — Liu et al., 2023, NeurIPS Oral (arXiv:2304.08485)
> 원문: https://arxiv.org/abs/2304.08485 · 분류: 최신

Flamingo[2]는 강력했지만 복잡했습니다. LLM 내부에 교차 어텐션을 끼우는 구조라 구현과 재현이 까다로웠죠. LLaVA는 정반대 방향을 택합니다. 아주 단순한 연결만으로도 강력한 멀티모달 어시스턴트를 만들 수 있음을 보였고, 그 데이터와 레시피를 공개해 오픈 VLM의 사실상 표준이 되었습니다. 구조는 최대한 단순하게, 대신 학습 데이터로 승부한다는 전략이 핵심입니다.

## 배경: 두 개의 장벽

당시 VLM을 만드는 데에는 두 가지 장벽이 있었습니다. 첫째는 구조의 복잡성입니다. Flamingo의 교차 어텐션 같은 정교한 결합은 구현이 어려웠습니다. 둘째는 데이터의 부족입니다. "이미지를 보고 지시를 따르는" 멀티모달 instruction 데이터, 즉 이미지에 대한 질문과 좋은 답이 짝지어진 데이터가 부족했습니다. LLM을 챗봇으로 만든 instruction tuning을 멀티모달로 하려면 이런 데이터가 필요한데, 그것을 사람이 일일이 만드는 것은 비쌌습니다.

> 핵심 질문: 단순한 구조로, 그리고 부족한 멀티모달 지시 데이터를 어떻게 채워 강한 어시스턴트를 만들까?

## 핵심 아이디어: 단순한 다리와 합성 데이터

LLaVA의 핵심은 두 가지입니다.

- **단순 연결(projection)**: 비전 인코더(CLIP[3])의 출력을 작은 선형 변환(또는 간단한 MLP)만 거쳐 LLM의 입력 토큰으로 넣습니다. Flamingo의 복잡한 교차 어텐션 대신 가벼운 다리 하나로 비전과 언어를 잇습니다.
- **시각 instruction 데이터 자동 생성**: 강한 LLM(GPT-4)을 시켜 이미지에 대한 (질문, 좋은 답) 데이터를 대량 생성하고, 그것으로 튜닝합니다. 합성 데이터를 멀티모달에 적용한 것입니다.

> 비유 — 통역사를 거창한 동시통역 부스(Flamingo) 대신 간단한 메모 전달자(선형 연결)로 두고, 대신 연습 문제집(GPT-4가 만든 지시 데이터)을 잔뜩 풀게 하는 것. 구조는 단순하게 두고 학습 데이터로 승부하는 전략입니다.

## 어떻게 작동하나

```
이미지 → CLIP 인코더 → [선형 projection] → 이미지 토큰
이미지 토큰 + 텍스트 지시 → [LLM] → 답
학습: ① 연결층 정렬 → ② GPT-4가 만든 시각 instruction 데이터로 튜닝
```

이미지를 CLIP으로 인코딩한 뒤 작은 projection으로 LLM이 알아들을 수 있는 토큰으로 바꾸기만 하면, 이미지가 마치 또 다른 텍스트 토큰처럼 LLM에 들어갑니다. 학습은 두 단계로, 먼저 연결층을 정렬한 뒤 시각 instruction 데이터로 전체를 튜닝합니다. 구조가 단순해 재현과 확장이 쉽고, 데이터 레시피가 공개되어 누구나 따라 만들 수 있습니다. LLaVA-1.5와 1.6 등으로 빠르게 개선되며 오픈 멀티모달의 기준선이 되었습니다.

| 항목 | Flamingo | LLaVA |
|---|---|---|
| 결합 방식 | 교차 어텐션(복잡) | 선형 projection(단순) |
| 학습 데이터 | 웹 짝 데이터 | GPT-4 합성 instruction |
| 재현성 | 낮음 | 높음(공개 레시피) |

## 임팩트: 무엇을 바꿨나

- **오픈 멀티모달 어시스턴트의 사실상 표준 레시피**가 되었습니다. "단순 연결 + 시각 instruction tuning"은 수많은 후속 VLM의 출발점입니다.
- 강한 LLM으로 멀티모달 학습 데이터를 합성하는 접근을 대중화했습니다. T04 정렬과 심화편 S5의 합성 데이터 흐름과 통합니다. 데이터가 없으면 강한 모델로 만든다는 발상입니다.
- 단순함이 곧 확장성임을 보여, 연구와 실무 양쪽에서 폭넓게 채택되었습니다. 복잡한 구조보다 단순한 구조에 좋은 데이터를 더하는 것이 효과적임을 입증했습니다.

## 한계와 주의점

- 학습 데이터를 강한 모델(GPT-4)에 의존해 생성하므로, 그 모델의 편향과 오류가 전파될 수 있습니다. 교사 모델의 한계가 학생 모델로 이어집니다.
- 단순 연결은 세밀한 시각 과제(고해상도, 문서, 정밀한 위치)에 한계가 있을 수 있습니다. 다음 글 Qwen2-VL[4]이 노리는 지점입니다.
- 합성 데이터의 품질이 결과를 좌우합니다. 거르는 과정이 필요합니다.

## 더 읽을거리와 연결

- 이 책 내부: 비전 인코더는 **CLIP**[3], 더 복잡한 결합은 **Flamingo**[2], 다음 글 **Chameleon**(통합형으로 가는 길), **Qwen2-VL**[4](고해상도).
- 심화편 S5(합성 데이터)와 함께 보면 좋습니다.

## 한 줄 요약

> LLaVA는 단순 선형 연결과 GPT-4로 만든 시각 instruction 데이터로 강력한 오픈 멀티모달 어시스턴트를 만들어, VLM의 표준 레시피가 되었다.
