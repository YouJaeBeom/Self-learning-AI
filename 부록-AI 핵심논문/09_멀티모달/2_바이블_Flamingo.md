# 동결 LLM에 눈을 달다 — Flamingo

> **Flamingo: a Visual Language Model for Few-Shot Learning** — Alayrac et al., 2022, NeurIPS (arXiv:2204.14198)
> 원문: https://arxiv.org/abs/2204.14198 · 분류: 바이블

CLIP[2]은 이미지와 텍스트를 정렬했지만, 사진을 보고 자유롭게 대화하지는 못합니다. CLIP은 "이 이미지가 이 텍스트와 얼마나 맞나"를 답할 뿐, 이미지를 보고 문장을 생성하지 않습니다. Flamingo는 한 걸음 더 나아갑니다. 이미 강력한 LLM은 그대로 두고, 비전을 어댑터로 이어 붙여 그림을 보고 말하고 few-shot으로 배우는 비전-언어 모델(VLM)의 길을 열었습니다. 거대한 LLM을 재학습하지 않고 눈만 다는 이 효율적 접근은, 이후 OpenFlamingo와 IDEFICS, LLaVA 같은 VLM 폭발의 토대가 되었습니다.

## 배경: 멀티모달을 처음부터 학습하는 비용

뛰어난 멀티모달 모델을 처음부터 거대하게 학습하는 것은 매우 비쌉니다. 그런데 이미 언어를 잘하는 강력한 LLM이 있습니다. 이 LLM의 언어 능력을 버리고 처음부터 다시 학습하는 것은 낭비입니다. 자연스러운 질문이 생깁니다. 기존 LLM의 언어 능력을 그대로 살리면서, 거기에 비전만 더할 수는 없을까?

> 핵심 질문: LLM을 다시 학습하지 않고, 눈(비전)만 달아 보고 말하게 할 수 없을까?

## 핵심 아이디어: 본체는 얼리고, 다리만 학습한다

Flamingo는 동결(frozen) LLM에 비전 인코더를 cross-attention으로 연결합니다. LLM의 가중치는 얼려 보존하고, 그 층 사이사이에 이미지를 참고하는 교차 어텐션을 끼워 넣습니다. 이미지 인코더가 본 것을, LLM이 글을 생성하며 필요할 때 참고하는 구조입니다.

> 비유 — 뛰어난 작가(LLM)는 그대로 두고, 그림을 설명해 주는 통역사(어댑터)를 옆에 붙이는 것. 작가는 다시 공부하지 않고도, 통역사 덕분에 그림을 본 듯 글을 씁니다. 새로 가르치는 것은 통역사뿐이라 비용이 작습니다.

## 어떻게 작동하나

```
이미지 → 비전 인코더 → Perceiver Resampler(가변 개수 → 고정 개수 토큰으로 요약)
                          │ gated cross-attention
텍스트(+예시 몇 개) → [동결 LLM 층들] ←──────┘ → 답 생성
LLM 가중치는 고정, 새로 학습하는 건 주로 잇는 부분(리샘플러 + 교차 어텐션)
```

두 가지 장치가 핵심입니다. Perceiver Resampler는 비전 인코더가 만든 가변 개수의 특징을 고정된 개수의 토큰으로 요약해, 이미지 크기에 상관없이 일정한 인터페이스를 제공합니다. 게이트가 달린 교차 어텐션(gated cross-attention)은 LLM의 각 층에 삽입되어 이미지 정보를 주입하되, 게이트로 그 영향을 조절해 학습 초기에 LLM을 망가뜨리지 않습니다. 학습 비용이 이 연결부에 집중되어 거대한 LLM을 재학습하지 않아도 됩니다(T03 효율의 어댑터 정신과 통합니다). 또한 이미지와 텍스트가 번갈아 섞인 입력(interleaved)을 다룰 수 있어, 예시 몇 개로 새 과제를 배우는 in-context few-shot이 가능합니다.

| 항목 | 처음부터 멀티모달 학습 | Flamingo(동결 LLM + 어댑터) |
|---|---|---|
| LLM 가중치 | 새로 학습 | 동결(보존) |
| 학습 비용 | 매우 큼 | 연결부에 집중(작음) |
| 언어 능력 | 새로 획득 | 기존 LLM 것 활용 |

### 게이트가 0에서 시작하는 이유

동결 LLM에 새 교차 어텐션 층을 끼워 넣을 때 한 가지 위험이 있습니다. 학습 초기의 무작위 어텐션이 잘 작동하던 LLM의 표현을 망가뜨릴 수 있다는 점입니다. Flamingo는 교차 어텐션의 출력에 `tanh` 게이트를 곱하고 그 게이트를 0으로 초기화해 이 문제를 피합니다. 학습 시작 시점에는 게이트가 0이라 LLM이 원래 그대로 동작하고(이미지 정보가 전혀 안 섞임), 학습이 진행되며 게이트가 서서히 열려 이미지 정보가 점진적으로 주입됩니다. 강한 LLM을 망가뜨리지 않으면서 비전을 부드럽게 붙이는 안전장치입니다. DiT의 adaLN-zero와 같은 "0에서 켜기" 발상이 멀티모달 결합에도 쓰인 셈입니다.

### interleaved 입력이 few-shot을 가능케 한다

Flamingo의 또 다른 핵심은 이미지와 텍스트가 번갈아 섞인(interleaved) 시퀀스로 학습한다는 점입니다. "이미지1 → 설명1 → 이미지2 → 설명2 → 이미지3 → ?" 같은 입력을 처리할 수 있어, 프롬프트에 예시 몇 쌍을 넣으면 새 과제를 추가 학습 없이 푸는 in-context few-shot이 가능해집니다. LLM이 텍스트에서 보여 준 "예시 몇 개 보고 패턴 파악" 능력을, 이미지가 섞인 맥락으로 확장한 것입니다. 웹에서 긁은 이미지-텍스트 혼합 문서로 학습했기에 이런 능력이 자연스럽게 길러졌습니다.

## 임팩트: 왜 바이블이 되었나

- **"동결 LLM + 어댑터" VLM 설계를 대중화**했습니다. OpenFlamingo, IDEFICS 등 후속 오픈 모델이 이 구도를 따랐습니다.
- 멀티모달을 처음부터 학습하지 않고 기존 LLM에 붙이는 효율적 길을 표준화해, 이후 LLaVA[3] 등 VLM 폭발의 토대가 되었습니다. 강력한 LLM이 계속 나오면 거기에 눈만 붙이면 된다는 재사용의 길을 연 것입니다.
- "few-shot으로 보고 배운다"는 LLM의 능력을 비전-언어로 가져왔습니다.

## 한계와 주의점

- 교차 어텐션을 LLM 내부에 끼우는 구조라 구현이 비교적 복잡합니다. 다음 글 LLaVA[3]가 이를 훨씬 단순화합니다.
- 동결 LLM의 언어 한계를 그대로 물려받습니다. 본체를 얼리는 대가입니다.
- 비전 인코더와 어댑터의 표현 한계가 세밀한 시각 이해를 제약할 수 있습니다.

## 한 줄 요약

> Flamingo는 동결 LLM에 비전을 cross-attention 어댑터로 이어 붙여, 재학습 없이 보고 말하고 few-shot으로 배우는 VLM의 길을 열었다.
