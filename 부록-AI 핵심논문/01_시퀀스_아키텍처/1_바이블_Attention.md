# 어텐션이 전부다 — Attention / Transformer

> **Attention Is All You Need** — Vaswani et al., 2017, NeurIPS (arXiv:1706.03762)
> 원문: https://arxiv.org/abs/1706.03762 · 분류: 바이블

2017년 이전까지 시퀀스 모델링의 왕좌는 순환 신경망(RNN), 그중에서도 LSTM[5]과 GRU가 차지하고 있었습니다. 기계 번역, 음성 인식, 언어 모델링 — 순서가 있는 데이터를 다루는 거의 모든 곳에서 "단어를 하나씩 차례로 읽는" 구조가 표준이었죠. 그런데 이 순차성이 두 가지 근본적인 족쇄였습니다. 하나는 병렬화가 불가능해 학습이 느리다는 것, 다른 하나는 멀리 떨어진 단어 사이의 관계를 잘 기억하지 못한다는 것이었습니다. 「Attention Is All You Need」는 제목 그대로, 순환 구조를 통째로 버리고 **어텐션 하나만으로** 이 두 족쇄를 동시에 끊어 버린 논문입니다. 이 글은 그 구조를 수식 수준까지 따라가 보면서, 왜 이 한 편이 오늘날 모든 대규모 언어 모델(LLM)의 설계도가 되었는지를 정리합니다.

## RNN의 시대, 그리고 두 개의 족쇄

RNN 계열 모델은 시간 단계 `t`의 은닉 상태 `h_t`를 이전 상태 `h_{t-1}`과 현재 입력 `x_t`로부터 계산합니다.

```
h_t = f(W_h · h_{t-1} + W_x · x_t + b)
```

이 식에는 `h_{t-1}`이 들어 있습니다. 즉 `t`번째를 계산하려면 반드시 `t-1`번째가 먼저 끝나야 합니다. 문장이 1,000단어면 1,000번의 계산을 **줄줄이 직렬로** 해야 한다는 뜻입니다. GPU는 수천 개의 연산을 동시에 처리하도록 만들어진 장치인데, RNN의 이 구조는 그 병렬성을 원천적으로 쓸 수 없게 만들었습니다.

두 번째 족쇄는 더 본질적입니다. 첫 단어의 정보가 마지막 단어까지 전달되려면 은닉 상태라는 좁은 통로를 1,000번 통과해야 합니다. 그 과정에서 신호는 곱셈을 반복하며 점점 희미해지거나(기울기 소실) 폭발합니다. LSTM의 게이트 구조[5]가 이를 완화했지만 근본적으로 없애지는 못했습니다. 결국 "30단어 앞의 주어와 지금의 동사를 일치시켜라" 같은 장거리 의존성(long-range dependency)은 RNN이 늘 약한 지점이었습니다.

이미 2014~2015년에 어텐션이라는 보조 장치가 등장해 있었습니다. Bahdanau 등[6]이 기계 번역에서 "디코더가 출력 단어를 낼 때마다 입력 문장의 어느 부분을 볼지 가중치로 고르게" 하는 어텐션을 제안했고, 이는 seq2seq[7]의 고정 길이 병목을 크게 풀어 줬습니다. 다만 그때까지 어텐션은 어디까지나 RNN을 **거드는** 부품이었습니다. 이 논문의 도발적인 질문은 이것입니다.

> 순환 구조를 *완전히 버리고*, 어텐션만으로 시퀀스를 모델링할 수 있을까?

| 구조 | 계층당 복잡도 | 순차 연산 | 최대 경로 길이 |
|---|---|---|---|
| RNN | `O(n · d²)` | `O(n)` | `O(n)` |
| CNN | `O(k · n · d²)` | `O(1)` | `O(log_k n)` |
| Self-Attention | `O(n² · d)` | `O(1)` | `O(1)` |

여기서 `n`은 시퀀스 길이, `d`는 표현 차원, `k`는 합성곱 커널 크기입니다. 이 표가 논문의 핵심 주장을 압축합니다. 셀프 어텐션은 **순차 연산이 `O(1)`** 이라 완전히 병렬화되고, 임의의 두 토큰 사이 **신호 전달 경로가 `O(1)`** 이라 아무리 멀리 떨어진 단어라도 한 번에 연결됩니다. 대가는 계층당 복잡도가 길이의 제곱 `O(n²·d)`이라는 점인데, 이 제곱이 훗날 모든 후속 연구의 숙제가 됩니다.

## 핵심 아이디어: 모두가 모두를 동시에 본다

셀프 어텐션의 직관은 한 문장으로 말할 수 있습니다. 단어를 순서대로 훑는 대신, **모든 단어가 모든 단어를 한꺼번에 바라보게** 하는 것입니다. 각 단어는 "나는 지금 누구를 참고해야 하나?"를 스스로 계산해, 관련이 깊은 단어에 더 큰 주의(attention)를 줍니다. 순서를 기다릴 필요가 없으니 전부 병렬로 처리됩니다.

회의실에 비유하면 이렇습니다. RNN이 한 사람씩 차례로 옆 사람에게 귓속말을 전달하는 방식이라면, 어텐션은 모두가 동시에 회의실 전체를 둘러보고 필요한 사람에게 곧장 시선을 주는 방식입니다. 전달이 빠르고, 멀리 앉은 사람과도 중간 전달자 없이 직접 소통합니다. "The animal didn't cross the street because **it** was too tired"에서 `it`이 `animal`을 가리킨다는 것을, 어텐션은 두 단어를 직접 연결해 한 번에 파악합니다.

## 수식으로 들어가기

이 논문에서 가장 많이 인용되는 한 줄은 스케일드 닷-프로덕트 어텐션(scaled dot-product attention)입니다[1].

```
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V
```

각 기호를 풀어 보겠습니다. 입력의 각 토큰은 세 개의 벡터로 변환됩니다.

- **Query(질문) `Q`**: "나는 어떤 정보를 찾고 있나"
- **Key(색인) `K`**: "나는 어떤 정보를 갖고 있나"
- **Value(내용) `V`**: "내가 실제로 전달할 내용"

도서관에 비유하면 `Q`는 내가 머릿속에 떠올린 검색어, `K`는 책등에 적힌 제목, `V`는 책의 실제 내용입니다. 동작은 네 단계로 나뉩니다.

1. **유사도 계산 `Q·Kᵀ`**: 각 단어의 질문 `Q`를 모든 단어의 색인 `K`와 내적합니다. 두 벡터의 내적은 방향이 비슷할수록 커지므로, 이 값이 곧 "이 단어가 저 단어에 얼마나 관련 있나"의 점수가 됩니다. `n`개 토큰이면 `n×n`짜리 점수 행렬이 나오고, 이 행렬을 만드는 데 드는 비용이 바로 `O(n²)`의 정체입니다.

2. **스케일 조정 `/√d_k`**: 점수를 차원의 제곱근으로 나눕니다. 왜 하필 `√d_k`일까요? `Q`와 `K`의 각 성분이 평균 0, 분산 1로 독립이라면, 둘의 내적은 `d_k`개 항을 더한 값이라 분산이 `d_k`로 커집니다. 차원이 큰 모델일수록 점수의 절댓값이 커지고, 그러면 softmax가 가장 큰 값 하나에 거의 1을 몰아주는 극단적으로 뾰족한 분포가 됩니다. 이렇게 되면 기울기가 거의 0이 되어 학습이 멈춥니다. `√d_k`로 나눠 분산을 다시 1 근처로 되돌리면 softmax가 적당히 부드러워져 학습이 안정됩니다. 작아 보이는 이 나눗셈 하나가 실제로는 깊은 모델을 학습 가능하게 만드는 장치입니다.

3. **확률화 `softmax`**: 각 행(=각 질문)에 softmax를 적용해 점수를 합이 1인 가중치로 바꿉니다. 이제 "이 단어는 저 단어를 30%, 그 단어를 50% 참고한다"처럼 해석할 수 있습니다.

4. **가중합 `· V`**: 그 가중치로 모든 단어의 내용 `V`를 섞어 새로운 표현을 만듭니다. 관련 깊은 단어의 내용이 더 진하게 반영됩니다.

### 멀티헤드 어텐션 — 여러 관점으로 동시에

한 번의 어텐션은 한 종류의 관계밖에 잡지 못합니다. 그래서 논문은 같은 어텐션을 `h`개의 독립된 '머리(head)'로 나눠 병렬 수행합니다[1].

```
MultiHead(Q,K,V) = Concat(head_1, …, head_h) · W^O
head_i = Attention(Q·W_i^Q, K·W_i^K, V·W_i^V)
```

각 머리는 `d_model` 차원을 `d_k = d_model / h`로 쪼갠 부분 공간에서 어텐션을 수행합니다. 한 머리는 문법적 일치(주어–동사)에, 다른 머리는 의미적 연관(대명사–선행사)에, 또 다른 머리는 바로 옆 단어에 집중하는 식으로 서로 다른 관계를 병렬로 포착합니다. 원논문은 `d_model=512`, `h=8`을 사용해 머리당 64차원으로 작업했습니다. 흥미롭게도 머리를 쪼개도 총 연산량은 단일 어텐션과 비슷하게 유지됩니다.

### 위치 인코딩 — 순서를 어떻게 알려주나

셀프 어텐션에는 치명적인 맹점이 하나 있습니다. 모든 단어를 동시에 보기 때문에 **순서 정보가 없습니다.** "개가 사람을 물었다"와 "사람이 개를 물었다"를 구분하지 못하는 것이죠. 그래서 입력 임베딩에 위치 정보를 따로 더해 줍니다. 원논문은 사인·코사인 함수를 씁니다[1].

```
PE(pos, 2i)   = sin( pos / 10000^(2i/d_model) )
PE(pos, 2i+1) = cos( pos / 10000^(2i/d_model) )
```

차원 인덱스 `i`마다 다른 주기의 파동을 만들어, 위치마다 고유한 '지문'을 부여하는 방식입니다. 낮은 차원은 짧은 주기로 빠르게 진동하고 높은 차원은 긴 주기로 천천히 변해서, 마치 여러 자릿수의 시계처럼 위치를 인코딩합니다. 사인·코사인을 택한 이유는 학습 때 본 적 없는 더 긴 위치로도 외삽(extrapolation)할 수 있으리라는 기대 때문이었는데, 실제로는 이 절대 위치 방식이 긴 문맥에서 약했고, 이 약점이 T02의 RoPE[8]·ALiBi로 이어지는 직접적인 출발점이 됩니다.

## 트랜스포머 전체 구조

위 부품들을 조립한 전체 구조는 인코더–디코더입니다. 각 층은 다음과 같은 블록의 반복입니다.

```
입력 임베딩 + 위치 인코딩
        │
   ┌────▼─────────────────────┐
   │ Multi-Head Self-Attention│
   └────┬─────────────────────┘
   Add & Norm  (잔차 연결 + LayerNorm)
        │
   ┌────▼─────────────────────┐
   │ Feed-Forward (FFN)       │   ← 위치별 2-layer MLP, 보통 4×d 확장
   └────┬─────────────────────┘
   Add & Norm
        │  (이 블록을 N번 쌓음 — 원논문은 6층)
```

두 개의 보조 장치가 깊은 학습을 떠받칩니다. **잔차 연결(residual connection)** 은 각 서브층의 출력을 입력과 더해(`x + Sublayer(x)`) 기울기가 층을 가로질러 곧장 흐르게 합니다(ResNet의 아이디어). **층 정규화(Layer Normalization)**[10]는 각 토큰 표현의 분포를 안정시켜 학습을 수렴시킵니다. FFN은 각 위치에 독립적으로 적용되는 2층 MLP로, 어텐션이 토큰 사이 정보를 섞는다면 FFN은 각 토큰 안에서 비선형 변환을 담당합니다. 디코더는 여기에 두 가지가 추가됩니다. 미래 토큰을 보지 못하게 막는 **마스킹된 셀프 어텐션**(자기회귀 생성을 위해)과, 인코더의 출력을 참조하는 **인코더–디코더 크로스 어텐션**입니다.

## 왜 바이블이 되었나

이 논문이 단순히 "성능 좋은 번역 모델" 하나로 끝났다면 바이블이 되지 못했을 것입니다. 진짜 임팩트는 세 가지입니다.

첫째, **모든 현대 LLM의 공통 뼈대**가 되었습니다. GPT, BERT, LLaMA, Claude — 이름과 세부는 달라도 골격은 전부 이 트랜스포머입니다. GPT 계열은 디코더만, BERT 계열은 인코더만 떼어 쓰는 식의 변주일 뿐입니다.

둘째, **병렬화가 스케일링의 문을 열었습니다.** 순차 족쇄가 풀리자 모델과 데이터를 수백 배로 키우는 것이 현실적으로 가능해졌고, 이것이 T10의 스케일링 법칙[9]과 거대 LLM 시대로 곧장 이어집니다. 어텐션이 없었다면 GPT-3 규모의 학습은 시간 안에 끝나지 않았을 것입니다.

셋째, 딥러닝 역사상 **가장 널리 인용되고 채택된 논문 중 하나**로 자리 잡았습니다. 정확한 인용 수치를 말하는 것보다, "사실상 거의 모든 후속 연구가 이 위에 섰다"는 사실 자체가 임팩트의 본질을 더 잘 보여 줍니다.

## 한계와 주의점

- **`O(n²)` 비용**: 모든 토큰 쌍의 점수를 계산하므로 문맥 길이가 늘면 연산과 메모리가 제곱으로 폭발합니다. 4K 토큰을 8K로 늘리면 비용은 두 배가 아니라 네 배가 됩니다. 이것이 T02(긴 문맥)와 심화편의 핵심 숙제이며, FlashAttention[11]·희소 어텐션·선형 어텐션이 모두 이 한 줄을 공략합니다.
- **위치 인코딩의 취약성**: 원논문의 절대 위치 방식은 학습 길이를 넘어서는 외삽에 약합니다. RoPE[8]와 ALiBi가 이를 상대 위치 관점에서 고칩니다(T02).
- **귀납적 편향의 부재**: 어텐션은 구조적 가정이 거의 없어 표현력이 크지만, 그만큼 많은 데이터를 요구합니다. 작은 데이터셋에서는 CNN·RNN보다 불리할 수 있습니다.

그리고 바로 이 `O(n²)`이 다음 글 Mamba의 출발점입니다. "어텐션을 더 싸게"가 아니라 "어텐션 없이 갈 수는 없을까?"라는 질문 말이죠.

## 더 읽을거리와 연결

- 이 책 내부: **T02 긴 문맥**(`O(n²)`과 위치 인코딩을 직접 공략), **T03 효율·서빙**(서빙 단계의 어텐션 비용), **T10 스케일링**(병렬화가 연 거대화).
- 기초편 4단계(어텐션 해부)·심화편 S3(어텐션 효율)와 함께 보면 원리부터 최신 우회로까지 입체적으로 잡힙니다.

## 한 줄 요약

> 순환을 버리고 '모든 단어가 모든 단어를 동시에 보는' 셀프 어텐션으로 시퀀스를 모델링해, 병렬화와 장거리 의존성을 한 번에 풀고 모든 현대 LLM의 토대가 된 논문.
