# T03 · 효율·서빙 — 어떻게 싸고 빠르게

거대 모델은 학습만 비싼 것이 아닙니다. 일단 만든 모델을 돌리는 비용, 즉 추론과 서빙의 비용이 실제 사용 가능성을 좌우합니다. 아무리 똑똑한 모델이라도 한 번 질문에 답하는 데 큰 비용이 들면 서비스로 쓰기 어렵습니다. 이 테마는 같은 품질을 더 적은 연산·메모리·비용으로 내는 방법들을 다섯 편의 논문으로 봅니다. 모델 구조, 서빙 메모리, 파인튜닝 비용, 생성 속도라는 네 전선에서 효율을 공략합니다.

## 바이블에서 최신까지, 한눈에

| 분류 | 논문 | 핵심 한 줄 |
|---|---|---|
| 바이블 | Sparsely-Gated MoE (2017) | 토큰별로 일부 전문가만 켜는 조건부 연산 — 모든 MoE의 출발 |
| 바이블 | PagedAttention / vLLM (2023) | OS 페이징식 KV 캐시 관리로 서빙 처리량 2~4배 |
| 최신 | QLoRA (2023) | 4비트 양자화 + LoRA로 단일 GPU 파인튜닝 |
| 최신 | Mixtral (2024) | 오픈 MoE 레퍼런스 — 적은 활성 연산으로 강한 성능 |
| 최신 | Speculative Decoding (2023) | 작은 모델이 추측, 큰 모델이 검증 — 분포 보존 가속 |

## 이 테마를 관통하는 한 문장

모델을 조건부로 희소하게 만들고(MoE), 서빙 메모리를 알뜰히 관리하고(vLLM), 가중치를 압축해 싸게 튜닝하고(QLoRA), 추측·검증으로 빠르게 생성합니다(Speculative). 학습 비용을 넘어 추론과 서빙의 비용을 공략하는 네 갈래입니다. 이들은 서로 독립적이라 함께 쓸 수 있습니다. MoE 모델을 vLLM으로 서빙하면서 speculative decoding으로 가속하는 식의 조합이 실전에서 일어납니다.
