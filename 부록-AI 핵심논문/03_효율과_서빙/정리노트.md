# T03 정리노트 — 계보와 요약

## 계보: 효율의 네 전선

효율은 한 갈래가 아닙니다. MoE(2017)는 모델 구조를 희소하게 만들어(일부 전문가만 켜기) 용량과 연산을 분리했습니다. PagedAttention/vLLM(2023)은 서빙 메모리를 OS 페이징식으로 알뜰히 관리해 처리량을 끌어올렸습니다. QLoRA(2023)는 파인튜닝 비용을 4비트 양자화로 낮춰 단일 GPU 튜닝을 열었고, Mixtral(2024)은 MoE를 공개 강력 모델로 실현했습니다. Speculative Decoding(2023)은 생성 속도를 추측·검증으로 가속하되 품질을 지켰습니다.

한 문장으로: **구조를 희소하게(MoE), 서빙 메모리를 알뜰히(vLLM), 튜닝을 싸게(QLoRA), 생성을 빠르게(Speculative) — 효율의 네 전선.**

| 단계 | 논문 | 분류 | 공략 지점 |
|---|---|:---:|---|
| 1 | Sparsely-Gated MoE (2017) | 바이블 | 모델 구조(조건부 연산) |
| 2 | PagedAttention / vLLM (2023) | 바이블 | 서빙 메모리(KV 캐시 관리) |
| 3 | QLoRA (2023) | 최신 | 파인튜닝 비용(4비트 + 어댑터) |
| 4 | Mixtral (2024) | 최신 | MoE의 공개 실현 |
| 5 | Speculative Decoding (2023) | 최신 | 생성 속도(추측·검증) |

## 한 줄 요약 (논문별)

| 논문 | 분류 | 한 줄 핵심 |
|---|:---:|---|
| Sparsely-Gated MoE (Shazeer et al., 2017) | 바이블 | 토큰별 top-k 전문가만 켜는 조건부 연산 |
| PagedAttention / vLLM (Kwon et al., 2023) | 바이블 | KV 캐시를 페이지 블록으로, 처리량 2~4배 |
| QLoRA (Dettmers et al., 2023) | 최신 | 4비트 NF4 + LoRA로 단일 GPU 파인튜닝 |
| Mixtral (Jiang et al., 2024) | 최신 | 8전문가 중 2개 활성, 오픈 MoE 레퍼런스 |
| Speculative Decoding (Leviathan et al., 2023) | 최신 | 추측·검증으로 분포 보존 2~3배 가속 |

## 효율 기법의 성격 구분

| 기법 | 무엇을 바꾸나 | 품질 영향 |
|---|---|---|
| MoE / Mixtral | 모델 구조(용량↔연산 분리) | 설계에 따라 향상 |
| PagedAttention | 메모리 관리(결과 불변) | 없음 |
| QLoRA | 정밀도 압축(4비트) | 약간 손실 가능 |
| Speculative Decoding | 생성 절차(분포 보존) | 없음 |

## 연결

- 심화편 S7(효율·서빙)의 원전 논문들, 기초편 5단계(추론).
