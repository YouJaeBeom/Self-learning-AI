# 4비트로 단일 GPU 파인튜닝 — QLoRA

> **QLoRA: Efficient Finetuning of Quantized LLMs** — Dettmers et al., 2023, NeurIPS (arXiv:2305.14314)
> 원문: https://arxiv.org/abs/2305.14314 · 분류: 최신

거대 모델을 파인튜닝하려면 보통 여러 장의 고급 GPU가 필요합니다. 그래서 대형 모델을 자기 데이터에 맞게 다듬는 일은 오랫동안 큰 조직의 전유물이었습니다. QLoRA는 이 문을 활짝 엽니다. 단일 GPU 한 장으로 650억 파라미터급 모델을 파인튜닝할 수 있게 만들어, 개인과 소규모 팀도 자기 모델을 다듬는 시대를 열었습니다. 그 비결은 모델을 4비트로 극단적으로 압축해 올리면서도 품질을 거의 잃지 않는 영리한 양자화에 있습니다.

## 배경: 파인튜닝은 왜 그렇게 메모리를 먹나

전체 파인튜닝(full fine-tuning)이 메모리를 많이 쓰는 이유는 가중치만 올리는 게 아니기 때문입니다. 학습하려면 세 가지를 메모리에 둬야 합니다. 모델 가중치, 각 가중치의 기울기, 그리고 옵티마이저 상태(Adam은 가중치당 모멘텀과 분산 두 개를 더 둡니다). 16비트로 65B 모델을 전체 파인튜닝하면 가중치 130GB에 더해 기울기와 옵티마이저 상태까지 합쳐 780GB가 넘는 메모리가 필요합니다. A100 80GB 여러 장이 있어야 한다는 뜻입니다.

이를 줄이는 첫 돌파구가 LoRA(Low-Rank Adaptation)였습니다. LoRA는 원본 가중치 `W`를 고정한 채, 그 옆에 작은 저랭크 행렬 `B·A`만 더해 학습합니다.

```
h = W·x + B·A·x      (W는 고정, B와 A만 학습)
W: d×d (예: 수백만~수십억 개)   vs   B·A: d×r + r×d (r은 보통 8~64로 작음)
```

학습 대상이 전체의 1% 미만으로 줄어 기울기와 옵티마이저 상태가 작아집니다. 그런데 한 가지 부담이 남습니다. 학습은 안 해도 원본 모델 `W` 자체는 여전히 메모리에 올려야 합니다. 65B 모델이면 16비트로도 130GB입니다.

> 핵심 질문: 원본 모델을 극단적으로 압축해 올리면서도, 파인튜닝 품질을 거의 잃지 않을 수 없을까?

## 핵심 아이디어: 4비트로 얼리고, 어댑터만 학습

QLoRA는 원본 모델을 4비트로 양자화해 고정(freeze)하고, 그 위에 LoRA 어댑터만 학습합니다.

> 비유 — 무거운 백과사전을 축소 복사해 두고 메모만 붙이기. 두꺼운 원본(모델)은 압축 복사본(4비트)으로 책장에 꽂아 두고 다시 인쇄하지 않습니다(고정). 새로 배우는 내용은 얇은 포스트잇(LoRA 어댑터)에만 적습니다. 원본을 다시 펼쳐 고쳐 쓰지 않으니 작업 공간이 확 줄어듭니다.

## 어떻게 작동하나: 세 가지 장치

QLoRA는 세 가지 기법을 결합해 단일 48GB GPU에서 65B 파인튜닝을 가능하게 합니다.

**4비트 NormalFloat(NF4) 양자화.** 신경망 가중치는 대체로 평균 0의 정규분포를 따릅니다. NF4는 이 정규분포에 정보이론적으로 최적이 되도록 양자화 구간을 설계한 4비트 데이터 타입입니다. 균등 간격으로 나누는 일반적인 4비트 양자화와 달리, 값이 몰려 있는 0 근처에 더 촘촘한 구간을 배치해 같은 4비트로 더 정확하게 표현합니다.

**이중 양자화(double quantization).** 양자화에는 블록마다 스케일을 맞추는 보조 상수(quantization constant)가 필요한데, 이 상수들도 모이면 적지 않은 메모리를 차지합니다. QLoRA는 이 상수들 자체를 한 번 더 양자화해, 파라미터당 평균 메모리를 추가로 약 0.5비트 절약합니다.

**페이지드 옵티마이저(paged optimizers).** 긴 시퀀스에서 메모리 사용량이 순간적으로 치솟을 때, OS의 페이징처럼 옵티마이저 상태를 CPU 메모리로 잠시 흘려보내 GPU 메모리 초과(OOM) 오류를 방지합니다.

```
얼린 4비트 원본 모델  +  학습되는 LoRA 어댑터(16비트, 작음)
계산 시점에만 4비트 가중치를 16비트로 역양자화해 사용
→ 단일 48GB GPU에서 65B 모델 파인튜닝
```

핵심 결과는 이 극단적 압축에도 불구하고 16비트 전체 파인튜닝에 필적하는 품질을 유지했다는 것입니다. 논문이 이 레시피로 만든 Guanaco 모델군은 당시 벤치마크에서 강한 성능을 보였습니다.

| 방식 | 학습 파라미터 | 원본 모델 메모리 | 65B 파인튜닝 |
|---|---|---|---|
| 전체 파인튜닝(16bit) | 전체 | 130GB+ | 다수 GPU |
| LoRA(16bit 원본) | 어댑터만 | 130GB | 여전히 다수 GPU |
| QLoRA(4bit 원본) | 어댑터만 | 약 33GB | 단일 48GB GPU |

## 임팩트: 무엇을 바꿨나

- **소비자·단일 GPU 파인튜닝의 사실상 표준**이 되었습니다. 개인과 소규모 팀이 대형 모델을 자기 데이터로 다듬는 일이 일상이 되었습니다.
- 오픈 모델 생태계의 맞춤화 폭발을 뒷받침했습니다. 수많은 파인튜닝 변형 모델이 QLoRA 위에서 만들어졌습니다.
- "양자화 + 어댑터"라는 조합이 효율적 후학습(post-training)의 기본기로 자리 잡았습니다.

## 한계와 주의점

- 양자화는 과제에 따라 약간의 품질 손실을 동반할 수 있습니다. NF4가 손실을 크게 줄이긴 하지만, 정밀도가 결정적인 작업에서는 신중히 검증해야 합니다.
- QLoRA는 파인튜닝 효율 기법입니다. 처음부터 학습하는 사전학습(pre-training)을 대체하지 않습니다.
- 양자화 추론(PTQ) 쪽 계보로는 마스터의 편집 메모대로 GPTQ[5](Frantar et al., 2022, arXiv:2210.17323)와 AWQ[6](Lin et al., 2023, arXiv:2306.00978)가 보조 참고입니다. QLoRA가 파인튜닝을 위한 양자화라면, 이들은 배포용 추론을 위한 양자화입니다.

## 한 줄 요약

> QLoRA는 원본 모델을 4비트 NF4로 얼려 올리고 작은 LoRA 어댑터만 학습해, 단일 GPU로 수백억급 모델을 거의 손실 없이 파인튜닝한다. 대형 모델 튜닝을 누구나 할 수 있게 만들었다.
