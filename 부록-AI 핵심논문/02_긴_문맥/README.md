# T02 · 긴 문맥 — 어떻게 더 길게 보나

트랜스포머는 강력하지만 문맥 길이 앞에서 두 개의 벽을 만납니다. 하나는 위치를 어떻게 알려 줄 것인가(학습보다 긴 길이로 일반화하는 문제)이고, 다른 하나는 어텐션의 `O(n²)` 비용(메모리와 속도)입니다. 이 테마는 그 두 벽을 차례로 허무는 다섯 편의 논문을 따라갑니다. 위치를 똑똑하게 주는 두 바이블에서 시작해, 단일 칩의 메모리 병목을 푸는 커널, 그리고 여러 장치로 확장하는 분산까지 이어집니다.

## 바이블에서 최신까지, 한눈에

| 분류 | 논문 | 핵심 한 줄 |
|---|---|---|
| 바이블 | RoPE / RoFormer (2021) | Query·Key를 회전시켜 상대 위치를 자연스럽게 — 사실상 표준 |
| 바이블 | ALiBi (2021) | 위치 임베딩 대신 거리 비례 패널티로 길이 외삽 |
| 최신 | FlashAttention (2022) | IO 인식 타일링으로 정확 어텐션을 빠르게, 메모리 선형 |
| 최신 | FlashAttention-3 (2024) | 최신 GPU(Hopper)의 비동기·FP8로 1.5~2배 가속 |
| 최신 | Ring Attention (2023) | 디바이스 분산으로 초장문맥을 근사 없이 |

## 이 테마를 관통하는 한 문장

먼저 위치를 똑똑하게 주고(RoPE·ALiBi), 그다음 단일 칩의 메모리 병목을 풀고(FlashAttention), 마지막으로 여러 장치에 나눠 초장문맥을 엽니다(Ring). 두 벽 — 위치 일반화와 `O(n²)` 비용 — 을 순서대로 공략하는 이야기입니다. 중요한 것은 이 다섯 기법이 경쟁이 아니라 **층층이 쌓이는 보완**이라는 점입니다. 실전 긴 문맥 모델은 RoPE로 위치를 주고, FlashAttention 커널로 계산하고, 필요하면 Ring으로 분산하는 식으로 이들을 함께 씁니다.
