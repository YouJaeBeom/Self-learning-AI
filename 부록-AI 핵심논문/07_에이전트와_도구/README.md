# T07 · 에이전트·도구 — 말하는 모델을 행동하게

LLM은 말은 잘하지만 최신 정보를 모르고, 정확한 계산에 약하며, 실제로 무언가를 실행하지 못합니다. 똑똑한 두뇌가 손발 없이 갇혀 있는 셈입니다. 이 테마는 그 모델을 추론하고 도구를 쓰고 행동하는 에이전트로 키운 계보를 따라갑니다. 생각과 행동을 엮는 기본 루프에서 출발해, 도구 사용을 학습으로 내재화하고, 실패에서 배우게 하고, 마침내 그 능력을 현실적으로 측정하는 데까지 이어집니다.

## 바이블에서 최신까지, 한눈에

| 분류 | 논문 | 핵심 한 줄 |
|---|---|---|
| 바이블 | ReAct (Yao et al., 2022/23) | 추론과 행동을 번갈아 — 에이전트의 기본 루프 |
| 바이블 | Toolformer (Schick et al., 2023) | 모델이 스스로 도구(API) 호출을 학습 |
| 최신 | Reflexion (Shinn et al., 2023) | 실패를 말로 복기해 다음 시도를 개선 |
| 최신 | SWE-bench (Jimenez et al., 2023) | 실제 GitHub 이슈로 에이전트를 측정 |

## 이 테마를 관통하는 한 문장

ReAct가 생각→행동→관찰 루프로 에이전트의 골격을 세우자, 그 루프 안의 도구 사용을 어떻게 잘하느냐가 과제가 되었습니다(Toolformer). 그다음은 실패에서 배우는 법(Reflexion)과 그 능력을 제대로 재는 법(SWE-bench)으로 이어집니다. 모델을 똑똑하게 만드는 것을 넘어 일을 해내게 만드는 공학의 흐름입니다.
