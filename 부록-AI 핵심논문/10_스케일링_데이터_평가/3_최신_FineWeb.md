# 무엇을 먹일까 — FineWeb

> **The FineWeb Datasets: Decanting the Web for the Finest Text Data at Scale** — Penedo et al. (Hugging Face), 2024, NeurIPS D&B (arXiv:2406.17557)
> 원문: https://arxiv.org/abs/2406.17557 · 분류: 최신

Chinchilla[2]가 "데이터를 충분히 먹여라"를 말했다면, 자연스러운 다음 질문은 "무엇을 먹일까"입니다. 같은 양이라도 질이 다르면 모델이 달라집니다. 그런데 강력한 모델들의 학습 데이터 레시피는 대부분 비공개라, 무엇이 좋은 데이터인지 연구하기 어려웠습니다. FineWeb은 대규모 공개 사전학습 데이터를 만들면서, 어떤 필터링과 중복 제거가 실제로 효과 있는지를 ablation 실험으로 투명하게 공개했습니다. 비밀 레시피의 시대에 재현 가능한 공개 표준을 제시한 것입니다.

## 배경: 비공개 레시피의 시대

데이터가 중요하다는 것은 Chinchilla 이후 분명해졌습니다. 하지만 정작 "어떤 데이터가 좋은가"는 불투명했습니다. GPT, Gemini, Claude 같은 강력한 모델들의 학습 데이터 구성과 정제 방법은 영업 비밀이었습니다. 연구자들은 데이터 정제의 어떤 단계가 실제로 성능을 올리는지 체계적으로 알 수 없었습니다. "중복을 제거하면 좋다"거나 "품질 필터가 중요하다" 같은 직관은 있었지만, 그것을 대규모로 검증한 공개 연구가 드물었습니다.

> 핵심 질문: 대규모 웹 데이터를 어떻게 거르고 중복 제거해야 좋은 사전학습 데이터가 되는가, 그리고 그것을 투명하게 공개할 수 없을까?

## 핵심 아이디어: 투명한 큐레이션과 검증

FineWeb의 핵심은 데이터 큐레이션 과정을 ablation으로 검증하고 공개한 것입니다. 방대한 웹 크롤(Common Crawl)을 품질 필터링과 중복 제거를 거쳐 15조 토큰 규모의 공개 데이터 FineWeb으로 만들고, 거기서 교육적 텍스트를 추린 FineWeb-Edu도 함께 공개했습니다.

> 비유 — 재료 손질법을 공개한 레시피북. "이 재료는 씻고, 이건 골라내고, 중복은 버린다. 그러면 맛이 이만큼 는다"를 근거와 함께 공유한 것입니다. 어느 손질 단계가 맛을 얼마나 올리는지 하나하나 비교 실험으로 보여 주니, 누구나 따라 하고 개선할 수 있습니다.

## 어떻게 작동하나

```
대규모 웹 크롤
→ 품질 필터링 (저품질·스팸·기계 생성 텍스트 제거)
→ 중복 제거 (dedup: 같거나 비슷한 문서 제거)
→ (교육 텍스트 선별 = FineWeb-Edu)
각 단계의 효과를 ablation으로 측정·공개
```

핵심 교훈은 데이터의 양만이 아니라 질과 정제가 성능을 좌우한다는 것입니다. 특히 중복 제거와 품질 필터의 효과가 컸습니다. 흥미롭게도 FineWeb-Edu처럼 교육적 가치가 높은 텍스트를 추리면, 더 적은 데이터로도 더 나은 성능을 내는 경우가 있었습니다. 같은 양을 먹여도 무엇을 먹이느냐가 다른 결과를 낳는 것입니다. 공개 데이터에 공개 절차이므로, 누구나 재현하고 개선할 수 있습니다.

| 정제 단계 | 목적 | 효과 |
|---|---|---|
| 품질 필터링 | 저품질 텍스트 제거 | 성능 향상 |
| 중복 제거(dedup) | 반복 학습 방지 | 성능 향상 |
| 교육 텍스트 선별 | 고가치 데이터 집중 | 효율 향상 |

## 임팩트: 무엇을 바꿨나

- **공개 사전학습 데이터 큐레이션의 표준**이 되었습니다. 폐쇄적이던 데이터 레시피를 투명한 공개 자산으로 끌어내, 오픈 LLM 생태계의 토대가 되었습니다.
- Chinchilla의 "충분한 데이터"에 "좋은 데이터"라는 축을 더해, 데이터 중심 연구를 끌어올렸습니다. 모델 구조 혁신만큼이나 데이터 품질이 중요하다는 인식을 굳혔습니다.
- 합성 데이터(심화편 S5)와 함께, "모델이 아니라 데이터가 다음 도약의 열쇠"라는 흐름을 대표합니다.

## 한계와 주의점

- 주로 영어 웹 텍스트 중심이라, 다국어나 전문 도메인은 별도 큐레이션이 필요합니다. 한국어 등 다른 언어에는 그대로 적용되지 않습니다.
- "좋은 데이터"의 기준은 목표와 평가에 의존합니다. 한 벤치마크에 맞춘 필터가 다른 데서는 손해일 수 있습니다.
- 웹 데이터의 고갈, 저작권, 편향 문제는 여전히 남은 과제입니다. 양질의 공개 텍스트가 무한하지 않다는 점이 장기적 제약입니다.

> FineWeb은 대규모 공개 사전학습 데이터를 만들며 어떤 필터와 중복 제거가 효과 있는지를 ablation으로 투명하게 공개해, 데이터 큐레이션의 표준이 되었다.
