# T10 · 스케일링·데이터·평가 — 키우고, 먹이고, 재다

LLM은 "크게 만들수록 똑똑해진다"는 경험칙 위에 서 있습니다. 그런데 세 가지 질문이 남습니다. 얼마나, 어떤 비율로 키워야 하나? 무엇을 먹여야 하나? 그리고 제대로 늘었는지 어떻게 재나? 이 테마는 대형화의 법칙(바이블)에서 출발해 최신 데이터 큐레이션과 평가까지, LLM 개발의 세 기둥을 따라가며 이 책 전체를 닫습니다.

## 바이블에서 최신까지, 한눈에

| 분류 | 논문 | 핵심 한 줄 |
|---|---|---|
| 바이블 | Scaling Laws (Kaplan et al., 2020) | 손실이 모델·데이터·연산의 멱법칙 |
| 바이블 | Chinchilla (Hoffmann et al., 2022) | 모델:토큰을 동등 비율로 키워야 최적 |
| 최신 | FineWeb (Penedo et al., 2024) | 공개 사전학습 데이터 큐레이션의 표준 |
| 최신 | MT-Bench / LLM-judge (Zheng et al., 2023) | LLM 심판으로 평가를 자동화 |
| 최신 | MMLU (Hendrycks et al., 2020/21) | 57과제 지식·추론 벤치(기초 성격) |

## 이 테마를 관통하는 한 문장

이 테마는 LLM 개발의 세 기둥 — 키우고(스케일링), 먹이고(데이터), 재다(평가) — 를 따라갑니다. 법칙으로 대형화를 정당화하고(Scaling Laws), 그 배분을 균형으로 교정하고(Chinchilla), 데이터의 질을 투명하게 다루고(FineWeb), 마침내 "정말 좋아졌나"를 묻는 평가(MT-Bench·MMLU)로 마무리합니다.
