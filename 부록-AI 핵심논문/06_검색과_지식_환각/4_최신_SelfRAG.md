# 검색을 스스로 판단하다 — Self-RAG

> **Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection** — Asai et al., 2023, ICLR 2024 Oral (arXiv:2310.11511)
> 원문: https://arxiv.org/abs/2310.11511 · 분류: 최신

기존 RAG[2]는 질문이 오면 무조건 검색하고, 가져온 문서를 무조건 끼워 답했습니다. 그런데 어떤 질문에는 검색이 불필요하고("1 더하기 1은?"), 가져온 문서가 엉뚱하거나 답과 어긋날 수도 있습니다. 검색을 항상 켜 두고 그 결과를 무비판적으로 쓰는 것은 때로 느리고 때로 해롭습니다. Self-RAG는 그 판단을 모델이 스스로 하게 만듭니다. 언제 검색할지, 가져온 문서가 근거로 타당한지를 모델이 직접 비평하는 적응형 RAG입니다.

## 배경: 무조건 검색의 두 가지 문제

기존 RAG에는 두 가지 맹점이 있습니다. 첫째, 불필요한 검색입니다. 상식 질문이나 모델이 이미 잘 아는 질문에도 검색하면 느려지고, 관련 없는 문서가 오히려 답을 흐립니다. 둘째, 무비판적 사용입니다. 가져온 문서가 틀렸거나 답을 뒷받침하지 않아도 그대로 끼워 넣으면 환각이 그대로 남습니다. 검색이 답의 근거가 되는지를 아무도 확인하지 않는 것입니다. 요약하면 기존 RAG는 언제 검색할지도, 검색 결과가 답을 정말 뒷받침하는지도 따지지 않았습니다.

> 핵심 질문: 모델이 "지금 검색이 필요한가", "이 문서가 근거로 맞는가"를 스스로 비평하게 할 수 없을까?

## 핵심 아이디어: 반성 토큰

Self-RAG의 핵심은 반성 토큰(reflection token)입니다. 모델이 답을 생성하면서, 특별한 토큰을 출력해 자기 행동을 스스로 평가하도록 학습됩니다. 평가는 네 가지를 다룹니다.

- 검색 필요 여부: 지금 검색이 필요한가?
- 관련성: 가져온 문서가 질문과 관련 있는가?
- 근거 부합(충실성): 내 답이 그 문서에 실제로 근거하는가?
- 유용성: 이 답이 질문에 쓸모 있는가?

> 비유 — 스스로 점검하는 오픈북 응시자. "이 문제는 책을 펴야 하나? 필요 없으면 그냥 답하고, 폈다면 이 페이지가 맞는 자료인가? 내 답이 정말 이 페이지에 적힌 내용에 근거하나?"를 매 단계 자문합니다. 책을 펴는 것과 펴서 제대로 쓰는 것을 모두 점검하는 것입니다.

## 어떻게 작동하나

```
질문 → [검색이 필요한가?] ──아니오→ 그냥 생성
                          └─예→ 검색 → 각 문서에 대해
                                  [관련 있나?] [내 답이 근거하나?] 평가
                                  → 가장 근거 있는 답 + 자기 평가 점수
```

중요한 점은 검색, 생성, 비평을 별도의 모듈이 아니라 같은 모델이 한 흐름에서 수행한다는 것입니다. 모델이 일반 토큰과 반성 토큰을 함께 출력하도록 학습되어, 생성 도중에 스스로 검색 여부와 근거 부합을 판단합니다. 이 자기 평가 점수로 여러 답 후보 중 가장 근거 있는 것을 고르거나 검색 여부를 조절합니다. 검색을 항상 켜 두는 것이 아니라 필요할 때, 검증하며 쓰는 적응형 RAG가 되는 것입니다.

| 항목 | 기존 RAG | Self-RAG |
|---|---|---|
| 검색 시점 | 항상 | 필요할 때(스스로 판단) |
| 근거 검증 | 없음 | 반성 토큰으로 자기 점검 |
| 구조 | 검색 + 생성 | 검색 + 생성 + 비평(한 모델) |

## 임팩트: 무엇을 바꿨나

- **적응형·자기반성 RAG 계열의 대표작**입니다. "검색은 항상 켜 두는 것이 아니라 필요할 때, 검증하며"라는 관점을 정착시켰습니다.
- 검색의 충실성(답이 근거에 부합하는가)을 모델 내부 신호로 끌어들여, 환각 완화의 새 축을 열었습니다. 검색을 했느냐만이 아니라 검색 결과를 제대로 썼느냐를 본 것입니다.
- 이후 "언제 무엇을 검색하고 어떻게 검증할지"를 다루는 적응형 RAG 연구의 기준점이 되었습니다. 자기비평이라는 발상은 T07의 Reflexion과도 통합니다.

## 한계와 주의점

- 반성 토큰을 학습시키려면 그런 평가가 달린 학습 데이터가 필요합니다. 이 데이터를 구축하는 비용이 듭니다.
- 모델의 자기 평가가 틀릴 수 있습니다. 스스로 "근거 충분"이라고 착각하면 환각이 그대로 통과됩니다. 이는 LLM이 자기 출력을 평가할 때의 근본 한계(T10의 LLM-as-judge 한계)와 같은 뿌리입니다.
- 단순한 사실 검색에는 기존 RAG로 충분할 수 있어, 복잡한 질의에서 값을 합니다.

## 한 줄 요약

> Self-RAG는 반성 토큰으로 언제 검색할지와 근거가 맞는지를 모델이 스스로 비평하게 한 적응형 RAG로, 무조건 검색과 무비판 사용의 빈틈을 메웠다.
