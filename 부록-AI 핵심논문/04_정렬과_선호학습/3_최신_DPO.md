# 보상 모델을 건너뛰다 — DPO

> **Direct Preference Optimization: Your Language Model is Secretly a Reward Model (DPO)** — Rafailov et al., 2023, NeurIPS (arXiv:2305.18290)
> 원문: https://arxiv.org/abs/2305.18290 · 분류: 최신

InstructGPT의 RLHF[2]는 강력하지만 복잡하고 불안정합니다. 별도의 보상 모델을 학습하고, 튜닝이 까다로운 강화학습(PPO)을 돌려야 하며, 이 과정에서 학습이 자주 흔들립니다. DPO는 이 두 무거운 부품을 통째로 건너뜁니다. 선호 데이터로 모델을 직접 학습해, RLHF가 도달하려던 바로 그 지점에 더 단순하고 안정적으로 닿습니다. 제목 "당신의 언어 모델은 사실 보상 모델이다"가 이 논문의 핵심 통찰을 압축합니다.

## 배경: RLHF는 왜 그렇게 복잡한가

RLHF 파이프라인의 복잡성은 두 가지에서 옵니다. 첫째, 보상 모델을 따로 학습해 유지해야 합니다. 둘째, 그 보상을 최적화하는 PPO는 온폴리시 강화학습이라 학습 중 계속 새 샘플을 생성하고, 보상 모델·정책·참조 모델을 동시에 메모리에 올려야 하며, 하이퍼파라미터에 민감합니다. 실무자들에게 RLHF는 "되면 강력하지만 돌리기 까다로운" 기법이었습니다.

> 핵심 질문: 보상 모델과 강화학습이라는 중간 단계 없이, 선호 데이터로 바로 정렬할 수 없을까?

## 핵심 아이디어: 언어 모델이 곧 보상 모델

DPO의 통찰은 보상 모델을 따로 세울 필요가 없다는 것입니다. 선호 정보를 하나의 분류 손실로 바꿔 모델을 직접 밀면 됩니다.

> 비유 — 평론가를 거치지 않는 직접 교육. RLHF가 맛 평론가(보상 모델)를 키우고 그 점수로 요리사를 훈련했다면, DPO는 "손님이 A를 B보다 좋아했다"는 기록을 요리사에게 곧장 주며 "이런 걸 더, 저런 걸 덜"이라고 가르칩니다. 중간의 평론가를 없앤 것입니다.

## 수식으로 들어가기: 어떻게 보상 모델이 사라지는가

이 통찰의 핵심은 RLHF의 최적해를 들여다보는 데 있습니다. RLHF가 푸는 KL 제약 보상 최대화 문제의 최적 정책은 닫힌 형태로 알려져 있습니다.

```
π*(y|x) = (1/Z(x)) · π_ref(y|x) · exp( r(x,y) / β )
```

이 식을 보상 `r`에 대해 풀면(양변에 로그를 취해 정리하면), 보상을 정책으로 표현할 수 있습니다.

```
r(x,y) = β · log( π*(y|x) / π_ref(y|x) ) + β · log Z(x)
```

즉 보상은 "정책이 참조 모델 대비 이 답을 얼마나 더 좋아하는가"의 로그 비율로 나타납니다. 이제 이 표현을 보상 모델 학습에 쓰던 Bradley-Terry 선호 손실에 대입하면, 다루기 까다로운 정규화 항 `Z(x)`가 선호 비교 과정에서 상쇄되어 사라지고, 다음의 깔끔한 DPO 손실이 남습니다.

```
L_DPO = − log σ( β·log(π_θ(y_w|x)/π_ref(y_w|x)) − β·log(π_θ(y_l|x)/π_ref(y_l|x)) )
```

이 손실은 선호된 답 `y_w`의 확률(참조 대비)은 올리고 거부된 답 `y_l`의 확률은 내리되, `β`가 참조 모델에서 벗어나는 정도를 조절합니다. 별도 보상 모델도, 강화학습 루프도 없이, 선호쌍에 대한 지도학습형 손실 하나로 RLHF가 풀려던 문제를 직접 풉니다. `β`에 RLHF의 KL 제약이 그대로 녹아 있다는 점도 주목할 만합니다.

| 항목 | RLHF (InstructGPT) | DPO |
|---|---|---|
| 보상 모델 | 따로 학습·유지 | 불필요(정책에 흡수) |
| 학습 방식 | 강화학습(PPO) | 지도학습에 가까움 |
| 안정성 | 민감·불안정 | 상대적으로 안정 |
| 한 줄 | 채점기를 만들고 점수를 올린다 | 선호를 손실로 바꿔 바로 배운다 |

## 임팩트: 무엇을 바꿨나

- **정렬을 훨씬 쉽게** 만들어 널리 채택되었습니다. Llama 3, Tülu 등 여러 오픈 모델의 정렬 단계에 쓰였습니다.
- KTO, IPO, GRPO 등 직접 선호 최적화 계열의 폭발을 촉발했습니다. 다음 글 GRPO[4]도 이 흐름의 일부입니다.
- "정렬은 복잡한 강화학습이어야 한다"는 통념을 깨고, 지도학습에 가까운 길을 열었습니다. 진입 장벽을 크게 낮춘 것입니다.

## 한계와 주의점

- **DPO가 RLHF보다 항상 낫지는 않습니다.** 더 단순하고 안정적이지만, RLHF가 더 유연하고 세밀한 제어에 유리한 경우도 있습니다. 둘은 같은 목표를 다른 길로 갑니다.
- 선호 데이터의 품질에 민감하고, 학습 분포에서 너무 벗어난 정책에서는 약점이 보고됩니다. 온폴리시 샘플링이 없다는 점이 양날의 검입니다.
- 변형으로는 마스터 메모의 KTO[5](Ethayarajh et al., 2024, arXiv:2402.01306)와 IPO(Azar et al., 2023/24)가 있습니다.

## 한 줄 요약

> DPO는 보상 모델과 강화학습을 건너뛰고 선호쌍을 하나의 분류 손실로 바꿔 모델을 직접 정렬한다. "언어 모델은 그 자체로 보상 모델"이라는 통찰로 정렬을 단순하고 안정적으로 만들었다.
