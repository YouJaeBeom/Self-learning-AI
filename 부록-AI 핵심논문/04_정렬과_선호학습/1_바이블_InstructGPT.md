# RLHF 3단계의 정석 — InstructGPT

> **Training language models to follow instructions with human feedback (InstructGPT)** — Ouyang et al., 2022, NeurIPS (arXiv:2203.02155)
> 원문: https://arxiv.org/abs/2203.02155 · 분류: 바이블

사전학습만 끝낸 언어 모델은 인터넷 글을 흉내 낼 뿐입니다. 질문을 던지면 답을 하는 대신 또 다른 질문을 이어 쓰거나, 그럴듯하지만 무례하거나 위험한 말을 하기도 합니다. 다음 단어를 잘 맞히도록 학습됐을 뿐, 사람을 돕도록 학습된 적이 없기 때문입니다. 이 논문은 그런 모델을 사람의 지시를 따르는 조력자로 바꾸는 RLHF(인간 피드백 기반 강화학습) 3단계 레시피를 확립했습니다. ChatGPT를 비롯한 거의 모든 상용 챗봇 정렬의 원형이 바로 이 한 편입니다.

## 배경: 능력과 유용함은 다르다

핵심 통찰은 능력(capability)과 유용함(helpfulness)이 별개라는 것입니다. "다음 단어를 잘 맞히는 것"과 "사람에게 도움이 되고 안전하게 답하는 것"은 다른 목표입니다. 사전학습은 전자를 극대화하지만, 우리가 챗봇에 원하는 것은 후자입니다.

문제는 "좋은 답"을 규칙으로 적기가 매우 어렵다는 점입니다. 무엇이 도움이 되고 무엇이 무례한지를 코드로 명세할 수 없습니다. 이것은 "고양이를 픽셀 규칙으로 정의할 수 없어서 예시로 배우게 한" 딥러닝의 출발점과 같은 구조입니다. 규칙을 못 적으면 예시로 배워야 하고, 여기서 예시는 사람의 선호입니다.

> 핵심 질문: 모델을 사람이 원하고, 도움이 되며, 안전한 답에 어떻게 맞출까?

## 핵심 아이디어: 사람의 선호를 모아 배운다

같은 질문에 모델이 여러 답을 내놓게 하고, 사람이 어느 쪽이 더 나은지 고른 데이터를 모읍니다. 그리고 모델을 그 선호 쪽으로 끌어당깁니다.

> 비유 — 요리사 길들이기. 재료(지식)는 다 아는 요리사에게, 손님(사람)이 A 접시를 B보다 좋아한다는 평가를 모아 줍니다. 요리사는 그 선호에 맞춰 간을 다듬어 갑니다. 모든 손님의 입맛을 일일이 규칙으로 적지 않고도, 비교 평가만으로 방향을 잡는 것입니다.

## 어떻게 작동하나: 세 단계

```
① 지도 파인튜닝(SFT) : 사람이 쓴 모범 답안으로 기본 대화법을 지도학습
② 보상 모델(RM) 학습 : 사람의 선호 비교(A > B)로, 답에 점수를 매기는 모델을 학습
③ 강화학습(PPO)      : 모델이 답 생성 → RM이 점수 → 점수가 오르도록 정책을 갱신
```

**1단계, SFT.** 사람이 직접 쓴 모범 답안으로 사전학습 모델을 파인튜닝해, 일단 질문에 답하는 기본 말투를 잡습니다.

**2단계, 보상 모델.** 한 질문에 여러 답을 생성해 사람에게 순위를 매기게 하고, 이 비교 데이터로 답에 점수를 매기는 보상 모델 `r(x, y)`를 학습합니다. 학습에는 Bradley-Terry 모델이 쓰입니다. 사람이 답 `y_w`를 `y_l`보다 선호했다면, 보상 모델이 그 선호를 재현하도록 다음 손실을 최소화합니다.

```
L_RM = − log σ( r(x, y_w) − r(x, y_l) )
```

`σ`는 시그모이드입니다. 선호된 답의 점수가 거부된 답보다 높을수록 손실이 작아집니다. 즉 보상 모델은 사람의 취향을 흉내 낸 자동 채점기가 됩니다.

**3단계, PPO.** 이제 모델(정책 `π`)이 답을 생성하면 보상 모델이 점수를 매기고, 그 점수를 높이도록 강화학습으로 정책을 갱신합니다. 목적 함수는 다음과 같습니다.

```
maximize  E[ r(x, y) ]  −  β · KL( π_θ(y|x) ‖ π_ref(y|x) )
```

여기서 두 번째 항이 핵심 안전장치입니다. 정책이 원래 SFT 모델 `π_ref`에서 너무 멀어지지 않도록 KL 발산으로 고삐를 매는 것입니다. 이 제약이 없으면 모델이 보상 점수만 노리다가 이상한 말투로 망가지거나(보상 해킹), 다양성을 잃고 같은 문구만 반복하게 됩니다. `β`는 이 고삐의 세기를 조절합니다.

| 단계 | 입력 | 산출물 | 학습 방식 |
|---|---|---|---|
| SFT | 모범 답안 | 기본 대화 모델 | 지도학습 |
| RM | 선호 비교 | 자동 채점기 | Bradley-Terry |
| PPO | RM 점수 | 정렬된 모델 | 강화학습 + KL |

이 논문의 가장 인상적인 결과는, 잘 정렬된 1.3B 모델이 정렬되지 않은 175B GPT-3보다 사람 선호 평가에서 앞섰다는 것입니다. 100배 이상 작은 모델이 이긴 셈인데, 이는 정렬이 크기 못지않게 중요하다는 것을 강하게 보여 줍니다.

## 임팩트: 왜 바이블이 되었나

- **모든 상용 정렬의 레시피**가 되었습니다. SFT → RM → PPO라는 3박자는 이후 거의 모든 챗봇 학습의 표준 출발점이 되었습니다.
- "사전학습 → 파인튜닝 → 정렬"이라는 LLM 제작의 큰 그림에서 마지막 박자를 정립했습니다.
- ChatGPT가 단순한 텍스트 생성기가 아니라 대화 상대처럼 느껴지는 비결이 바로 이 정렬 단계입니다. 정렬을 막연한 개념에서 재현 가능한 공학으로 만든 분기점입니다.

## 한계와 주의점

- **RLHF는 복잡하고 불안정**합니다. 보상 모델을 따로 학습하고, 튜닝이 까다로운 PPO를 돌려야 합니다. 이 복잡성을 줄이려는 시도가 다음의 DPO[3]입니다.
- **보상 해킹**: 보상 모델은 사람 취향의 근사일 뿐이라, 모델이 실제로 좋은 답 대신 보상 모델의 빈틈을 파고드는 헛수를 둘 수 있습니다.
- **정렬 세금(alignment tax)**: 너무 강하게 정렬하면 원래의 능력이 약간 깎이기도 합니다. 유용함과 능력 사이의 균형이 필요합니다.

## 한 줄 요약

> InstructGPT는 SFT → 보상 모델 → PPO의 RLHF 3단계로 모델을 사람이 원하고 안전한 답에 맞추는 레시피를 확립했다. 잘 정렬된 작은 모델이 거대 모델을 이겨 정렬의 중요성을 입증한, 모든 상용 정렬의 원형이다.
