# T04 정리노트 — 계보와 요약

## 계보: 단순화와 확장의 행진

정렬은 "좋은 답을 규칙으로 못 적으니 선호로 배운다"에서 출발합니다. InstructGPT(2022)가 SFT → 보상 모델 → PPO의 RLHF 3단계를 확립했고, Constitutional AI(2022)는 사람 대신 AI 피드백(RLAIF)으로 확장했습니다. 그다음은 단순화의 행진입니다. DPO(2023)가 보상 모델을 없앴고, GRPO(2024)가 크리틱까지 덜어 냈습니다. 마지막으로 DeepSeek-R1(2025)이 보상을 검증 가능한 정답에 걸어, 정렬을 능력(추론) 향상으로 확장했습니다.

한 문장으로: **사람 RL(InstructGPT) → AI 피드백(CAI) → 보상 모델 제거(DPO) → 크리틱 제거(GRPO) → 정답 보상으로 능력 확장(R1).**

| 단계 | 논문 | 분류 | 계보적 역할 |
|---|---|:---:|---|
| 1 | InstructGPT (2022) | 바이블 | RLHF 3단계 확립 |
| 2 | Constitutional AI (2022) | 바이블 | AI 피드백(RLAIF)으로 확장 |
| 3 | DPO (2023) | 최신 | 보상 모델 없이 직접 최적화 |
| 4 | GRPO (2024) | 최신 | 크리틱 없는 가벼운 강화학습 |
| 5 | DeepSeek-R1 (2025) | 최신 | 정렬을 추론 인센티브로 (Nature) |

## 한 줄 요약 (논문별)

| 논문 | 분류 | 한 줄 핵심 |
|---|:---:|---|
| InstructGPT (Ouyang et al., 2022) | 바이블 | SFT→RM→PPO, 작은 정렬 모델이 거대 모델을 이김 |
| Constitutional AI (Bai et al., 2022) | 바이블 | 헌법으로 자기비평, AI 선호 보상(RLAIF) |
| DPO (Rafailov et al., 2023) | 최신 | 보상 모델 없이 선호쌍 직접 최적화 |
| GRPO (Shao et al., 2024) | 최신 | 그룹 내 상대 보상으로 크리틱 제거 |
| DeepSeek-R1 (DeepSeek-AI, 2025) | 최신 | 검증 가능한 보상으로 정렬→능력 확장, Nature |

## 무엇을 덜어 냈나

| 논문 | 없앤 부품 | 남긴 것 |
|---|---|---|
| InstructGPT | (기준선) | 보상 모델 + PPO + 크리틱 |
| DPO | 보상 모델, 강화학습 루프 | 선호쌍 + 분류 손실 |
| GRPO | 크리틱(가치함수) | 강화학습 + 그룹 상대 보상 |

## 연결

- 기초편 6단계(정렬)·심화편 S5(정렬·후학습)의 원전들. DPO 변형은 마스터 메모의 KTO(arXiv:2402.01306)·IPO.
- 이 책 내부: T05 추론·테스트타임에서 DeepSeek-R1을 순수 RL로 추론이 창발하는 관점으로 다시 봅니다.
