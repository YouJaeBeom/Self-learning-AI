# 연합 + 거대 LLM — FederatedScope-LLM

> **FederatedScope-LLM: A Comprehensive Package for Fine-tuning Large Language Models in Federated Learning** — Kuang et al., 2024, KDD (arXiv:2309.00363)
> 원문: https://arxiv.org/abs/2309.00363 · 분류: 최신

LLM은 공개 데이터로 사전학습되지만, 정말 가치 있는 데이터 — 병원, 기업, 법률 — 는 모을 수 없습니다. 프라이버시와 규제 때문이죠. 연합학습으로 이 데이터에 LLM을 파인튜닝하려는 순간, 거대한 모델 크기가 벽이 됩니다. 수십억 파라미터를 매 라운드 주고받을 수는 없습니다. 이 논문은 그 벽을 넘는 종합 패키지를 제시해, 연합학습을 파운데이션 모델 시대로 이끌었습니다.

## 배경: 규모의 벽과 생태계의 부재

연합 LLM 파인튜닝에는 두 가지 장벽이 있습니다. 첫째, 규모의 벽입니다. 수십억 파라미터 LLM을 FedAvg[2]로 그대로 연합하면, 매 라운드 모델 전체를 주고받아야 해 통신과 계산이 폭발합니다. 70억 파라미터 모델이면 한 번 주고받는 데만 수십 GB가 오갑니다. 모바일은커녕 기관 간에도 비현실적입니다. 둘째, 생태계의 부재입니다. 연합 LLM 파인튜닝을 재현하고 비교할 표준 데이터셋, 도구, 파이프라인이 없었습니다. 각자 따로 구현하면 결과를 비교할 수 없습니다.

> 핵심 질문: 거대 LLM을, 데이터를 모으지 않고, 현실적인 통신 비용으로 여러 기관이 함께 파인튜닝하게 하려면 무엇이 필요할까?

## 핵심 아이디어: 어댑터만 교환하고, 도구를 묶는다

두 축으로 풉니다. 첫째, PEFT(파라미터 효율적 파인튜닝) 통합입니다. 거대 모델 전체가 아니라 작은 어댑터(LoRA 등) 파라미터만 학습하고 주고받습니다. 통신량이 극적으로 줄어 연합이 현실적이 됩니다. 둘째, 종합 패키지입니다. 연합 LLM 파인튜닝을 위한 데이터셋, 벤치마크, 파이프라인, 도구를 한데 묶어, 연구와 실무가 바로 쓰고 비교할 수 있게 합니다.

> 비유 — 두꺼운 매뉴얼의 '수정 스티커'만 교환. 표준 매뉴얼(LLM) 전체를 다시 인쇄해 돌리는 대신, "여기 이렇게 고치세요"라는 얇은 스티커(LoRA)만 각 기관이 만들어 모읍니다. 본문은 그대로 두고 스티커만 합치니, 주고받는 양이 수백 분의 1로 줄어듭니다. LoRA가 본체를 얼리고 작은 어댑터만 학습하는 것을 떠올리면 됩니다.

## 어떻게 작동하나

```
① PEFT 기반 로컬 파인튜닝: 각 클라이언트가 자기 데이터로 어댑터만 학습
② 어댑터 집계: 작은 어댑터 파라미터만 서버로 보내 연합 집계
③ 벤치마크·파이프라인: 다양한 연합 시나리오(이질 클라이언트·리소스 제약)와 평가를 표준화해 제공
④ 엔지니어링: 메모리·통신 최적화 등 현실 배포를 위한 실용 도구
```

핵심은 ①과 ②의 결합입니다. 각 기관은 자기 데이터로 LoRA 어댑터만 학습하고, 그 작은 어댑터만 서버로 보냅니다. 서버는 어댑터들을 집계해 공유 어댑터를 만듭니다. 거대한 본체 모델은 한 번 배포되면 고정이고, 매 라운드 오가는 것은 작은 어댑터뿐입니다. 여기에 더해 다양한 연합 시나리오를 재현할 수 있는 벤치마크와 파이프라인을 제공해, 후속 연구가 공정하게 비교할 수 있는 공통 기반을 마련했습니다.

## 임팩트: 왜 최신에서 중요한가

- **연합 + 파운데이션 모델의 대표 실용 프레임워크**입니다. 연합학습이 "작은 모델"을 넘어 LLM 시대로 들어섰음을 보여 줍니다.
- **PEFT × FL의 결합**을 표준화해, 모을 수 없는 도메인 데이터(의료·금융·법률)로 LLM을 특화하는 길을 실용화했습니다. 가장 가치 있지만 가장 못 모으는 데이터를 활용하는 길입니다.
- 재현 가능한 벤치마크로 후속 연구의 공통 기반을 제공했습니다.

## 한계와 주의점

- **여전히 무겁습니다.** PEFT로 통신은 줄어도, 클라이언트의 로컬 계산과 메모리는 LLM이라 부담이 큽니다. 본체 모델을 메모리에 올려야 하니까요.
- **이질성과 품질**입니다. non-IID, 클라이언트 리소스 격차, 데이터 품질 문제는 그대로 남습니다.
- **프라이버시는 별도 설계**입니다. 어댑터 교환에도 정보 누수가 가능해, 보안 집계·DP를 별도로 고려해야 합니다.
- **빠른 변화**입니다. LLM과 PEFT 생태계가 빠르게 바뀌어, 구체 기법은 계속 갱신됩니다(2026 기준).

## 더 읽을거리와 연결

- 이 책 내부: **FedAvg**[2](집계의 원형), **Secure Aggregation**(프라이버시). LLM 파인튜닝 맥락은 Part 1의 정렬(T04)·효율(T03 QLoRA) 테마와 이어집니다.
- 기초편 07·심화편 07(연합 + 파운데이션 모델)과 함께 보면 좋습니다.

## 한 줄 요약

> FederatedScope-LLM은 PEFT(작은 어댑터만 교환)로 거대 LLM의 통신 벽을 낮추고, 연합 LLM 파인튜닝의 데이터셋·벤치마크·도구를 종합한, 연합학습을 파운데이션 모델 시대로 이끈 대표 실용 프레임워크다.
