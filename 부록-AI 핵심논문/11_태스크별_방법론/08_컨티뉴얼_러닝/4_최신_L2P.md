# 고정 백본 + prompt로 망각 회피 — L2P

> **Learning to Prompt for Continual Learning (L2P)** — Wang et al., 2022, CVPR (arXiv:2112.08654)
> 원문: https://arxiv.org/abs/2112.08654 · 분류: 최신

EWC[2]·iCaRL·LwF는 모두 모델을 처음부터 순차 학습한다고 가정했습니다. 그런데 강력한 사전학습 백본(ViT[3] 등)이 흔해진 시대엔 다른 길이 있습니다. 백본을 안 건드리는 것입니다. 백본 가중치를 바꾸지 않으면 덮어쓰기로 인한 망각이 애초에 일어나지 않습니다. L2P는 "백본을 얼리고 prompt만 배우자"는 발상으로 컨티뉴얼 러닝의 패러다임을 바꿨습니다. rehearsal도 태스크 ID도 필요 없으면서 강력한, prompt 기반 CL의 출발점입니다.

## 배경: 망각의 근원과 태스크 ID 의존

기존 기법에는 두 가지 근원적 문제가 있습니다. 첫째, 순차 학습의 근원적 망각입니다. 백본 가중치를 계속 바꾸는 한, "가중치 덮어쓰기"라는 망각의 근원이 남습니다. EWC가 가중치를 묶고 iCaRL이 데이터를 재생하는 것은 모두 이 덮어쓰기를 완화하려는 노력입니다. 둘째, 태스크 ID 의존입니다. 많은 기존 기법이 추론 시 "지금 어떤 작업인지"를 알아야 했는데, 현실에서는 모를 때가 많습니다. 들어온 입력이 몇 번 작업의 것인지 알려 주지 않으니까요.

> 핵심 질문: 강한 사전학습 백본을 완전히 고정하고, 작업마다 아주 작은 것만 학습해 망각 자체를 회피할 수 없을까? 태스크 ID 없이도?

## 핵심 아이디어: 백본은 얼리고 prompt만 고른다

핵심은 이것입니다. 사전학습 백본은 동결하고, 학습 가능한 작은 prompt 벡터들의 풀(pool)만 둡니다. 입력마다 적절한 prompt를 골라 백본에 끼워 행동을 조절합니다.

- **prompt pool**: 여러 개의 작은 학습 가능한 prompt를 저장합니다.
- **key-query 선택**: 입력의 특징을 쿼리로, prompt들의 키와 매칭해 관련 prompt 몇 개를 선택해 백본 입력에 결합합니다.

백본을 안 바꾸니 작업 간 간섭(덮어쓰기)이 미미해 망각이 극적으로 줄어듭니다. 게다가 추론 시 prompt를 알아서 고르므로 태스크 ID가 필요 없습니다.

> 비유 — 베테랑 직원 + 업무 메모. 모든 걸 잘 아는 직원(고정 백본)을 재교육하지 않고, 업무마다 짧은 메모(prompt)만 건넵니다. 직원은 상황에 맞는 메모를 골라 읽고 처리하니, 옛 업무를 잊지 않습니다. 메모는 작아서 여러 개 보관해도 부담이 없고, 직원의 본래 실력(사전학습 지식)은 그대로 유지됩니다.

## 어떻게 작동하나

```
① 백본 동결: 사전학습된 모델 가중치를 고정
② prompt pool 학습: 작은 prompt 집합과 각 prompt의 키를 학습
③ 선택: 입력 특징(쿼리)과 키의 유사도로 상위 prompt 선택 → 입력 토큰에 결합해 백본 통과
④ rehearsal·태스크 ID 불필요: 옛 데이터 없이, 작업 구분 없이 작동
```

핵심은 key-query 매칭으로 prompt를 자동 선택한다는 점입니다. 각 prompt에는 키가 붙어 있고, 입력의 특징을 쿼리로 삼아 가장 잘 맞는 prompt를 고릅니다. 이 선택이 자동이라 어떤 작업의 입력인지 미리 알 필요가 없습니다. 학습되는 것은 작은 prompt 풀뿐이고 거대한 백본은 그대로이므로, 새 작업을 배워도 옛 작업의 prompt는 영향을 받지 않습니다.

## 임팩트: 왜 최신에서 중요한가

- **prompt 기반 CL의 출발점**입니다. 고정 백본 + 작은 prompt라는 새 범주를 열어, DualPrompt 등 후속 계열을 낳았습니다.
- 사전학습 시대에 rehearsal-free이면서 강력하고 태스크 ID 불필요라는, 기존 기법이 못 가진 조합을 달성했습니다.
- 컨티뉴얼 러닝 연구의 무게중심을 "처음부터 학습"에서 "사전학습 활용"으로 옮긴 대표작입니다. LLM·ViT 시대에 맞는 새 접근입니다.

## 한계와 주의점

- **사전학습 의존**입니다. 강한 백본이 전제입니다. 백본이 약하거나 도메인이 크게 다르면 효과가 줄어듭니다.
- **공정성 논쟁**이 있습니다. 성능 향상이 사전학습 백본 덕인지 CL 기법 덕인지 구분해 평가해야 한다는 지적이 있습니다. 강한 백본을 쓰면 어떤 방법이든 잘 나오기 때문입니다.
- **prompt 선택 안정성**입니다. 잘못된 prompt 선택이 성능을 떨어뜨릴 수 있습니다.
- **분포 격차**입니다. 사전학습 분포와 매우 다른 작업엔 prompt만으론 부족할 수 있습니다.

## 더 읽을거리와 연결

- 이 책 내부: **EWC·iCaRL·LwF**(처음부터 학습)와 대비. 후속으로 DualPrompt(Wang et al., 2022, ECCV). 사전학습 백본은 Part 1 시퀀스 아키텍처(ViT·트랜스포머)와 닿습니다.
- 기초편 08·심화편 08(사전학습 기반 CL)과 함께 보면 좋습니다.

## 한 줄 요약

> L2P는 강한 사전학습 백본을 동결하고 작업별로 작은 prompt만 골라 학습해, 백본을 안 건드려 망각을 회피하고 rehearsal·태스크 ID 없이 강력한 prompt 기반 CL의 출발점이다.
