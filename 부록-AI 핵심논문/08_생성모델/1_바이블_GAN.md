# 두 신경망의 경쟁으로 짓다 — GAN

> **Generative Adversarial Nets** — Goodfellow et al., 2014, NeurIPS (arXiv:1406.2661)
> 원문: https://arxiv.org/abs/1406.2661 · 분류: 바이블

새 이미지를 만든다는 것은 오랫동안 어려운 문제였습니다. 무엇이 그럴듯한지를 직접 수식으로 적기 어려웠기 때문입니다. 고양이 사진을 생성하라고 할 때 "고양이다움"을 픽셀 단위 손실로 정의하면, 모델은 모든 고양이의 평균을 그려 흐릿한 이미지를 내놓습니다. GAN은 발상을 뒤집습니다. 그럴듯함의 기준을 사람이 적는 대신, 두 신경망을 서로 경쟁시켜 그 기준 자체를 학습으로 빚어 내는 것입니다. 이 적대적 학습은 2010년대 생성 모델의 패러다임을 정의했고, 학습되는 비평가로 품질을 끌어올린다는 발상은 생성을 넘어 널리 영향을 줬습니다.

## 배경: 그럴듯함을 어떻게 가르치나

생성 모델의 핵심 난제는 "진짜 같은가"를 어떻게 학습 신호로 만드느냐입니다. 가장 단순한 접근은 생성한 이미지와 진짜 이미지의 픽셀 차이(예: 평균제곱오차 MSE)를 줄이는 것입니다. 그런데 이 방식에는 치명적 문제가 있습니다. 한 입력에 대해 여러 그럴듯한 출력이 가능할 때, MSE를 최소화하면 그 여러 답의 평균을 그립니다. 평균은 어느 것도 아닌 흐릿한 이미지입니다. 사람 얼굴을 생성할 때 코의 위치가 조금씩 다른 여러 정답이 있다면, MSE는 그것들을 뭉개 버립니다.

> 핵심 질문: 그럴듯함의 기준을 사람이 적지 말고, 학습되는 비평가에게 맡길 수 없을까?

## 핵심 아이디어: 위조범과 감별사의 군비 경쟁

GAN은 두 신경망을 적대적(adversarial)으로 경쟁시킵니다. 생성자(generator)는 가짜 이미지를 만들어 판별자를 속이려 하고, 판별자(discriminator)는 진짜와 가짜를 가려내려 합니다. 둘이 경쟁하며 함께 강해집니다.

> 비유 — 위조범과 감별사. 위조범(생성자)은 더 정교한 위조 지폐를 만들고, 감별사(판별자)는 더 예리한 감별법을 익힙니다. 이 군비 경쟁이 거듭될수록 위조 지폐는 진짜와 구별할 수 없을 만큼 정교해집니다. 핵심은 감별사가 고정된 규칙이 아니라 함께 발전하는 학습된 비평가라는 점입니다.

## 수식으로 들어가기

GAN의 학습은 하나의 가치 함수 `V(D, G)`를 두고 벌이는 minimax 게임으로 표현됩니다.

```
min_G max_D  V(D,G) = E_{x∼데이터}[ log D(x) ]  +  E_{z∼잡음}[ log(1 − D(G(z))) ]
```

각 항을 풀어 봅시다. `D(x)`는 입력 `x`가 진짜일 확률입니다. 첫 항은 판별자가 진짜 이미지 `x`에 높은 확률을 주도록(`log D(x)` 최대화), 둘째 항은 가짜 이미지 `G(z)`에 낮은 확률을 주도록(`D(G(z))`를 0에 가깝게) 만듭니다. 판별자는 이 `V`를 최대화하려 합니다. 반대로 생성자는 둘째 항을 최소화하려 하는데, 이는 판별자가 자기 가짜를 진짜로 착각하게(`D(G(z))`를 1에 가깝게) 만드는 것입니다. 두 목표가 정확히 반대라 minimax입니다.

이 게임의 아름다운 점은 이론적 균형입니다. 판별자가 최적일 때 생성자가 푸는 문제는 생성 분포와 진짜 데이터 분포 사이의 옌센-섀넌 발산(Jensen-Shannon divergence)을 최소화하는 것과 같음을 논문이 보였습니다. 즉 이상적 균형에서 생성 분포는 진짜 데이터 분포에 수렴합니다. 실전에서는 학습 초기에 `log(1 − D(G(z)))`의 기울기가 약해, 생성자가 `log D(G(z))`를 최대화하는 형태(non-saturating loss)로 바꿔 학습합니다.

```
무작위 잡음 z → [생성자 G] → 가짜 이미지
진짜 / 가짜 이미지 → [판별자 D] → "진짜일 확률"
D: 진짜는 1, 가짜는 0으로 맞히도록    G: D가 1로 착각하도록  (목표가 정반대)
```

| 구성 | 목표 | 비유 |
|---|---|---|
| 생성자 G | 판별자를 속임 | 위조범 |
| 판별자 D | 진짜·가짜 구별 | 감별사 |
| 균형 | 생성 분포 → 데이터 분포 | 위조가 진짜와 구별 불가 |

## 임팩트: 왜 바이블이 되었나

- **2010년대 생성 모델 패러다임을 정의**했습니다. 사실적인 얼굴과 이미지 생성의 물결(StyleGAN 등)이 모두 이 적대 학습의 후손입니다.
- "학습되는 비평가로 품질을 끌어올린다"는 발상은 생성을 넘어 널리 영향을 줬습니다. T04 정렬의 보상 모델도 사상적으로 통합니다. 무엇이 좋은지를 직접 정의하는 대신 학습된 평가자에게 맡긴다는 점에서 같은 계열입니다.
- 오랜 영향력으로 시간이 검증한(Test-of-Time) 고전으로 꼽힙니다.

## 한계와 주의점

- **학습이 불안정**합니다. 두 네트워크의 균형을 잡기가 까다롭고, 한쪽이 너무 강해지면 학습이 발산합니다. 판별자가 너무 완벽하면 생성자에게 줄 기울기가 사라집니다.
- **모드 붕괴(mode collapse)**: 생성자가 몇 가지 그럴듯한 출력만 반복 생산해 다양성을 잃을 수 있습니다. 판별자를 속이는 안전한 몇 장만 만들고 마는 것입니다.
- 이런 불안정성 때문에, 이후 더 안정적인 확산 모델(다음 글 DDPM[2])이 주류 자리를 이어받습니다.
