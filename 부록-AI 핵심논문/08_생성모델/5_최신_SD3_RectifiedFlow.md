# 직선 경로 샘플링 — Stable Diffusion 3 / Rectified Flow

> **Scaling Rectified Flow Transformers for High-Resolution Image Synthesis** — Esser et al., 2024, ICML (arXiv:2403.03206)
> 원문: https://arxiv.org/abs/2403.03206 · 분류: 최신

DDPM[2]식 확산은 잡음에서 이미지로 갈 때 구불구불한 여러 단계를 거칩니다. 그래서 느립니다. SD3는 두 가지로 이를 개선합니다. rectified flow로 그 경로를 더 곧게 펴서 적은 단계로 가게 하고, MM-DiT로 텍스트와 이미지를 대등하게 섞어 텍스트 정합성을 높였습니다. flow matching이라는 정식화를 최신 텍스트-이미지 SOTA의 주류로 끌어올린 논문입니다.

## 배경: 곡선 경로의 비용과 텍스트 정합성

확산을 이해하는 한 가지 관점은, 잡음 분포에서 데이터 분포로 가는 경로를 따라가는 것입니다. 문제는 DDPM이 학습하는 이 경로가 구불구불한 곡선이라는 점입니다. 곡선을 정확히 따라가려면 작은 보폭으로 여러 번 움직여야 합니다. 큰 보폭으로 가면 곡선을 벗어나 품질이 떨어집니다. 이것이 확산이 많은 샘플링 단계를 필요로 하는 근본 이유입니다. 또한 텍스트-이미지 생성에서는 텍스트 정보와 이미지 정보를 어떻게 잘 섞느냐가 정합성(프롬프트를 충실히 반영하는가)을 좌우합니다.

> 핵심 질문: 잡음에서 이미지로 가는 경로를 더 곧게 펴서 적은 단계로 가고, 텍스트와 이미지를 더 잘 섞을 수 없을까?

## 핵심 아이디어: 길을 곧게 펴고, 두 모달리티를 대등하게

SD3는 두 가지를 결합합니다.

- **Rectified Flow(직선화 흐름)**: 잡음에서 이미지로 가는 경로를 가능한 한 직선으로 학습합니다. 길이 곧으면 큰 보폭으로 움직여도 경로를 벗어나지 않아, 더 적은 단계로 정확히 도달합니다. 이는 Flow Matching[4] 계열의 정식화를 텍스트-이미지에 맞게 개선한 것입니다.
- **MM-DiT(멀티모달 DiT)**: DiT[3]를 확장해, 텍스트 토큰과 이미지 토큰을 대등하게 어텐션으로 섞는 백본입니다. 한쪽이 다른 쪽에 종속되는 것이 아니라 서로 영향을 주고받습니다.

> 비유 — 목적지까지 구불구불한 산길 대신 곧게 뚫은 도로를 내는 것. 길이 곧으면 같은 거리를 더 적은 정거장으로 갑니다. 거기에 텍스트와 이미지가 대등하게 대화하는 통역 구조(MM-DiT)를 더한 셈입니다.

## 어떻게 작동하나

```
학습: 잡음 ──(가능한 한 직선 경로)── 이미지    ← rectified flow
      직선 경로를 따라가는 속도장(velocity)을 학습
백본: MM-DiT — 텍스트 토큰 ↔ 이미지 토큰을 어텐션으로 대등하게 결합
효과: 더 적은 샘플링 단계 + 향상된 텍스트 정합성·품질
```

경로가 곧을수록 샘플링 효율이 좋아지고, MM-DiT가 텍스트 지시를 이미지에 더 충실히 반영합니다. DiT의 확장성 위에 효율적 샘플링을 얹은 것입니다. 확산이 잡음을 점진적으로 제거하는 관점이었다면, flow는 잡음에서 데이터로 흐르는 속도장을 직접 학습하는 관점입니다. 둘은 깊이 연결되어 있지만, 직선화된 flow가 더 적은 단계로 갈 수 있다는 실용적 이점을 줍니다.

| 항목 | DDPM식 확산 | Rectified Flow(SD3) |
|---|---|---|
| 경로 | 곡선 | 직선에 가깝게 |
| 샘플링 단계 | 많음 | 적음 |
| 백본 | U-Net/DiT | MM-DiT(텍스트·이미지 대등) |

## 임팩트: 무엇을 바꿨나

- **rectified flow / flow matching**을 최신 텍스트-이미지 SOTA의 주류 정식화로 끌어올렸습니다. "확산이냐 흐름이냐"의 논의에서 흐름 기반이 실용적 중심에 서게 했습니다.
- DiT 백본(트랜스포머 확산)과 결합해, 고품질·고해상도 생성의 표준 레시피가 되었습니다(2026-06 기준).
- 텍스트와 이미지를 대등하게 섞는 MM-DiT 설계는 멀티모달 생성의 방향을 제시했습니다. 텍스트가 이미지를 일방적으로 조건 짓는 것을 넘어 양방향 상호작용으로 나아간 것입니다.

## 한계와 주의점

- 여전히 대규모 학습 자원이 필요합니다. 최신 SOTA 모델 학습은 막대한 비용이 듭니다.
- "직선화"가 품질과 효율을 늘 보장하는 만능은 아닙니다. 데이터와 설정에 따라 이점이 달라집니다.
- 빠르게 발전하는 영역이라(2026-06 기준), 이후 더 효율적인 정식화가 계속 나오고 있습니다. 특정 방법의 우위를 단정하기보다 흐름으로 읽는 편이 좋습니다.
