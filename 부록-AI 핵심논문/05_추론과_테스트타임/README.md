# T05 · 추론·테스트타임 — 어떻게 더 잘 생각하나

모델을 더 크게 만들고 더 많이 학습시키는 것 말고 다른 길이 있을까요. 답할 때 더 오래, 더 신중히 생각하게 하면 어려운 문제를 풀 수 있지 않을까요. 이 테스트타임 컴퓨트(test-time compute)라는 발상이 추론 모델의 시대를 열었습니다. 이 테마는 추론을 프롬프트로 끌어내는 데서 시작해, 그것을 학습으로 내재화하고, 마침내 정답 보상만으로 추론이 저절로 창발하는 데까지 이어지는 다섯 편의 논문을 따라갑니다.

## 바이블에서 최신까지, 한눈에

| 분류 | 논문 | 핵심 한 줄 |
|---|---|---|
| 바이블 | Chain-of-Thought (2022) | 중간 추론 단계를 적게 하면 추론력 향상 |
| 바이블 | STaR (2022) | 스스로 만든 성공 풀이로 추론을 부트스트랩 |
| 최신 | PRM / Let's Verify Step by Step (2023) | 과정에 보상을 주면 결과 보상보다 낫다 |
| 최신 | OpenAI o1 (2024) | 대규모 강화학습으로 더 오래 생각 (블로그/리포트) |
| 최신 | DeepSeek-R1 (2025) | 순수 강화학습만으로 추론이 창발, Nature |

## 이 테마를 관통하는 한 문장

추론을 끌어내고(CoT), 스스로 길러 내고(STaR), 과정을 채점하고(PRM), 마침내 학습으로 긴 추론을 내재화합니다(o1·R1). "생각을 잘 시키는 법"에서 "생각을 잘하도록 배우는 법"으로 가는 이야기입니다. 이 흐름의 끝에서 R1은 사람이 추론을 설계해 넣지 않아도, 정답에 보상하는 것만으로 모델이 스스로 깊이 생각하게 됨을 보여 줍니다.
