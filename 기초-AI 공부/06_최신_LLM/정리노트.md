## 한 줄 요약 노트

| 주제 | 목적 (어떤 문제를 푸는가) | 핵심 아이디어 (한 문장) |
|---|---|---|
| **정렬 (RLHF·DPO)** | 똑똑함을 '쓸모·안전'으로 | RLHF는 선호로 보상 모델을 만들어 강화학습, DPO는 보상 모델 없이 선호를 직접 최적화(단순·안정) |
| **추론 모델** | 어려운 문제를 더 잘 풀려고 | '답할 때 더 오래 생각하기'(테스트타임 연산)+생각의 사슬로 단계를 밟는다 |
| **효율화 (MoE)** | 큰 모델을 싸게 굴리려고 | 총 파라미터는 크게·활성은 작게 — 게이트가 전문가 일부만 켠다, 양자화·긴 문맥 |
| **RAG** | 환각을 줄이려고 | 답하기 전 외부 자료를 임베딩 유사도로 검색해 근거로 삼는 오픈북 방식 |
| **에이전트·멀티모달** | 말만 하는 한계를 넘으려고 | 도구로 [생각→행동→관찰] 루프를 돌고, 모든 형태를 같은 벡터 공간에 모은다 |
| **평가·안전·한계** | 속지 않으려고 | 벤치마크 점수를 의심하고, 환각·편향·정렬이라는 세 근본 한계를 기억한다 |

> 이 표의 오른쪽을 **가리고** 왼쪽만 보며 말해 보세요. 그게 진짜 점검입니다.

## 이 장의 핵심 수식 (모두 개념 수준)

이 장은 수식보다 *아이디어*가 핵심입니다. 그래도 두 가지는 말로 풀 수 있어야 합니다.

1. **RLHF 목적함수의 직관:** `보상(사람이 좋아할 점수)을 최대화하되, 원본 모델에서 멀어지면(KL) 벌점`
   → "사람 취향에 맞추되, 원래 능력을 잃을 만큼 변하지는 마라."
2. **코사인 유사도(RAG·임베딩):** `cos_sim(a, b) = (a · b) / (|a| · |b|)`
   → 두 벡터의 *각도*만 본다. 같은 방향이면 1(의미 비슷), 직각이면 0(무관). 4단계 "내적=유사도"를 0~1로 정규화한 것.

> 보조 개념: **DPO**=선호를 손실로 바꿔 직접 학습 · **MoE 게이팅**=입력마다 전문가 일부만 켜는 라우터 · **희소 활성화**=총≫활성.

## 이해 점검 (커리큘럼 기준)

말로 막힘없이 답할 수 있으면 체크하세요.

- [ ] **RLHF와 DPO가 '무엇을 위한' 단계이고 어떻게 다른지 설명할 수 있다.**
  - 힌트: 둘 다 정렬 / RLHF는 보상 모델+강화학습, DPO는 보상 모델 없이 직접.
- [ ] **추론 모델이 일반 LLM과 어떻게 다른지(테스트타임 연산) 설명할 수 있다.**
  - 힌트: 모델을 키우는 대신 '답할 때 더 오래 생각', 생각의 사슬.
- [ ] **MoE가 왜 효율적인지 한 문장으로 설명할 수 있다.**
  - 힌트: 총 파라미터는 크게, 활성 파라미터는 작게.
- [ ] **RAG가 환각을 줄이는 원리를 설명할 수 있다.**
  - 힌트: 오픈북 — 외부 자료를 의미 검색해 근거로 답한다.
- [ ] **현재 LLM의 대표적 한계 3가지를 말할 수 있다.**
  - 힌트: 환각 · 편향 · 정렬 문제.

## 이 책 이후 — 심화 방향

축하합니다. 0단계의 지도 그리기에서 출발해 **퍼셉트론 → 통계학습 → 딥러닝 → 트랜스포머 → LLM → 최신 LLM**까지, AI가 걸어온 길을 *원리로* 다시 걸었습니다.
이제 한 가지를 기억하세요 — 이 로드맵의 목적은 *진도 완주*가 아니라 **"원리를 말로 설명할 수 있게 되는 것"**이었습니다. 각 단계의 이해 점검에 막힘없이 답할 수 있다면, 당신은 이미 그 개념을 *아는* 것입니다.

다음으로 나아갈 세 갈래 길:

- **이론 심화** — PRML(Bishop), Probabilistic ML(Murphy) 같은 정통 교재로 수학적 깊이를 더하고, 관심 분야의 논문을 정독해 보세요.
- **응용 이해** — RAG·에이전트·평가 같은 *실제 시스템이 왜 그렇게 설계되는지*를 사례 중심으로 파 보세요.
- **구현으로 확장** — 개념이 잡혔다면 Karpathy의 nanoGPT 등으로 '손으로도' 확인하면 이해가 한층 단단해집니다.

> 수학 앞에서 멈추지 말고, *의미부터* 천천히 — 그러나 꾸준히 — 나아가세요. 유행어는 바뀌어도 **원리는 남습니다.**
