# 1. AI · 머신러닝 · 딥러닝 · LLM — 네 용어의 지도

뉴스에서 "AI", "머신러닝", "딥러닝", "LLM(거대 언어모델)"이 거의 같은 뜻처럼 섞여 쓰입니다.
하지만 이 넷은 **크기가 다른 동심원**입니다. 이 관계를 정확히 잡는 것이 모든 공부의 출발점입니다.

---

## 목적 — 왜 이 구분이 먼저인가

길을 잃지 않기 위해서입니다.
"딥러닝을 공부한다"고 할 때, 그것이 더 큰 '머신러닝'이라는 땅의 한 지역이고,
'머신러닝'은 다시 '인공지능'이라는 대륙의 한 나라임을 알면,
새 개념을 만날 때마다 **"이건 지도의 어디쯤인가?"**를 스스로 자리매김할 수 있습니다.

---

## 핵심 개념 — 포함 관계

네 용어는 다음처럼 **점점 좁아지는 포함 관계**를 이룹니다.

```
┌─────────────────────────────────────────────┐
│  인공지능 (AI)                                │
│  "사람처럼 지적인 일을 하는 기계"             │
│  ┌───────────────────────────────────────┐  │
│  │  머신러닝 (ML)                          │  │
│  │  "데이터로부터 스스로 규칙을 배우는 AI" │  │
│  │  ┌─────────────────────────────────┐  │  │
│  │  │  딥러닝 (DL)                      │  │  │
│  │  │  "여러 층의 신경망을 쓰는 ML"     │  │  │
│  │  │  ┌───────────────────────────┐  │  │  │
│  │  │  │  LLM (거대 언어모델)       │  │  │  │
│  │  │  │  "언어를 다루는 거대 DL"   │  │  │  │
│  │  │  └───────────────────────────┘  │  │  │
│  │  └─────────────────────────────────┘  │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘

      AI ⊃ 머신러닝 ⊃ 딥러닝 ⊃ LLM
```

각 원을 한 줄로 정의하면 이렇습니다.

- **인공지능(AI, Artificial Intelligence)** — 가장 넓은 개념. *지적으로 보이는 일*을 하는 기계 전부. 꼭 학습하지 않아도 됩니다. (예: 정해진 규칙으로 길을 찾는 내비게이션 알고리즘, 체스 규칙을 손으로 짠 프로그램도 AI입니다.)
- **머신러닝(ML, Machine Learning)** — AI의 한 갈래. 사람이 규칙을 직접 짜는 대신, **데이터에서 규칙을 스스로 찾아내는** 방식. (예: 수천 장의 사진으로 고양이를 구별하는 법을 배우는 것.)
- **딥러닝(DL, Deep Learning)** — 머신러닝의 한 갈래. **여러 층(deep)으로 쌓은 인공 신경망**을 쓰는 방법. 이미지·음성·언어처럼 복잡한 데이터에서 특히 강합니다.
- **LLM(Large Language Model, 거대 언어모델)** — 딥러닝의 한 응용. **방대한 텍스트로 학습한 아주 큰 신경망**으로, 언어를 이해하고 생성합니다. ChatGPT, Claude, Gemini가 여기에 속합니다.

---

## 직관 — 러시아 인형(마트료시카)

이 관계는 **인형 속의 인형**과 같습니다.
가장 바깥의 큰 인형이 AI, 그 안에 ML, 또 그 안에 DL, 가장 안쪽 작은 인형이 LLM.

- 안쪽일수록 **더 구체적이고 특수한** 기술입니다.
- 바깥쪽일수록 **더 넓고 일반적인** 개념입니다.
- **모든 LLM은 딥러닝이고, 모든 딥러닝은 머신러닝이고, 모든 머신러닝은 AI**입니다.
- 하지만 그 반대는 성립하지 않습니다. **모든 AI가 머신러닝은 아닙니다.**

---

## 방법론 — 이 관계를 어떻게 써먹는가

새로운 기술이나 모델 이름을 만나면, 스스로에게 두 가지를 물으세요.

1. **"이건 지도의 어느 원에 속하는가?"**
   - 예: "트랜스포머"는? → 딥러닝(DL)의 한 구조. 그 위에서 LLM이 만들어집니다.
2. **"왜 더 안쪽(특수한) 기술이 필요했는가?"**
   - 바깥 원의 일반적 방법으로 안 풀리는 문제가 있었기 때문에, 더 안쪽의 특수한 기술이 태어났습니다.
   - 이 "왜"가 곧 이 커리큘럼 전체의 줄거리입니다.

---

## 흔한 오해

- **"AI = 딥러닝"이 아닙니다.** 규칙만으로 짠 프로그램, 통계 기반 모델도 모두 AI입니다. 딥러닝은 AI의 *일부*입니다.
- **"머신러닝 = 딥러닝"이 아닙니다.** 결정트리, SVM, 선형회귀처럼 신경망을 쓰지 않는 머신러닝도 아주 많습니다. (2단계에서 다룹니다.)
- **"LLM = AI 전체"가 아닙니다.** LLM은 *언어*에 특화된 딥러닝 응용일 뿐, AI의 가장 안쪽 작은 원입니다. 최근 가장 화제일 뿐, 가장 크지는 않습니다.

---

## 한 줄 요약

> **AI ⊃ 머신러닝 ⊃ 딥러닝 ⊃ LLM.**
> 바깥일수록 넓고 일반적, 안쪽일수록 좁고 특수하다. 새 개념을 만나면 "이건 어느 원인가"부터 묻자.
