# 03 · 차원축소와 표현학습 — 많은 숫자를 적은 숫자로

> **이 태스크의 목적**
> 데이터를 설명하는 숫자(차원)가 *너무 많을 때* 생기는 문제와, 그걸 *핵심만 남겨 줄이는* 방법을 잡는다.
> "왜 줄여야 하는가 → 가장 기본적인 줄이는 법(PCA) → 줄여서 눈으로 보기(시각화)"의 흐름을 직관으로 이해한다.

> ← **연결** — [기초편 1단계 · 선형대수](../../01_머신러닝_수학/1_선형대수.md)에서 *벡터·축·분산*의 감을 잡고 오면 좋습니다. 차원축소는 결국 **벡터를 더 좋은 축으로 다시 보는 일**입니다.

---

## 이 태스크에서 답하는 질문

이 폴더를 다 읽고 나면, 다음을 **말로** 답할 수 있어야 합니다.

- 차원(특징의 개수)이 많으면 *무엇이* 나빠지는가? ('차원의 저주')
- 데이터를 적은 숫자로 줄이면서도 *핵심*은 어떻게 지키는가? (PCA — 분산이 큰 축)
- 100차원짜리 데이터를 사람이 *눈으로 보려면* 어떻게 하는가? (2D 시각화 임베딩)
- "줄인다"와 "표현을 배운다"는 어떻게 이어지는가?

> 한마디로 이 폴더의 핵심 질문 — **"숫자가 너무 많을 때, 무엇을 버리고 무엇을 남겨야 데이터를 더 잘 볼 수 있을까?"**

## 이 태스크에 필요한 수학

거의 없습니다. **벡터(숫자 묶음)** 와 **분산(퍼진 정도)**, **거리** 정도의 감만 있으면 충분합니다.
수식은 모두 *의미 위주*로만 다룹니다. 유도는 하지 않습니다.

## 이 폴더를 통과하는 기준

아래를 **말로 설명할 수 있으면** 통과입니다.

- [ ] 차원이 많아질 때 생기는 문제(데이터 희박·거리의 무의미함)를 예로 말할 수 있다.
- [ ] PCA가 *왜 '분산이 큰 축'*을 고르는지 한 문장으로 설명할 수 있다.
- [ ] 고차원 데이터를 2D로 줄여 보는 것이 *왜 유용한지*, 그리고 *무엇을 조심해야 하는지* 말할 수 있다.
