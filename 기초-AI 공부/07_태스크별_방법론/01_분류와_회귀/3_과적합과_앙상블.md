# 3. 과적합과 앙상블 — 외우는 병과 그 처방

결정트리 하나는 강력하지만 *위험한* 습관이 있습니다 — 데이터를 **통째로 외워** 버립니다. 이 글은 그 병(과적합)이 무엇인지 직관과 수식으로 잡고, 가장 널리 쓰이는 처방 두 가지 — **배깅**과 **부스팅** — 을 해부합니다. 이 두 전략은 단순한 트릭이 아니라, 현대 정형데이터 머신러닝(랜덤포레스트·XGBoost·LightGBM)을 떠받치는 기둥입니다. 캐글 대회 상위권의 단골 우승 모델이 여전히 부스팅 트리라는 점만 봐도, 여기서 깊이 파 둘 가치가 충분합니다.

---

## 목적 — "외운 학생"의 함정

깊게 자란 결정트리는 학습 데이터의 *잡음 한 점 한 점까지* 맞춰 버립니다.

> 학습 데이터에서는 점수가 거의 만점인데, **새 데이터에서 뚝 떨어지는** 현상 — 이것이 **과적합(overfitting)**입니다.

비유하면, *기출문제의 답 번호까지 통째로 외운 학생*입니다. 본 문제는 다 맞히지만, 숫자만 살짝 바꾼 새 문제에서 무너집니다. 우리가 원하는 건 *외우기*가 아니라 *일반화*(처음 보는 데이터에도 통하는 것)입니다. (→ [2단계 · 2_일반화와_과적합](../../02_머신러닝_개념과_방법론/2_일반화와_과적합.md))

왜 트리는 특히 잘 외울까요? 트리는 데이터를 계속 잘라 내려가다 보면, *극단적으로는 점 하나마다 잎 하나*를 만들 수도 있습니다. 그러면 학습 데이터는 100% 맞히지만, 그건 패턴을 배운 게 아니라 데이터를 *복사*한 것입니다. 깊이 제한·잎 최소 샘플 수 같은 제약(가지치기, pruning)으로 막을 수 있지만, 근본 처방은 따로 있습니다 — 바로 앙상블입니다.

---

## 핵심 개념 — 과적합과 편향·분산

- **과적합(overfitting)**: 모델이 너무 복잡해 *데이터의 잡음까지* 맞춤. 학습 성능↑, 시험 성능↓.
- **과소적합(underfitting)**: 모델이 너무 단순해 *진짜 패턴도* 못 잡음. 학습·시험 둘 다↓.
- 둘 사이의 *적당한 복잡도*를 찾는 게 핵심입니다. 이를 **편향–분산 트레이드오프(bias-variance tradeoff)**로 봅니다.

이 트레이드오프는 수식으로도 깔끔하게 떨어집니다. 회귀에서 새 점에 대한 기대 시험오차를 분해하면 이렇게 됩니다 [5].

`기대오차 = 편향² + 분산 + 잡음(줄일 수 없음)`

- **편향(bias)**: 모델이 너무 단순해서 생기는 *구조적 빗나감*입니다. 직선으로 곡선을 근사하면 아무리 데이터를 늘려도 못 맞히는 부분이 남죠 — 그게 편향입니다(과소적합 쪽).
- **분산(variance)**: 학습 데이터가 조금만 바뀌어도 결과가 *크게 출렁이는* 불안정함입니다. 데이터를 다시 뽑을 때마다 모델이 완전히 달라지면 분산이 큽니다(과적합 쪽).
- **잡음**: 데이터 자체에 낀 무작위성. 어떤 모델로도 줄일 수 없는 하한선입니다.

```
오차 │                          ╱ 시험오차(전체)
     │ ＼편향²              ╱
     │   ＼＿＿＿＿＿＿＿╱        ← 최적 복잡도
     │        ＼＿＿＿╱  분산↗
     │  학습오차 ＼＿＿＿＿＿＿
     └────────────────────────── 모델 복잡도 →
      (단순)                    (복잡)
   왼쪽=과소적합(편향↑)  오른쪽=과적합(분산↑)
```

> 트리 하나는 보통 **편향은 낮지만 분산이 큰** 모델입니다 — 데이터를 살짝만 바꿔도 가지가 완전히 달라집니다. 그래서 처방의 방향이 정해집니다: *분산을 잡아야 한다.* 앙상블, 특히 배깅이 바로 이 일을 합니다.

---

## 직관 — 한 전문가보다 위원회

여러 모델의 예측을 **모아** 더 강한 하나를 만드는 것이 **앙상블(ensemble)**입니다.

> 비유 — **한 명의 전문가 vs 다양한 전문가의 위원회.** 한 전문가는 특정 편향이 있을 수 있지만, *서로 다른 실수를 하는* 여러 전문가의 투표를 모으면 실수가 **상쇄**됩니다. 한 사람이 헛다리를 짚어도 나머지가 바로잡아 주니까요.

핵심 조건은 **"서로 달라야 한다"**입니다. 똑같은 전문가 100명은 한 명과 다를 게 없습니다. *다른 실수를 하는*(상관이 낮은) 모델을 모아야 효과가 납니다. 그래서 앙상블은 *어떻게 서로 다른 모델을 만드느냐*가 관건이고, 그 방법이 크게 두 갈래입니다 — 데이터를 섞어 *따로* 키우거나(배깅), *이어달리며* 앞 실수를 보완하거나(부스팅).

---

## 방법론 — 배깅과 부스팅

### ① 배깅 (bagging) — "따로따로, 그리고 평균"

**서로 다른 데이터 조각으로 여러 모델을 *독립적으로* 키운 뒤, 투표/평균**합니다. 이름은 *bootstrap aggregating*의 줄임말입니다 [1].

```
원본 데이터 ──┬─ 부트스트랩 표본 1 → 트리 1 ┐
              ├─ 부트스트랩 표본 2 → 트리 2 ├─ 투표/평균 → 최종 답
              └─ 부트스트랩 표본 3 → 트리 3 ┘
```

- **부트스트랩(bootstrap)**: 원본에서 *복원추출*로(같은 점이 여러 번 뽑힐 수 있게) 표본을 다시 뽑는 것. 매번 조금씩 다른 데이터가 되니, 트리들이 서로 다르게 자랍니다.
- 각 트리는 데이터의 *다른 조합*을 보고 자라, 서로 조금씩 다른 실수를 합니다.
- 대표 = **랜덤포레스트(Random Forest)**. 여기에 *분할마다 특성도 무작위로* 일부만 후보로 골라(예: 전체 특성의 √p개) 트리들을 더 다양하게 만듭니다. 이 특성 무작위화가 트리 간 상관을 더 떨어뜨려 분산 감소 효과를 키웁니다 [2].
- 효과: 트리들의 *분산을 평균이 깎아 내려* 과적합을 완화. 편향은 거의 그대로 두고 분산만 줄이는 게 핵심입니다. 트리들을 서로 독립적으로 키우므로 *병렬 처리*가 쉬워 빠릅니다.

### ② 부스팅 (boosting) — "이어달리기, 앞의 실수를 보완"

**모델을 *순서대로* 키우되, 각 모델이 *앞 모델이 틀린 부분*을 집중 보완**합니다.

```
트리 1 (대충 예측) → 틀린 부분 확인
   → 트리 2 (그 틀림을 집중 보완) → 또 틀린 부분 확인
      → 트리 3 (다시 보완) → … → 전부 가중 합산 → 최종 답
```

- 부스팅에는 두 계보가 있습니다.
  - **AdaBoost**: 앞 모델이 *틀린 샘플의 가중치를 키워*, 다음 모델이 그 어려운 샘플에 집중하게 합니다 [3].
  - **그래디언트 부스팅(GBM)**: 앞 모델이 *못 맞힌 양(잔차)*을 다음 모델의 목표로 삼아, 손실 함수의 경사를 따라 한 걸음씩 내려갑니다 [4]. XGBoost·LightGBM·CatBoost가 모두 이 계보입니다(→ [심화편](../../../심화-AI%20공부/11_태스크별_방법론/01_분류와_회귀/)).
- 효과: 약한 모델(얕은 트리)을 모아 *주로 편향을 줄여* 정확도를 끌어올립니다. 대신 순서대로 앞 실수를 파고들다 보니 *과하면 잡음까지 학습해 다시 과적합*될 수 있어, 트리 개수·학습률로 멈출 줄 알아야 합니다.

```
                배깅                    부스팅
구조        병렬(독립적)            순차(이어달리기)
다양성 원천  데이터·특성 무작위        앞 모델의 실수
주 효과      분산 ↓                  편향 ↓
베이스 모델  깊은 트리(저편향·고분산)  얕은 트리(고편향·저분산)
과적합 위험  낮음(평균이 보호)         있음(과하면 잡음 학습)
속도        병렬화 쉬움              순차라 상대적으로 느림
```

> 한 줄 대비: **배깅은 "여럿이 따로 풀고 평균"(분산↓), 부스팅은 "이어달리며 앞 실수 보완"(편향↓).** 정반대 전략으로 트레이드오프의 양쪽 끝을 공략합니다.

## 수학 — 평균이 흔들림을 줄인다

배깅의 핵심은 *평균의 마법*입니다. `B`개의 모델을 평균낼 때, 각 모델의 분산이 `σ²`이고 서로 상관이 `ρ`라면 평균의 분산은 이렇게 됩니다 [5].

`평균의 분산 = ρσ² + (1−ρ)/B · σ²`

> 의미: `B`(모델 수)를 키우면 둘째 항은 0으로 사라집니다. 남는 건 첫째 항 `ρσ²` — **모델 간 상관 `ρ`가 낮을수록 분산이 더 많이 줄어듭니다.** 랜덤포레스트가 *특성까지 무작위로* 고르는 이유가 바로 이것입니다. 트리 간 상관 `ρ`를 떨어뜨려야 평균이 분산을 더 깎아 주니까요. 동전을 한 번 던지면 0/1로 들쭉날쭉하지만 100번 평균내면 0.5 근처로 안정되는 것과 같은 원리입니다.

부스팅은 반대로 **잔차(residual)**를 따라갑니다. 그래디언트 부스팅을 회귀로 보면:

`다음 모델의 목표 ≈ (실제값 − 지금까지의 예측) = 아직 못 맞힌 잔차`
`새 예측 = 지금까지의 예측 + η · (다음 모델의 예측)`

> 의미: "아직 못 맞힌 만큼"을 다음 모델이 맞히도록 시킵니다. 여기서 `η`(학습률, learning rate)는 *한 걸음의 보폭*입니다. 작게 잡으면(예: 0.1) 천천히 신중하게 내려가 과적합이 덜하지만 트리가 더 많이 필요하고, 크게 잡으면 빨리 가지만 헛디딜 위험이 큽니다. 이 잔차 따라가기를 반복하면 못 맞힌 양이 점점 줄어 예측이 정답에 다가갑니다. (자세한 원리는 심화편 1_부스팅의_원리)

## 흔한 오해

- **"앙상블은 항상 더 정확하다"가 아닙니다.** 모델들이 *서로 비슷하게* 틀리면(상관이 높으면) 모아도 효과가 적습니다. 위 분산 공식의 `ρσ²` 항이 그대로 남기 때문입니다. *다양성*이 핵심입니다.
- **"부스팅은 과적합이 없다"가 아닙니다.** 앞 실수를 계속 파고들다 보면 *잡음까지* 보완하려 들어 다시 과적합됩니다. 트리 개수·깊이·학습률·조기종료로 멈출 줄 알아야 합니다.
- **"딥러닝 시대라 트리 앙상블은 한물갔다"가 아닙니다.** 표(table) 형태 정형데이터에서는 여전히 부스팅 트리가 *사실상 표준급* 강자입니다. 여러 벤치마크에서 표 데이터에 한해 부스팅 트리가 딥러닝을 앞선다는 보고가 꾸준합니다 — 이 논쟁이 심화편·논문편의 큰 주제입니다.
- **"배깅과 부스팅은 비슷한 것"이 아닙니다.** 배깅은 *독립·병렬·분산↓*, 부스팅은 *순차·보완·편향↓* — 정반대 전략입니다. 베이스 모델도 배깅은 깊은 트리(고분산), 부스팅은 얕은 트리(고편향)로 정반대를 씁니다.

---

## 한 줄 요약

> **트리 하나는 데이터를 외워 과적합(분산↑)되기 쉽다. 여럿을 모으는 앙상블이 처방이다.**
> 배깅(랜덤포레스트)은 *부트스트랩으로 따로 키워 평균*해 분산을 줄이고(상관이 낮을수록 효과↑), 부스팅(GBM·XGBoost…)은 *잔차를 따라 이어달리며 앞 실수를 보완*해 편향을 줄인다.
