# 4. 트랜스포머의 전체 구조 — 어텐션을 둘러싼 부품들

앞 글에서 심장인 셀프 어텐션을 해부했습니다. 하지만 어텐션 하나만으로는 트랜스포머가 되지 않습니다. 어텐션은 강력하지만 *혼자서는 풀 수 없는 약점*들이 있고, 그 빈틈을 메우려고 **네 개의 보조 부품**이 둘레에 붙습니다. 각 부품이 *어떤 문제를 푸는지*만 잡으면 구조 전체가 한눈에 들어옵니다.

이 글의 읽기 전략은 "암기"가 아니라 **"결핍 → 처방"**입니다. 각 부품을 "어텐션의 어떤 결핍을 채우는 처방인가"로 읽으면, 복잡해 보이는 구조도 필연으로 보입니다.

---

## 목적 — 어텐션의 빈틈을 메우기

셀프 어텐션은 강력하지만 혼자서는 세 가지 결핍이 있습니다.

| 어텐션의 결핍 | 무엇이 문제인가 | 처방 |
|---|---|---|
| **순서를 모른다** | "철수가 영희를 좋아한다"와 "영희가 철수를 좋아한다"를 구분 못 함 | **위치 인코딩** |
| **한 관점만 본다** | 한 번의 어텐션은 한 종류의 관계(예: 문법)만 포착 | **멀티헤드** |
| **깊이 쌓으면 학습이 무너진다** | 수십 층 쌓으면 기울기 소실이 재발 | **잔차연결 · 정규화** |

여기에 표현을 단어별로 한 번 더 가공하는 **피드포워드**까지 더하면, 어텐션을 둘러싼 네 부품이 완성됩니다.

---

## 핵심 개념 — 네 부품

### ① 멀티헤드 어텐션 (multi-head attention) — 여러 관점으로 보기

어텐션을 **한 번이 아니라 여러 번**(여러 헤드, 보통 8~96개) *병렬로* 수행하고 결과를 이어 붙입니다.

- 핵심 메커니즘: `dₖ` 차원을 한 덩어리로 쓰지 않고, **여러 작은 부분 공간으로 쪼개** 각 헤드에 맡깁니다. 예를 들어 512차원을 8개 헤드가 64차원씩 나눠 갖습니다. 각 헤드는 자기만의 `Wᵠ, Wᴷ, Wⱽ`를 학습하므로 *서로 다른 관계*를 봅니다.
- 발상: 한 헤드는 *문법 관계*(주어-동사), 다른 헤드는 *의미 관계*(대명사-지시 대상), 또 다른 헤드는 *멀리 떨어진 단어*에 주목하는 식으로, **서로 다른 종류의 관계를 동시에** 포착합니다.
- 직관: 같은 문장을 **여러 전문가가 각자의 관점으로** 읽고 의견을 종합하는 것.

```
멀티헤드 = 한 문장을 여러 관점으로 동시에 읽기

         ┌── Head 1 (문법 관계: 주어↔동사) ──┐
입력 ────┼── Head 2 (지시 관계: 그것↔명사) ──┼── 이어붙임 → 선형변환 → 출력
         ├── Head 3 (인접 단어) ───────────┤
         └── … Head 8 ────────────────────┘
```

### ② 위치 인코딩 (positional encoding) — 순서 정보 주입

어텐션이 잃어버린 *단어의 순서*를, 각 위치마다 고유한 **신호 벡터**를 임베딩에 더해 넣어 줍니다.

- 방법: 위치마다 **사인/코사인(sin/cos)** 곡선 값을 여러 주파수로 조합해 벡터를 만듭니다. 원논문의 식은 이렇습니다.

```
PE(pos, 2i)   = sin( pos / 10000^(2i/d) )
PE(pos, 2i+1) = cos( pos / 10000^(2i/d) )
   (pos = 위치, i = 차원 인덱스, d = 임베딩 차원)
```

- 그림으로 보면, 차원마다 *주파수가 다른 파동*입니다. 앞쪽 차원은 빨리 출렁이고(짧은 주기), 뒤쪽 차원은 천천히 출렁입니다(긴 주기). 이진수의 1·2·4·8 자리처럼, **빠른 파동과 느린 파동을 겹치면 모든 위치가 유일한 패턴**을 갖게 됩니다.

```
위치:   0    1    2    3    4    5    6    7  →
차원0 ∿∿∿∿∿∿∿∿  (빠름)   ┐
차원1 ∿  ∿  ∿  ∿        ├ 여러 주파수를 겹쳐
차원2 ∿     ∿     ∿     │ 위치마다 고유한 '지문'을 만든다
차원3 ∿        ∿        ┘ (느림)
```

- 왜 사인/코사인인가:
  ① **여러 주파수의 파동**을 겹쳐 각 위치에 *고유한 지문*을 만듭니다(시계의 초·분·시 바늘 조합으로 시각이 유일해지는 것과 같음).
  ② 위치가 멀어질수록 신호가 *부드럽게* 달라져, 모델이 "이 둘은 가깝다/멀다"는 **상대적 거리**를 감지할 수 있습니다.
  ③ 학습 때 본 적 없는 *더 긴 위치*에도 같은 공식으로 자연스럽게 확장됩니다(외삽).

> 참고: 원논문은 이 고정된 사인/코사인 방식을 썼지만, 이후 모델들은 **학습형 위치 임베딩**이나 **RoPE(회전 위치 인코딩)**, **ALiBi** 같은 발전형을 많이 씁니다. 핵심 아이디어("순서를 명시적으로 더해 준다")는 동일합니다 — 자세한 발전형은 심화편에서 다룹니다.

### ③ 잔차연결 & 레이어 정규화 — 깊이 쌓기 위한 안전장치

트랜스포머는 이 블록을 수십 층(GPT-3는 96층) 쌓습니다. 깊어지면 3단계의 **기울기 소실**이 다시 위협합니다.

- **잔차연결(residual connection)**: 각 부품의 출력에 **입력을 그대로 더합니다** — `출력 = 입력 + 부품(입력)`. 정보가 *지름길(shortcut)*로 흐를 길을 터 줘서, 역전파 때 기울기가 그 지름길을 타고 **곧장 흘러** 깊어도 죽지 않습니다 ([2]의 ResNet 아이디어).
  > 직관: "새로 계산한 것 = 원본 + 바뀐 부분(차이)". 부품은 *전체*가 아니라 *변화량(residual)*만 학습하면 되니 학습이 쉽고, 원본을 잃지 않으니 안전합니다.
- **레이어 정규화(layer normalization)**: 각 단어 벡터의 값들을 *평균 0, 분산 1로 다듬은* 뒤 스케일을 재조정해, 층마다 값의 분포가 들쭉날쭉해지지 않게 안정시킵니다 [3].
  > Pre-LN vs Post-LN: 원논문은 부품 *뒤*에 정규화를 뒀지만(Post-LN), 요즘 큰 모델은 부품 *앞*에 두는 **Pre-LN**을 즐겨 씁니다. 학습이 더 안정적이기 때문입니다. (`x + 부품(LN(x))` 형태)

### ④ 피드포워드 신경망 (feed-forward network, FFN) — 단어별 가공

어텐션으로 문맥을 섞은 뒤, **각 단어 벡터를 따로** 작은 2층 신경망(3단계의 MLP)에 통과시켜 표현을 한 번 더 비선형적으로 가공합니다. 보통 차원을 4배로 부풀렸다(`d → 4d`) 다시 줄이는(`4d → d`) 구조입니다.

- 역할 분담이 핵심입니다: **어텐션이 "단어들 *사이*의 정보 교환"이라면, 피드포워드는 "각 단어 *안*의 정보 가공"입니다.** 어텐션이 가로 방향 소통, FFN이 세로 방향 심화라고 보면 됩니다.
- 트랜스포머 파라미터의 약 2/3가 사실 이 FFN에 있습니다. "지식이 저장되는 곳"으로도 자주 지목됩니다.

## 방법론 — 한 블록의 흐름

이 부품들이 모여 **트랜스포머 블록** 하나를 이룹니다. 입력이 흐르는 순서는 이렇습니다(Pre-LN 기준).

```
입력 임베딩 + 위치 인코딩
        │
        ▼
 ┌────────────────────────┐
 │ (LayerNorm →) 멀티헤드 어텐션 │ ← 단어들 사이 정보 교환 (가로 소통)
 └────────────────────────┘
        │  ＋ 입력 (잔차연결: 지름길)
        ▼
 ┌────────────────────────┐
 │ (LayerNorm →) 피드포워드 신경망 │ ← 각 단어 안 정보 가공 (세로 심화)
 └────────────────────────┘
        │  ＋ 입력 (잔차연결: 지름길)
        ▼
   다음 블록으로  ─────  이 블록을 N번 반복해 쌓음 (N = 12, 24, 96 …)
```

이 블록을 여러 층 쌓을수록 **점점 더 추상적이고 깊은 언어 이해**가 형성됩니다. 연구로 관찰된 경향은 이렇습니다 — **낮은 층은 문법·품사 같은 표면 구조**를, **중간 층은 구문·의미 관계**를, **높은 층은 맥락·과제 특화 추상**을 잡습니다 [5]. 마치 CNN의 낮은 층이 모서리를, 높은 층이 얼굴을 잡는 것과 같은 위계입니다.

### 전체 모델을 한 장으로

블록 하나가 아니라 모델 전체를 보면 이렇습니다. 입력 토큰이 들어와 출력 확률이 나오기까지의 큰 줄기입니다(디코더 계열 기준).

```
   토큰 ID들  "나는 어제 사과를"
       │
       ▼  임베딩 룩업(1번 글) + 위치 인코딩(②) 더하기
   [ 단어 벡터들 ]
       │
       ▼
   ┌──────────────┐
   │ 트랜스포머 블록 1 │
   ├──────────────┤
   │ 트랜스포머 블록 2 │   각 블록 = 멀티헤드 어텐션 + FFN
   │      ⋮        │              (＋잔차·정규화)
   │ 트랜스포머 블록 N │   N = 12(GPT-2 small) ~ 96(GPT-3)
   └──────────────┘
       │
       ▼  마지막 층 정규화
   [ 문맥 벡터들 ]
       │
       ▼  출력 임베딩(보통 입력 임베딩과 공유) → softmax
   다음 토큰 확률분포   P("먹었다")=0.7, P("샀다")=0.1, …
```

마지막 단계에서 어휘 전체에 대한 확률이 나오는데, 이 "다음 토큰 확률"을 어떻게 학습하고 활용하는지가 **5단계의 출발점**입니다.

### 인코더 블록 vs 디코더 블록

같은 블록이지만 인코더용과 디코더용은 미묘하게 다릅니다. 5번 글의 BERT/GPT 구분이 사실 여기서 시작됩니다.

| | 인코더 블록 (BERT) | 디코더 블록 (GPT) |
|---|---|---|
| 셀프 어텐션 | 마스크 없음(양방향) | 인과 마스크(앞만) |
| 크로스 어텐션 | 없음 | (인코더-디코더 구조일 때만) 있음 |
| FFN·잔차·정규화 | 동일 | 동일 |

> 결국 **부품은 같고, 마스킹과 연결만 바뀝니다.** 트랜스포머가 강력한 건, 이 단순한 블록 하나를 *어떻게 마스킹하고 몇 개 쌓느냐*만으로 이해형·생성형·변환형 모델을 모두 만들어 내기 때문입니다.

## 흔한 오해

- **"위치 인코딩이 없어도 순서는 알 것이다"가 아닙니다.** 어텐션 자체에는 순서 개념이 *전혀* 없습니다(이를 '순열 불변'이라 합니다). 위치 정보를 명시적으로 더해 주지 않으면, 단어를 뒤섞어도 출력이 똑같아집니다.
- **"잔차연결은 성능을 위한 장식이다"가 아닙니다.** 잔차연결이 없으면 깊은 트랜스포머는 학습이 거의 불가능합니다. 깊이를 *가능케 한* 필수 장치입니다.
- **"멀티헤드는 그냥 어텐션을 여러 번 한 것이다"가 절반만 맞습니다.** 핵심은 각 헤드가 *서로 다른 부분 공간*에서 *다른 관계*를 보도록 갈라진다는 점입니다. 단순 반복이 아니라 **관점의 분업**입니다.
- **"피드포워드는 별로 안 중요하다"가 아닙니다.** 모델 파라미터의 다수가 FFN에 있고, 사실적 지식이 주로 여기 저장된다는 연구가 많습니다. 어텐션만큼이나 핵심 부품입니다.

---

## 한 줄 요약

> **트랜스포머 = (위치 인코딩으로 순서 주입) + (멀티헤드 어텐션으로 단어 사이 교환) + (피드포워드로 단어 안 가공), 그리고 잔차연결·정규화로 깊게 쌓기.**
> 각 부품은 셀프 어텐션의 빈틈(순서·관점·깊이)을 메우려고 들어온 처방이다.
