## 한 줄 요약 노트

| 주제 | 목적 (왜 필요한가) | 핵심 아이디어 (한 문장) |
|---|---|---|
| **다음 토큰 예측** | "GPT가 무엇을 배웠나"에 답하려고 | 단 하나의 목표=다음 토큰 예측(=로그우도↑=교차엔트로피↓), 능력은 그 부산물 |
| **토큰화·컨텍스트** | 텍스트를 모델의 언어(숫자)로 바꾸려고 | BPE=자주 쓰는 덩어리는 통째로·드문 건 잘게. 컨텍스트=한 번에 보는 작업대 크기 |
| **사전학습 규모** | '거대함'이 왜 힘인지 알려고 | 데이터·파라미터·연산의 *균형*. 사전학습→파인튜닝→정렬 3단계 |
| **스케일링·창발** | 키우면 무슨 일이 생기는지 알려고 | 멱법칙으로 매끄러운 향상(수확 체감) + 일부 능력의 갑작스러운 출현(논쟁 중) |
| **샘플링·퍼플렉서티** | LLM을 쓰고 평가하려고 | temperature=모험심·top-k/p=후보 거르기, 퍼플렉서티=평균 망설임(교차엔트로피의 지수) |

> 오른쪽을 가리고 왼쪽만 보며 말해 보세요. 그게 진짜 점검입니다.

## 이 장의 핵심 수식

| 수식 | 한 줄 의미 |
|---|---|
| `P(문장)=Πₜ P(xₜ\|x₁…xₜ₋₁)` | 문장 확률 = 각 토큰의 '앞 보고 다음 맞히기' 확률의 곱 |
| `목표: Σ log P(xₜ\|…) 최대화 = −Σ log q 최소화` | 다음 토큰 예측 = 로그우도 최대화 = **교차엔트로피 최소화**(1단계 ⑤) |
| `compute ≈ 파라미터 × 토큰 수 × 상수` | 연산은 모델 크기와 데이터 양의 곱 — 균형 문제의 근원 |
| `L ≈ N^(−α) + 상수` | 스케일링: 키울수록 손실↓, 단 멱법칙(수확 체감)·바닥 존재 |
| `PPL = exp(교차엔트로피)` | 퍼플렉서티 = 평균 망설임의 갈래 수(낮을수록 좋음) |
| `q' = softmax(logits / T)` | temperature T: 작으면 날카롭게(보수), 크면 평평하게(과감) |

> 5단계의 모든 수식은 1단계 **교차엔트로피**에 뿌리를 둡니다. LLM은 새 손실이 아니라, *분류 손실을 어휘 전체에 대해* 거대 규모로 쓴 것입니다.

## 이해 점검 (커리큘럼 기준)

말로 막힘없이 답할 수 있으면 체크하세요.

- [ ] **'다음 토큰 예측'이 어떻게 광범위한 언어 능력으로 이어지는지 설명할 수 있다.**
  - 힌트: 다음 단어를 *정말 잘* 맞히려면 지식·상식·추론이 필요 → 능력은 부산물.
- [ ] **토큰화가 왜 필요하고 BPE의 기본 아이디어가 무엇인지 안다.**
  - 힌트: 글자/단어의 절충. 자주 붙는 쌍을 합쳐 서브워드 어휘집을 만든다.
- [ ] **스케일링 법칙과 창발이 무엇인지 설명할 수 있다.**
  - 힌트: 멱법칙으로 매끄러운 향상(도서관) vs 갑작스러운 능력 출현(끓는점, 논쟁 중).
- [ ] **퍼플렉서티가 무엇을 측정하는지, temperature가 출력을 어떻게 바꾸는지 안다.**
  - 힌트: 퍼플렉서티=평균 망설임(교차엔트로피 지수). temperature↑=과감, ↓=보수.

### 한 걸음 더 (스스로 점검)

- [ ] 다음 토큰 예측이 '검색·복사'가 아니라 '확률분포에서 생성'임을 설명할 수 있다.
- [ ] 사전학습·파인튜닝·정렬 3단계가 각각 무엇을 하는지 비유로 말할 수 있다.
- [ ] 스케일링의 '수확 체감'이 왜 "무한히 좋아짐"이 아닌지 안다.
- [ ] top-k와 top-p의 차이(개수 고정 vs 확률 질량 고정)를 말할 수 있다.
- [ ] 퍼플렉서티가 낮아도 '유용·안전·사실성'은 따로 봐야 하는 이유를 안다.

## 다음 장 예고 — 6단계 · 최신 LLM (정렬·추론·효율·응용)

5단계에서 LLM이 **사전학습 → 파인튜닝 → 정렬**의 3단계로 만들어진다고 했습니다. 그 마지막 **정렬**부터가 6단계의 출발점입니다.

- **정렬(alignment)**: 사전학습 모델을 '사람이 원하는·안전한 답'에 맞추기 — **RLHF·DPO**. (퍼플렉서티만으론 못 재는 '유용·안전'을 다루는 단계.)
- **추론 모델(reasoning models)**: '생각을 더 오래' 하게 해 어려운 문제를 푸는 법(테스트타임 연산·생각의 사슬).
- **효율(MoE)·검색 증강(RAG)·에이전트**: 스케일링의 수확 체감(4절)을 넘어, *더 크게 대신 더 영리하게* 가는 길들.

> 5단계가 'LLM이 *무엇을 어떻게* 배우는가'였다면, 6단계는 'LLM을 *어떻게 더 쓸모 있고 안전하게* 만드는가'입니다. 마지막 단계로 넘어가세요.
