# 3. 확률과 통계 — 불확실성과 추론의 언어

앞의 두 절은 '확실한' 변환과 변화를 다뤘습니다. 하지만 현실의 데이터는 **불확실**합니다. 같은 증상이라도 병이 다를 수 있고, 같은 사진이라도 고양이일 '확률'이 있을 뿐입니다. 그래서 모델은 '정답'이 아니라 **확률**을 출력합니다.

이 절은 그 확률을 다루는 언어이자, 머신러닝의 손실함수 상당수가 **태어난 고향**입니다. 분류 모델이 왜 하필 교차엔트로피를 손실로 쓰는지, LLM이 왜 "다음 토큰의 로그우도"를 최대화하는지 — 그 답이 전부 이 한 절에서 나옵니다. 수학이 조금 깔린 절이지만, 끝에 가면 "확률을 출력한다 → 손실함수가 저절로 따라 나온다"는 단 하나의 사슬로 묶입니다.

---

## 목적 — 모델은 '확률'을 출력한다

"이 메일은 스팸이다"라고 딱 잘라 말하는 모델보다, **"스팸일 확률 92%"**라고 말하는 모델이 더 솔직하고 쓸모 있습니다.

- 확률로 말하면 **얼마나 확신하는지**까지 담깁니다. 0.51과 0.99는 둘 다 "스팸"이지만 확신의 정도가 전혀 다르죠.
- 그리고 결정적으로, **"정답에 얼마나 가까운가"를 잴 손실함수를 확률에서 끌어낼 수 있습니다.** 딱 잘라 "맞다/틀리다"만 말하는 모델은 "얼마나 틀렸나"를 잴 수 없어서, 어느 방향으로 고쳐야 할지(=기울기)도 알 수 없습니다. 확률은 그 연속적인 '틀린 정도'를 만들어 줍니다.

이 절의 목표는 두 가지입니다. ① 확률·통계의 기본 어휘를 잡는다. ② 그것이 어떻게 **교차엔트로피**라는 손실로 이어지는지 본다. 이 두 번째가 1단계와 3·5단계를 잇는 다리입니다.

> 핵심 결론을 먼저 말하면 — **분류 손실로 교차엔트로피를 쓰는 건 임의의 선택이 아니라, "데이터를 가장 그럴듯하게 만든다"는 확률론적 원리가 강제한 결론입니다.**

## 핵심 개념 — 분포에서 교차엔트로피까지

### ① 확률분포 (probability distribution) — "가능성의 배분"

일어날 수 있는 결과들에 가능성을 나눠 준 것입니다. 핵심 성질은 두 가지 — 모든 확률은 0 이상이고, 다 더하면 1입니다(전부 합쳐 100%). 자주 쓰는 셋만 봅니다.

- **베르누이(Bernoulli)**: 결과가 둘뿐(예/아니오, 스팸/정상). "성공 확률 p"로 요약. → 이진 분류. 모델의 마지막 시그모이드 출력이 이것.
- **범주형(categorical)**: 결과가 여러 개(개/고양이/새…). 각 후보에 확률을 배분, 다 더하면 1. → 다중 분류. 모델의 마지막 softmax 출력이 바로 이것.
- **정규분포(normal/Gaussian)**: 종 모양. 평균 둘레에 몰리고 멀수록 드뭅니다. 키·오차·노이즈 등 '평균 근처에 모이는' 연속값에 두루 등장. 회귀 모델에서 "오차가 정규분포를 따른다"고 가정하면, 그 MLE가 정확히 **최소제곱(평균제곱오차)**이 됩니다.

| 분포 | 결과의 형태 | 대표 쓰임 | 출력층 |
|---|---|---|---|
| 베르누이 | 둘 중 하나 | 이진 분류 | 시그모이드 |
| 범주형 | 여럿 중 하나 | 다중 분류 | softmax |
| 정규분포 | 연속값 | 회귀, 노이즈 모델 | (선형 출력) |

### ② 기댓값·분산 (expectation, variance) — "평균과 흩어짐"

- **기댓값 E[X]**: 결과를 확률로 가중평균한 값. *장기적으로 평균 얼마인가*. 식으로는 `E[X] = Σ x·P(x)` — 각 결과에 그 확률을 곱해 더한 것입니다. 주사위라면 `1·(1/6) + … + 6·(1/6) = 3.5`.
- **분산/표준편차**: 결과가 평균에서 얼마나 흩어졌나. *예측이 얼마나 들쭉날쭉한가*. 식으로는 `Var[X] = E[(X − E[X])²]`, 곧 "평균에서 벗어난 거리의 제곱"의 평균입니다. (2단계 '편향–분산 트레이드오프'에서 이 분산을 다시 만납니다.)

머신러닝의 손실함수는 거의 다 *기댓값* 형태입니다. "전체 데이터에 대한 평균 손실을 줄인다"는 문장이 곧 "손실의 기댓값을 최소화한다"는 뜻이죠. 그래서 이 어휘가 알게 모르게 모든 학습식의 바탕에 깔려 있습니다.

### ③ 조건부확률·베이즈 정리 (conditional probability, Bayes) — "증거로 믿음을 갱신"

- **조건부확률 P(A|B)**: "B가 일어났다는 걸 알 때 A일 확률". *추가 정보가 들어왔을 때의 확률.* 표본 공간을 'B가 일어난 세계'로 좁혀서 다시 보는 것입니다.
- **베이즈 정리(Bayes' theorem)**: 관찰한 증거로 기존 믿음을 **뒤집어 갱신**하는 공식.

  ```
  P(A|B) = P(B|A) · P(A) / P(B)
  ```

  - `P(A)`: **사전확률** — 증거 보기 전의 믿음.
  - `P(B|A)`: **우도(likelihood)** — A가 사실이라면 이 증거가 나올 만한가.
  - `P(A|B)`: **사후확률** — 증거를 본 뒤 갱신된 믿음.
  - `P(B)`: 증거 자체가 나올 전체 확률(정규화 상수, 분모로서 합을 1로 맞춤).

베이즈 정리가 왜 성립하는지는 한 줄로 보입니다. 조건부확률의 정의에서 `P(A,B) = P(A|B)P(B)`이자 `P(A,B) = P(B|A)P(A)`. 두 표현이 같은 `P(A,B)`이므로 등식으로 묶고 `P(B)`로 나누면 끝입니다. 어려운 특수 공식이 아니라, *동시에 일어날 확률을 두 방향으로 적은 것*뿐입니다.

직관적인 예 하나. 인구 1만 명 중 100명만 걸리는 희귀병(사전확률 1%)이 있고, 검사 정확도가 99%라 합시다. 양성이 나왔을 때 실제로 병일 확률은? 놀랍게도 약 50%에 불과합니다. 병자 100명 중 99명이 양성, 건강한 9,900명 중에서도 1%인 99명이 거짓 양성으로 나오니까요. **"검사 99% 정확 ≠ 양성이면 99% 환자"** — 이 흔한 착각을 베이즈가 정확히 교정합니다. 스팸 필터·의료 진단의 뼈대가 바로 이 계산입니다.

### ④ 최대우도추정 (MLE, Maximum Likelihood Estimation) — "데이터를 가장 잘 설명하는 손잡이"

모델의 파라미터를, **관측된 데이터가 나올 확률(우도)을 가장 크게** 만드는 값으로 고르는 방법입니다.

- 한 문장: *"이 데이터가 가장 그럴듯해 보이게 하는 파라미터가 정답"*.
- 실전에서는 우도를 직접 최대화하지 않고 **로그우도**를 최대화합니다. 데이터가 독립이면 우도는 각 데이터 확률의 *곱*인데(`P(전체) = ∏ P(xᵢ)`), 수백만 개의 0~1 사이 값을 곱하면 0으로 사라져(언더플로) 컴퓨터가 다루지 못합니다. 로그를 씌우면 곱이 합으로 바뀌어(`log ∏ = Σ log`) 수치적으로 안정되고, 로그는 증가함수라 최댓값의 위치도 그대로입니다.
- 이것이 학습의 한 정의입니다. 그리고 로그우도를 최대화하는 것은, 부호만 뒤집으면 **교차엔트로피를 최소화**하는 것과 정확히 같아집니다. (5단계 LLM의 목표함수가 바로 이것!)

### ⑤ 엔트로피·교차엔트로피·KL 발산 — "정보와 거리"

- **엔트로피(entropy)**: 한 분포의 **불확실성/놀라움의 양**. `H(p) = −Σ p(x) log p(x)`. 결과가 고를수록(예측 어려울수록) 큽니다. 동전이 공정하면(50:50) 엔트로피 최대, 한쪽으로 치우칠수록 작아집니다. "한 사건을 알리는 데 평균 몇 비트가 필요한가"로도 읽힙니다.
- **교차엔트로피(cross-entropy)**: 정답 분포 `p`를 모델 분포 `q`로 설명할 때 드는 **평균 놀라움**. `H(p,q) = −Σ p(x) log q(x)`. 모델이 정답을 잘 맞힐수록 작아집니다. → **분류의 표준 손실함수**.
- **KL 발산(KL divergence)**: 두 분포가 얼마나 **다른가**(거리 비슷한 양). `KL(p‖q) = Σ p(x) log[p(x)/q(x)] ≥ 0`. 핵심 관계는 `교차엔트로피 = 엔트로피 + KL`, 곧 `H(p,q) = H(p) + KL(p‖q)`입니다. 정답 분포 `p`의 엔트로피 `H(p)`는 모델과 무관한 상수이므로, **교차엔트로피를 줄이는 것 = KL을 줄이는 것 = 모델 분포가 정답 분포에 가까워지는 것**입니다.

```
교차엔트로피 = 엔트로피 + KL 발산

  H(p, q)   =   H(p)    +   KL(p‖q)
 ┌───────┐    ┌──────┐    ┌────────┐
 │ 모델로 │    │ 정답 │    │ 모델과 │
 │ 설명할 │  = │ 자체의│ + │ 정답의 │
 │때 드는 │    │ 불가피한│   │ 어긋남 │
 │ 놀라움 │    │ 불확실성│   │(줄일 수 │
 └───────┘    └상수──┘    └있음)───┘
```

## 직관 — 교차엔트로피는 '놀란 정도의 벌점'

모델이 각 후보에 확률을 매겼다고 합시다. 정답이 공개됩니다. 이때 **정답에 모델이 매겼던 확률**을 봅니다.

- 정답에 0.95를 줬다 → "거의 맞췄네", 거의 안 놀람 → **벌점 작음**.
- 정답에 0.02를 줬다 → "이게 정답이라고?!", 크게 놀람 → **벌점 큼**.

교차엔트로피 `−log q(정답)`는 바로 이 **'놀란 정도'를 벌점으로 매긴 것**입니다. 왜 하필 로그일까요? 두 가지 성질 때문입니다. ① 확률이 1이면 `−log 1 = 0`(완벽히 맞히면 벌점 0). ② 확률이 0에 가까워지면 벌점이 **무한히** 커집니다(정답에 거의 0을 준 자신만만한 오답은 가혹하게 처벌). 이 "확신에 찬 오답일수록 더 큰 벌"이 모델을 빠르게 교정하는 비결입니다.

```
정답에 준 확률 q     →   벌점 −log q
   0.99                    0.01   (거의 맞힘, 벌점 ≈ 0)
   0.50                    0.69
   0.10                    2.30
   0.01                    4.61   (확신에 찬 오답, 큰 벌점)
   →0                      →∞
```

> 학습이란 이 벌점(놀라움)의 평균을 낮추는 일 = **"정답에 높은 확률을 주도록 모델을 길들이는 것"**. 이것이 분류 모델이 배우는 방식의 전부입니다.

## 방법론 — 확률이 손실함수가 되기까지

```
모델이 확률 출력 ──▶ MLE: 데이터가 가장 그럴듯하게 ──▶ 로그우도 최대화 ──▶ 교차엔트로피 최소화
 (softmax 등)         (관측 데이터의 우도 ↑)            (곱→합으로 안정화)      (= 분류 손실함수)
```

- 이 사슬이 핵심입니다. **"확률을 출력한다"에서 출발하면, 손실함수가 저절로 따라 나옵니다.**
- 그래서 분류 문제의 손실로 교차엔트로피를 쓰는 건 임의의 선택이 아니라 **확률론이 정해 준 결론**입니다. "왜 평균제곱오차가 아니라 교차엔트로피?"라는 질문의 답이 여기 있습니다 — 분류 출력은 범주형 분포이고, 그 MLE가 곧 교차엔트로피이기 때문입니다.
- 5단계에서 LLM의 학습 목표("다음 토큰의 로그우도 최대화")가 똑같은 사슬임을 보게 됩니다. 어휘 5만 개에 대한 거대한 softmax를 두고, "실제로 나온 다음 토큰"에 높은 확률을 주도록 교차엔트로피를 최소화하는 것 — 그게 GPT 학습의 전부입니다.

## 수학 — 두 핵심 식과 그 의미

```
베이즈 정리:  P(A|B) = P(B|A) · P(A) / P(B)
```

> **이 식의 의미:** 새 증거 `B`를 보고 가설 `A`에 대한 믿음을 갱신한다. (증거 보기 전 믿음 `P(A)`) × (그 증거의 그럴듯함 `P(B|A)`)에 비례해 사후 믿음이 정해진다. — *추론을 수식으로 적은 것.*

```
교차엔트로피:  H(p, q) = − Σₓ p(x) · log q(x)
```

> **이 식의 의미:** 정답 분포 `p`와 모델 분포 `q`가 얼마나 어긋나는지를 잰다. 정답인 항의 모델 확률 `q`가 1에 가까울수록 값이 0에 가까워진다. 학습은 이 값을 **최소화** = "정답에 높은 확률을 주게" 모델을 민다.

분류에서 정답 분포 `p`는 보통 **원-핫**입니다(정답 칸만 1, 나머지 0). 그러면 합 `Σ`에서 정답 항만 살아남아, 교차엔트로피가 깔끔하게 `−log q(정답)`로 줄어듭니다 — 위 직관 절의 '놀란 정도의 벌점'과 정확히 같은 식이죠.

## 흔한 오해

- **"확률을 출력하면 그게 진짜 확신도다"가 아닙니다.** 모델이 0.92를 내놓아도 실제로 92%만 맞는다는 보장은 없습니다. 이 어긋남을 '보정(calibration)' 문제라 하며, 출력 확률은 *확신의 표현*이지 *진리*가 아닙니다. 현대 거대 모델은 종종 과신(overconfident) 경향을 보입니다.
- **"교차엔트로피와 MLE는 다른 것이다"가 아닙니다.** 둘은 같은 동전의 양면입니다. **로그우도 최대화 = 교차엔트로피 최소화**. 부호만 뒤집힌 같은 목표입니다.
- **"베이즈 정리는 어려운 특수 공식이다"가 아닙니다.** 그저 *증거로 믿음을 갱신하는* 자연스러운 절차이며, 조건부확률의 정의에서 한 줄로 유도됩니다. 스팸 필터부터 의료 진단까지, 일상의 추론이 이 형태입니다.
- **"검사가 99% 정확하면 양성일 때 99% 환자다"가 아닙니다.** 사전확률(병의 희귀함)을 무시한 착각입니다. 베이즈가 알려 주듯, 드문 병에서는 양성이라도 실제 환자일 확률이 의외로 낮습니다(기저율 무시 오류).

---

## 한 줄 요약

> **모델은 '정답'이 아니라 '확률'을 출력하고, 그 손실은 확률론에서 나온다.**
> 베이즈 = 증거로 믿음 갱신. MLE = 데이터를 가장 그럴듯하게 하는 파라미터 찾기 = 로그우도 최대화 = 교차엔트로피(놀란 정도의 벌점) 최소화. 이 사슬이 1단계와 3·5단계를 잇는다.
