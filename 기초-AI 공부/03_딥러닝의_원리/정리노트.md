# 3단계 정리노트 — 한 줄 요약 & 이해 점검

> 이 장의 모든 본문을 〈목적 / 핵심 아이디어〉 두 줄로 압축했습니다.

## 한 줄 요약 노트

| 주제 | 목적 (왜 필요한가) | 핵심 아이디어 (한 문장) |
|---|---|---|
| **퍼셉트론·MLP** | 거대 신경망을 '부품의 반복'으로 보려고 | 뉴런=가중합+활성화, 한 층=`Wx+b`, 깊이=추상 판단으로 키우는 결재 단계 |
| **활성화·비선형성** | 깊이를 진짜 힘으로 만들려고 | 활성화 없으면 아무리 쌓아도 한 층. 선형=펴기, 활성화=접기, 접어야 복잡한 경계 |
| **순전파·역전파** | 자기 잘못을 거슬러 고치려고 | 역전파=손실 책임을 출력→입력으로 나누기=**연쇄법칙의 반복 적용** |
| **손실·옵티마이저** | 무엇을·어떻게 줄일지 고르려고 | 손실=목표(회귀 MSE/분류 교차엔트로피), 옵티마이저=전략(SGD·모멘텀·Adam) |
| **CNN·RNN** | 데이터 구조를 공짜로 쓰려고 | CNN=공간(지역성·가중치 공유), RNN=순서(재귀·기억), 학습 루프는 동일 |
| **학습 안정화** | 깊이의 대가를 치르려고 | 신호·기울기 음량을 1 부근으로 관리(ReLU·초기화·BN), 과적합은 드롭아웃 |

> 오른쪽을 가리고 왼쪽만 보며 말해 보세요. 그게 진짜 점검입니다.

## 이 장의 핵심 수식

| 수식 | 한 줄 의미 |
|---|---|
| `z = Wx + b`,  `a = f(z)` | 한 층 = 펴기(선형) 다음 접기(비선형) |
| `W₂(W₁x+b₁)+b₂ = W′x+b′` | 활성화 없으면 두 층이 한 층 — 비선형이 필수인 증명 |
| `∂L/∂x = (∂L/∂z)(∂z/∂y)(∂y/∂x)` | 역전파 = 연쇄법칙이 층 수만큼 길게 이어진 것 |
| `MSE = (1/n)Σ(y−ŷ)²` | 회귀 손실: 값의 거리를 제곱해 벌점 |
| `H(p,q) = −Σ p log q` | 분류 손실: 정답에 낮은 확률 줄수록 큰 벌점 |
| `θ ← θ − η·(가공된 ∇L)` | 옵티마이저는 ∇L 가공만 다른, 경사하강법의 변형 |
| `앞층 기울기 ≈ g₁·g₂·…·gₙ` | 곱이 길어져 0(소실)·∞(폭발) — 안정화가 필요한 이유 |

> 모든 수식의 뿌리는 1단계입니다. 특히 **연쇄법칙**(②)이 역전파로, **교차엔트로피**(⑤)가 분류 손실로 살아납니다.

## 이해 점검 (커리큘럼 기준)

말로 막힘없이 답할 수 있으면 체크하세요.

- [ ] **비선형 활성화 함수가 없으면 왜 층을 쌓는 의미가 사라지는지 설명할 수 있다.**
  - 힌트: 선형의 합성은 선형 하나. 종이를 '접어야' 복잡한 경계가 생긴다.
- [ ] **역전파가 '연쇄법칙의 반복 적용'임을 수식 흐름으로 설명할 수 있다.**
  - 힌트: 손실에서 입력으로 거슬러, 각 단계 민감도를 곱하며 책임 분배.
- [ ] **MSE와 교차엔트로피를 각각 언제 쓰는지, 왜 그런지 안다.**
  - 힌트: 회귀=MSE(값 거리), 분류=교차엔트로피(확률 어긋남, MLE와 연결).
- [ ] **기울기 소실 문제가 무엇이고 ReLU·정규화가 어떻게 돕는지 설명할 수 있다.**
  - 힌트: 작은 값의 긴 곱 → 0. ReLU(기울기 1 유지)·BN(음량 재조정).
- [ ] **CNN과 RNN이 각각 어떤 구조적 가정을 활용하는지 설명할 수 있다.**
  - 힌트: CNN=지역성·가중치 공유(공간), RNN=재귀·기억(순서).

### 한 걸음 더 (스스로 점검)

- [ ] 역전파(기울기 *구하기*)와 옵티마이저(가중치 *고치기*)가 별개 동작임을 안다.
- [ ] 가중치를 전부 0으로 초기화하면 왜 안 되는지(대칭 문제) 말할 수 있다.
- [ ] 드롭아웃이 추론 때는 꺼진다는 것과 그 이유를 안다.
- [ ] RNN의 '장거리 의존성·속도' 한계가 왜 트랜스포머를 부르는지 설명할 수 있다.

## 다음 장 예고 — 4단계 · 트랜스포머 & 언어 모델의 원리

3단계 마지막에서 RNN의 두 약점을 봤습니다 — **순차 처리라 느리고, 멀리 떨어진 단어의 관계를 잘 기억 못 한다.**
4단계의 주인공 **트랜스포머(Transformer)**는 바로 이 두 문제를 정면으로 푼 구조입니다.

- 핵심 아이디어: 단어를 순서대로 하나씩 읽는 대신, **모든 단어가 서로를 한꺼번에 바라보게** 한다 → **셀프 어텐션(self-attention)**.
- 1단계의 **내적**(닮음 재기)이 여기서 "어떤 단어가 어떤 단어를 얼마나 참고할지"를 정하는 데 쓰입니다.
- 핵심 수식 미리보기: **`softmax(QKᵀ / √dₖ) · V`** — 각 기호의 의미를 4단계에서 낱낱이 풉니다.

> 3단계가 '신경망이 배우는 법'이었다면, 4단계는 '언어를 다루는 신경망의 모양'입니다. 그리고 그 위에서 5단계의 LLM이 태어납니다.
