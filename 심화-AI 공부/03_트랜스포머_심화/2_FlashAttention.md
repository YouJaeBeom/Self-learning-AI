# 2. FlashAttention — 결과는 그대로, 메모리 병목만 풀기

앞 글에서 O(n²)이 연산뿐 아니라 메모리도 때린다고 했습니다. FlashAttention의 통찰이 바로 여기 있습니다. 어텐션이 느린 진짜 이유는 계산량이 아니라, 거대한 점수표를 느린 메모리에 쓰고 다시 읽느라 시간을 버리는 데 있다는 것입니다. 이 병목만 없애면 같은 결과를 훨씬 빠르게 얻을 수 있습니다. 이 글은 그 발상이 어떻게 "정확하면서도 빠른" 어텐션을 만들어 냈는지, 그래서 왜 사실상 표준이 됐는지를 다룹니다.

## 계산이 아니라 메모리 이동을 줄이기

GPU에는 두 종류의 메모리가 있습니다. 아주 빠르지만 작은 메모리(SRAM, 작업대)와, 크지만 느린 메모리(HBM, 창고)입니다. 기본 어텐션은 n 곱하기 n 점수표를 통째로 만들어 느린 창고(HBM)에 썼다가 다시 읽습니다. 이 왕복이 진짜 병목입니다.

FlashAttention의 목적은 거대한 점수표를 창고에 통째로 쓰지 않고, 작업대에서 조각조각 처리해 메모리 왕복을 최소화하는 것입니다. 그러면서 결과인 어텐션 출력은 정확히 동일합니다. 계산하는 함수를 바꾼 것이 아니라, 그 함수를 계산하는 방식(데이터를 어디서 어떻게 옮기며 계산하는가)을 바꾼 것입니다.

## 메모리 계층과 IO 인식

먼저 메모리 계층을 짚어 보겠습니다. SRAM(작업대)은 매우 빠르지만 작아서 한 번에 조금만 올릴 수 있습니다. HBM(창고)은 크지만 느려서, 데이터를 여기 두고 필요할 때 꺼내 씁니다. 핵심 사실은 현대 GPU에서 어텐션은 연산 장치가 놀고 있는데도 메모리 왕복을 기다리느라 느리다는 것입니다. 이렇게 연산이 아니라 메모리 이동이 발목을 잡는 작업을 메모리 바운드(memory-bound)라고 합니다.

FlashAttention은 "연산을 줄이자"가 아니라 "느린 창고 왕복을 줄이자"에 집중합니다. 그래서 IO 인식(IO-aware) 어텐션이라 불립니다. 어디가 진짜 병목인지를 정확히 짚은 설계입니다.

## 타일링과 온라인 softmax

이를 구현하는 두 기법이 핵심입니다. 첫째는 타일링(tiling)입니다. 거대한 점수표를 작은 타일(블록)로 쪼개, 작업대(SRAM)에 올라갈 만큼씩만 가져와 처리합니다. 그러면 전체 n 곱하기 n 점수표를 창고에 만들 필요가 없어집니다.

둘째는 온라인 softmax(online softmax)입니다[3]. softmax는 보통 한 줄 전체를 봐야 정확히 계산되는데, FlashAttention은 타일을 하나씩 보며 부분 결과를 누적하고 보정해 가며 정확한 softmax를 완성합니다. 전체 점수표를 저장하지 않고도 같은 답에 도달하는 것입니다.

결과적으로 n 곱하기 n 점수표를 창고에 만들지 않습니다. 메모리 사용이 제곱에서 거의 선형으로 확 줄고, 왕복이 줄어 훨씬 빨라집니다. 그러면서 출력은 비트 단위로 완전히 같지는 않더라도 수학적으로 동일한 정확한 어텐션입니다.

## 거대한 표를 책상에서 처리하기

회계사가 100만 행짜리 장부를 더해야 한다고 합시다. 기본 방식은 장부 전체를 창고(느림)에 펼쳐 두고, 한 줄 보러 창고를 오가며 합산하는 것입니다. 왕복에 하루를 다 씁니다. FlashAttention 방식은 장부를 한 묶음씩 가져와 책상(빠름) 위에서 처리하고, 중간 합계만 들고 다음 묶음으로 넘어가는 것입니다. 창고 왕복이 거의 없어 몇 시간 만에 끝납니다. 더한 결과는 완전히 같습니다. 단지 일하는 동선을 바꿨을 뿐입니다.

정리하면 FlashAttention은 계산을 바꾼 것이 아니라, 어디서 어떻게 옮기며 계산하는지(동선)를 바꿔 시간을 아낀 것입니다. 같은 답을 더 빨리 내는 셈입니다.

## 무엇이 달라지나

```
기본 어텐션:   Q,K,V → [n×n 점수표를 HBM(창고)에 생성] → softmax → ×V → 출력
                              ↑ 거대 표 왕복 = 병목

FlashAttention: Q,K,V를 타일로 쪼갬 → SRAM(작업대)에서 블록별 계산 + 온라인 softmax 누적
                              ↑ 거대 표를 창고에 안 만듦 → 메모리↓·속도↑ (결과 동일)
```

FlashAttention은 정확한(exact) 어텐션입니다. 앞 글의 희소·선형 어텐션처럼 근사가 아니라, 같은 결과를 더 싸게 줍니다. 이 점이 폭넓게 채택된 결정적 이유입니다(2026-06 기준 사실상 표준). 이후 FlashAttention-2[2], FlashAttention-3 등 후속이 GPU 활용을 더 끌어올리며 긴 문맥 학습과 추론을 현실적으로 만들었습니다.

여기서 얻을 교훈이 하나 있습니다. "더 빠르게"가 꼭 "연산을 줄이기"를 뜻하지는 않는다는 것입니다. 때로는 데이터를 어떻게 옮기느냐가 전부입니다. 같은 수식이라도 하드웨어를 의식해 구현하면 몇 배 빨라질 수 있고, 이것이 알고리즘과 하드웨어가 만나는 대표 사례입니다.

## 수식으로 보는 변화

```
출력:   softmax(QKᵀ / √dₖ) · V       ← 수학적으로 동일 (기초편 그대로)
메모리:  O(n²)  →  O(n)               ← n×n 표를 저장 안 함 (바뀐 것)
```

계산하는 함수는 기초편 어텐션과 똑같습니다. 바뀐 것은 메모리 사용량뿐입니다. 거대한 점수표를 창고에 저장하지 않아 메모리가 제곱에서 거의 선형으로 줄어듭니다. 결과를 바꾸지 않고 병목만 없앤 것입니다.

## 흔한 오해

"FlashAttention은 근사라 정확도가 떨어진다"는 오해입니다. 정확한 어텐션입니다. 희소·선형과 달리 결과를 바꾸지 않습니다. 바꾼 것은 메모리 동선뿐입니다.

"연산량을 줄여서 빨라졌다"는 오해입니다. 연산량(O(n²))은 비슷합니다. 메모리 왕복을 줄여 빨라진 것입니다. 병목이 어디였는지를 정확히 짚은 것이 핵심입니다.

"어텐션 알고리즘이 다 거기서 거기다"라는 오해입니다. 같은 수식도 하드웨어를 의식해 구현하면 몇 배 빨라집니다. 알고리즘과 하드웨어의 만남이 중요해진 대표 사례입니다.

## 정리

FlashAttention은 거대한 n 곱하기 n 점수표를 느린 메모리에 저장하지 않고, 작은 타일로 쪼개 빠른 메모리에서 처리(타일링+온라인 softmax)해 같은 결과를 훨씬 빠르게 냅니다. 줄인 것은 연산이 아니라 메모리 왕복(IO 인식)입니다. 근사가 아닌 정확한 어텐션이라 폭넓게 표준이 됐습니다. 다음 글에서는 또 다른 각도, 곧 위치를 더 똑똑하게 주어 긴 문맥에 강해지는 RoPE와 ALiBi로 넘어갑니다.
