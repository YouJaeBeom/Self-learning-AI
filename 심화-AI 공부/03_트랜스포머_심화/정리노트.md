# S3 정리노트 — 한눈에 보는 트랜스포머 심화

이 모듈의 본문을 한 장으로 압축했습니다. 네 글의 핵심을 다시 짚고, 큰 그림과 대표 논문, 다음 모듈로 가는 다리를 정리합니다.

## 네 글의 요지

| 주제 | 왜 필요한가 | 핵심 한 문장 |
|---|---|---|
| 어텐션의 비용 | 한계를 직시하려고 | 모든 토큰 쌍 계산 → 비용이 길이의 제곱 O(n²)(연산+메모리). "모두가 모두와 악수" |
| FlashAttention | 메모리 병목을 풀려고 | n×n 표를 저장 안 하고 타일로 처리(IO 인식). 결과는 같고 메모리 왕복만↓ |
| 위치 RoPE·ALiBi | 순서 감각 + 긴 문맥 | 절대 좌표→상대 거리. RoPE=Q·K 회전, ALiBi=먼 토큰 점수 감쇠 |
| 긴 문맥·KV 캐시 | 재계산을 피하려고 | 계산한 K·V를 저장·재활용(속도↑). 대가는 길이에 비례하는 캐시 메모리 |

## 이 모듈의 핵심 수식

| 수식 | 한 줄 의미 |
|---|---|
| `어텐션 비용 ∝ n²` | 토큰 쌍이 n×n → 길이의 제곱으로 폭발 |
| `softmax(QKᵀ/√dₖ)·V` (동일), 메모리 `O(n²)→O(n)` | FlashAttention: 함수는 그대로, 메모리만 절감 |
| `RoPE: Q·K를 위치만큼 회전` | 내적에 상대 거리(i−j)가 녹아듦 |
| `ALiBi: 점수 − m·\|i−j\|` | 먼 토큰일수록 점수를 비례 감쇠 |
| `KV 캐시 메모리 ∝ 길이 × 층 × 헤드` | 속도는 살리되 메모리가 길이에 비례(GQA로 절감) |

큰 그림은 이렇습니다. 하나의 적 O(n²)을 네 각도로 공격합니다. 정확히 더 싸게(FlashAttention·KV 캐시) 가는 길과 근사로 더 싸게(희소·선형) 가는 길을 구분하는 것이 핵심이고, 위치(RoPE·ALiBi)는 긴 문맥 일반화를 받칩니다.

## 이 장의 대표 논문

| 주제 | 논문 | 핵심 한 줄 |
|---|---|---|
| 비용 우회 | Sparse Transformers (Child et al., 2019)[1] | 일부 쌍만 보는 희소 어텐션 |
| 비용 우회 | Linear Transformers (Katharopoulos et al., 2020)[2] | 곱셈 순서를 바꿔 O(n) |
| 정확·고속 | FlashAttention (Dao et al., 2022)[3] | IO 인식으로 메모리 왕복↓, 정확 어텐션 |
| 위치 | RoPE / RoFormer (Su et al., 2021)[4] | Q·K 회전으로 상대 위치 (널리 채택) |
| 위치 | ALiBi (Press et al., 2021)[5] | 거리 비례 감쇠, 긴 문맥 외삽 |
| 캐시 효율 | GQA (Ainslie et al., 2023)[6] | 헤드 간 K·V 공유로 캐시↓ |
| 문맥 확장 | Position Interpolation (Chen et al., 2023)[7] | RoPE를 눌러 담아 학습보다 긴 문맥 |

2026-06 기준 FlashAttention·RoPE·GQA는 대형 LLM의 사실상 기본 부품이며, FlashAttention-2/3 같은 후속이 계속 나옵니다.

## 다음 모듈 예고 — S4 · LLM 사전학습 심화

S3는 어텐션 한 층을 빠르고 길게 만드는 공학이었습니다. S4는 시야를 넓혀, 그 트랜스포머를 수천 GPU와 수조 토큰으로 학습시키는 공학으로 갑니다. 질문이 "어텐션을 어떻게?"에서 "모델 하나를 어떻게 그 규모로 학습시키나?"로 바뀝니다. 데이터 큐레이션(무엇을 먹이나), 분산학습(왜 한 GPU로 안 되나), Chinchilla 최적(데이터 대 파라미터 균형), 혼합정밀 레시피를 다룹니다. 첫 대표 논문은 Scaling Laws(Kaplan et al., 2020), Chinchilla(Hoffmann et al., 2022)입니다.
