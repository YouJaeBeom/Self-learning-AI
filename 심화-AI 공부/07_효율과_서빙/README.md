# S7 · 효율·서빙 — 거대 모델을 싸고 빠르게

거대 모델을 싸고 빠르게 학습하고 배포하는 핵심 기법들을 잡는 모듈입니다. 기초편 6단계의 MoE·양자화를 게이팅·로드밸런싱·서빙 시스템 수준으로 확장합니다. 모델의 능력을 키우는 이야기가 아니라, 이미 만든 모델을 현실적 비용으로 굴리는 이야기입니다. 학습보다 추론(서빙) 비용이 누적되는 실제 서비스에서, 이 기법들이 곧 운영비를 좌우합니다.

기초편 6단계 3절(효율화: MoE와 양자화)에서 "총 파라미터는 크게, 활성은 작게"를 잡고 오면 좋습니다. 새 수학은 없습니다. 모두 연산·메모리를 어디서 아끼는가라는 공학적 직관입니다.

## 이 모듈에서 답하는 질문

- MoE의 게이팅은 어떻게 전문가를 고르고, 왜 '로드 밸런싱'이 따로 필요한가
- 양자화(GPTQ·AWQ·QLoRA)는 비트를 줄이면서 어떻게 성능을 지키는가
- speculative decoding은 어떻게 품질을 안 떨어뜨리고 속도만 올리는가
- KV 캐시는 무엇이고, PagedAttention(vLLM)은 서빙을 어떻게 빠르게 하는가

한마디로 "같은 모델을 더 적은 연산·메모리·시간으로 굴리는 방법은?"입니다.

총 파라미터 대 활성 파라미터(기초편 6단계 복습), 비트 수와 메모리·정밀도의 트레이드오프(양자화), 어텐션의 KV 캐시가 왜 반복 계산을 줄이는가면 충분합니다. 새 수학은 없으며, 모두 연산·메모리를 어디서 아끼는가라는 공학적 직관입니다.
