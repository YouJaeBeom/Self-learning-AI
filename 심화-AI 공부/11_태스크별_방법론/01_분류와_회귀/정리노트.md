# 01 분류와 회귀 (심화) · 정리노트

이 모듈의 본문을 한 장으로 압축했습니다. 세 글의 핵심을 다시 짚고, 꼭 구분할 짝과 핵심 식, 대표 논문을 정리합니다.

## 세 글의 요지

| 주제 | 왜 필요한가 | 핵심 한 문장 |
|---|---|---|
| 부스팅의 원리 | "보완"의 정체를 잡으려고 | 얕은 트리를 순차로 더하되, 각 트리가 잔차(=손실의 음의 기울기)를 맞춤 — 함수 공간의 경사하강 |
| 트리 앙상블 비교 | 무엇을 언제 쓰나 정하려고 | RF=배깅(분산↓·안전), GBM=부스팅(편향↓·고정확). 부스팅 3대: XGBoost(표준)·LightGBM(속도)·CatBoost(범주형·누수방지) |
| 트리 대 딥러닝 | 통념과 그 균열을 잡으려고 | 표 데이터는 불규칙·이질·소량이라 부스팅이 강했다. TabPFN은 in-context로 그 통념을 흔듦(Nature 2025) |

## 꼭 구분할 짝

| 짝 | 한쪽 | 다른쪽 |
|---|---|---|
| 배깅 대 부스팅 | 독립·병렬·분산↓ | 순차·잔차보완·편향↓ |
| 잔차 대 그래디언트 부스팅 | 제곱오차의 잔차 | 일반 손실의 음의 기울기(잔차의 일반화) |
| level-wise 대 leaf-wise | XGBoost(층 균형) | LightGBM(손실 큰 잎 우선) |
| ordered boosting | CatBoost — 타깃 누수 제거 | 일반 부스팅 — 미세 누수 위험 |
| 피팅 대 in-context | 데이터마다 새로 학습(트리) | 고정 모델이 데이터를 맥락으로 읽음(TabPFN) |

## 핵심 식의 의미

| 식 | 의미 |
|---|---|
| `Fₘ = Fₘ₋₁ + η·fₘ` | 지금 예측에 "손실 줄이는 방향" 트리를 학습률만큼 더함 |
| `fₘ ≈ −∂Loss/∂F` | 다음 트리가 맞힐 목표 = 손실의 음의 기울기(=제곱오차면 잔차) |
| `Ω = γ·잎수 + ½λ·w²` | XGBoost의 정규화 — 복잡한 트리에 직접 벌점 |
| `예측 = g(테스트행 \| 학습행 전부)` | TabPFN — 가중치 고정, 데이터를 맥락으로 추론 한 번 |

## 이 모듈의 대표 논문

| 논문 | 한 줄 |
|---|---|
| Random Forests (Breiman, 2001)[1] | 배깅+무작위 특성으로 분산을 잡은 앙상블 |
| XGBoost (Chen & Guestrin, 2016)[2] | 정규화·희소 인식으로 부스팅 대규모화 |
| LightGBM (Ke et al., 2017)[3] | GOSS·EFB로 부스팅 가속 |
| CatBoost (Prokhorenkova et al., 2018)[4] | ordered boosting으로 타깃 누수 제거 |
| TabPFN v2 (Hollmann et al., 2025, Nature)[5] | 합성 사전학습 트랜스포머의 in-context 정형 학습 |
