# 06 · 그래프와 GNN (심화) — 집계의 철학과 두 숙제

기초편의 "메시지 패싱" 한 틀이, 집계 방식에 따라 GCN·GraphSAGE·GAT로 갈라짐을 잡는 모듈입니다. 그리고 GNN의 두 근본 숙제 — 새 노드를 다루나(귀납 대 변환), 깊이의 저주(과평활) — 와 그 너머의 그래프 트랜스포머를 봅니다. 같은 메시지 패싱 위에서 "이웃을 어떻게 모으나"가 달라지면 무엇이 달라지는지가 이 모듈의 출발점입니다.

기초편 06 트랙에서 노드 임베딩·메시지 패싱의 직관을 잡고 오면 좋습니다. 새 모델 수학은 깊이 다루지 않습니다. 여기서는 집계 철학과 두 숙제(귀납·과평활)가 주인공입니다.

## 이 모듈에서 답하는 질문

- 같은 메시지 패싱인데 GCN·GraphSAGE·GAT는 집계에서 무엇이 다른가
- 변환적(transductive) 대 귀납적(inductive) — 학습 때 못 본 노드/그래프를 다룰 수 있나
- 층을 깊게 쌓으면 왜 망가지나(over-smoothing), 어떻게 완화하나
- 메시지 패싱의 한계(장거리 의존·병목)를 그래프 트랜스포머(GraphGPS)는 어떻게 넘나

## 이 모듈에 필요한 수학 (의미 수준)

정규화된 인접 행렬 전파(GCN의 `D⁻½ A D⁻½`)의 의미, 이웃 샘플링과 집계 함수(mean/pool/LSTM)의 트레이드오프(GraphSAGE), 어텐션 가중치 α_ij로 이웃 중요도 학습(GAT), 과평활을 보는 관점(반복 평균 → 표현 수렴)이면 충분합니다. 여기서는 집계 철학과 두 숙제(귀납·과평활)가 주인공입니다.
