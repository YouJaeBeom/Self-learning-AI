# 07 · 연합학습 (심화) — FedAvg가 부딪히는 현실의 벽

기초편의 FedAvg가 현실에서 부딪히는 네 도전 — non-IID·통신 비용·개인화·보안 집계 — 을 잡고, 2026년의 새 흐름인 연합 + 파운데이션 모델(LLM)까지 잇는 모듈입니다. 연합학습의 우아한 기본 알고리즘이 실제 배포 환경에서 왜 흔들리는지, 그리고 그것을 어떻게 보강하는지가 이 모듈의 핵심입니다.

기초편 07 트랙에서 FedAvg·프라이버시 동기의 직관을 잡고 오면 좋습니다. 새 모델 수학은 깊이 다루지 않습니다. 여기서는 네 도전과 그 해법의 직관이 주인공입니다.

## 이 모듈에서 답하는 질문

- 클라이언트 데이터가 서로 다를 때(non-IID) FedAvg는 왜 흔들리고, FedProx는 어떻게 잡나
- 통신 비용은 왜 연합학습의 진짜 병목인가
- 모두에게 하나의 모델이 맞나, 아니면 클라이언트별 개인화가 나은가(Ditto·FedBN)
- 서버가 개별 업데이트를 못 보게 하는 보안 집계는 어떻게 작동하나
- LLM 시대에 연합학습은 어떻게 변하나(FederatedScope-LLM)

## 이 모듈에 필요한 수학 (의미 수준)

non-IID에서 client drift(클라이언트들이 서로 다른 최적점으로 끌림), proximal term(글로벌에서 멀어지지 않게 당기는 정규화, FedProx), 개인화에서 글로벌과 로컬 모델의 절충(Ditto)과 BN 통계의 로컬 유지(FedBN), 보안 집계의 상쇄 마스크와 PEFT(LoRA 등)로 통신량 줄이기면 충분합니다. 여기서는 네 도전과 그 해법의 직관이 주인공입니다.
