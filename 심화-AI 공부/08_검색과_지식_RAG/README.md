# S8 · 검색·지식 (RAG 심화)

기초편 6단계의 RAG를 시스템 수준으로 확장하는 모듈입니다. "환각을 줄이는 검색 시스템은 실제로 어떤 부품으로 설계되는가?"에 답합니다. 모델은 자기 가중치 속 지식만 알고 모르는 것은 지어내는데, RAG는 그 한계를 모델 바깥의 검색으로 메웁니다. 다만 "검색해서 붙이면 끝"이 아니라, 각 부품의 설계가 품질을 좌우합니다.

기초편 6단계 4절(RAG)에서 "오픈북 시험" 직관을 잡고 오면 좋습니다. 이 모듈은 그 오픈북을 도서관 시스템으로 키웁니다. 새 수학은 코사인 유사도뿐이고, 나머지는 시스템 설계의 직관입니다.

## 이 모듈에서 답하는 질문

- 임베딩으로 의미 검색을 할 때, 수백만 문서에서 어떻게 빠르게 찾나(ANN·벡터DB)
- RAG의 전체 파이프라인(청킹 → 검색 → 증강 → 생성)에서 각 단계는 무엇을 정하나
- 키워드 검색(BM25)과 벡터 검색을 왜 섞고, 리랭킹은 무엇을 더하나
- GraphRAG는 일반 RAG가 못 하는 무엇을 하나, RAG는 어떻게 평가하나

한마디로 "모델 바깥의 지식을 정확히 찾아 붙이는 검색 시스템을 어떻게 짜는가?"입니다.

코사인 유사도(기초편 6단계 복습, 방향이 곧 의미), 근사 최근접 이웃(ANN, 정확함을 조금 양보해 속도를 사는 거래), BM25의 직관(키워드 빈도 + 희귀도)이면 충분합니다. 새 수학은 코사인 유사도뿐이고, 나머지는 시스템 설계의 직관입니다.
