# 5. 합성 데이터 — 후학습의 연료와 함정

앞 글에서 "AI가 피드백을 생성한다"를 봤습니다. 한 걸음 더 가면 데이터 자체를 모델이 만들어 쓰는 데 이릅니다. 합성 데이터(synthetic data)는 2025–2026 후학습의 핵심 연료입니다(2026-06 기준). 그런데 공짜 점심이 아닙니다. 잘 쓰면 사람 데이터의 한계를 넘는 연료가 되지만, 잘못 쓰면 모델을 서서히 망가뜨리는 독이 됩니다. 이 글은 합성 데이터의 쓸모와 함정을 함께 다룹니다. 특히 함정 쪽이 이 글의 절반입니다.

## 사람 데이터의 한계를 넘기

후학습에는 고품질 데이터가 많이 필요합니다. 그런데 사람이 만든 데이터는 비싸고 느리고 양이 한정적입니다. 특히 희귀한 능력 — 어려운 수학 풀이, 특정 코딩 패턴 — 은 사람 예시를 충분히 모으기 어렵습니다. 여기서 발상이 나옵니다. 이미 강한 모델로 학습용 데이터를 생성하고, 그것으로 (대개 더 작은) 모델을 가르치자는 것입니다.

## 합성 데이터의 종류

합성 데이터는 용도에 따라 몇 가지로 나뉩니다. 지시-응답 생성은 강한 모델이 (질문, 좋은 답) 쌍을 대량 생성해 SFT 데이터로 쓰는 것입니다[1]. 선호쌍 생성은 모델이 답을 여러 개 내고 스스로(또는 더 강한 모델이) 우열을 매겨 DPO·보상 모델 데이터로 쓰는 것입니다(앞 글의 RLAIF). 증류(distillation)는 강한 교사 모델의 출력을 학생 모델이 모방해 작은 모델에 능력을 이식하는 것입니다(S7 효율화와도 연결). 추론 흔적(reasoning traces)은 모델이 풀이 과정을 생성하고 정답에 이른 풀이만 골라 학습해 추론 능력을 강화하는 것입니다(S6에서 자세히).

핵심은 정답을 검증할 수 있는 영역(수학·코드)에서 특히 강력하다는 점입니다. 생성한 데이터의 옥석을 기계적으로 가릴 수 있기 때문입니다. 코드는 실행해 보면 맞는지 알 수 있고, 수학은 정답과 대조하면 됩니다.

## 생성보다 거르기가 본체다

합성 데이터의 품질은 거르기(filtering)에서 갈립니다. 막 생성한 데이터는 절반이 쓰레기일 수 있습니다.

```
① 생성: 강한 모델로 후보 데이터 대량 생산
② 거르기: 정답 검증 / 규칙 / 또 다른 모델 평가로 불량 제거
③ 학습: 살아남은 고품질만 후학습에 투입
```

수학·코드는 실행하거나 정답과 대조해 자동으로 거를 수 있어 합성이 매우 잘 통합니다. 반면 열린 글쓰기나 주관적 답은 옥석 가리기가 어려워 조심스럽게 써야 합니다. 검증 가능성이 합성 데이터의 성패를 가르는 셈입니다.

## 함정과 핵심 위험

여기서부터가 이 글의 핵심입니다. 합성 데이터에는 분명한 함정이 있습니다.

| 위험 | 무엇이 일어나나 |
|---|---|
| 모델 붕괴(model collapse) | 자기 출력으로만 반복 학습하면 분포의 꼬리(드문 사례)가 잘려 다양성이 무너지고 품질이 퇴화 |
| 편향·오류 증폭 | 생성 모델의 실수와 편향이 데이터에 박혀 다음 모델로 대물림 |
| 거짓 자신감 | 그럴듯하지만 틀린 데이터를 거르지 않고 쓰면 틀린 것을 확신 있게 배움 |
| 평가 오염 | 합성 데이터가 벤치마크 문제와 겹치면 점수가 부풀려짐 |

특히 모델 붕괴(model collapse)는 깊이 새겨 둘 만합니다. 모델이 자기 출력으로만 반복 학습하면, 분포의 꼬리(드문 사례)가 점점 잘려 나가 다양성이 무너지고 품질이 서서히 퇴화합니다[2]. "복사본의 복사본의 복사본"처럼 흐려지는 것입니다. 편향과 오류의 증폭도 같은 뿌리입니다. 생성 모델의 실수가 그대로 데이터에 박혀 다음 모델로 대물림됩니다(앞 글 CAI의 위험과 같은 구조). 그래서 원칙은 하나입니다. 합성 데이터는 "생성"이 아니라 "거르기"가 본체입니다. 검증 가능한 영역에 집중하고, 사람·실제 데이터와 섞어 쓰며, 다양성을 지켜야 합니다.

## 검증 가능성이 경계선을 긋는다

합성 데이터의 성패를 가르는 가장 중요한 한 가지를 꼽으라면 검증 가능성입니다. 정답을 기계적으로 확인할 수 있는 영역과 그렇지 않은 영역 사이에 뚜렷한 경계선이 그어집니다. 수학은 답을 대조하면 되고, 코드는 실행해 테스트를 통과하는지 보면 되고, 논리 퍼즐은 제약을 만족하는지 확인하면 됩니다. 이런 영역에서는 모델이 답을 잔뜩 생성한 뒤 틀린 것을 버리는 거부 표집(rejection sampling)만으로도 강력한 학습 데이터를 만들 수 있습니다. 생성은 부정확해도 검증이 정확하면, 살아남은 데이터는 믿을 수 있기 때문입니다.

반대편에는 검증이 어려운 영역이 있습니다. 열린 글쓰기, 요약의 품질, 대화의 자연스러움, 윤리적 판단 같은 것들입니다. 여기서는 "맞다/틀리다"를 기계적으로 가릴 수 없으니, 결국 또 다른 모델의 판단(RLAIF)이나 사람의 평가에 기대야 합니다. 그런데 그 판단 자체가 불완전하므로, 거른다고 걸러도 미묘한 오류와 편향이 데이터에 남습니다. 그래서 검증 불가능한 영역의 합성 데이터는 훨씬 조심스럽게, 사람·실제 데이터와 섞어 소량만 쓰는 것이 안전합니다.

이 경계선은 우연이 아니라 S6에서 다룰 추론 모델 학습의 핵심 원리이기도 합니다. o1·R1 같은 추론 모델이 수학과 코드에서 특히 강한 이유가 바로 이것입니다. 정답을 자동 검증할 수 있어, 모델이 스스로 만든 풀이 중 옳은 것만 골라 학습하는 선순환이 가능하기 때문입니다. 합성 데이터의 "거르기가 본체"라는 원칙이, 추론 능력을 길러 내는 엔진으로 발전하는 셈입니다.

## 자가 출판의 위험

요리사가 자기 요리 사진만 보고 새 요리를 배우면, 처음엔 빨리 늘지만 점점 자기 버릇만 강화돼 시야가 좁아집니다. 가끔 바깥 식당(실제 데이터)을 맛보고, 맛없는 시도는 버려야(거르기) 퇴화하지 않습니다. 합성 데이터도 똑같습니다. 섞고, 거르고, 검증해야 연료가 되지, 그냥 쓰면 독이 됩니다. 이 비유가 모델 붕괴의 본질을 그대로 담고 있습니다.

## 흔한 오해

"합성 데이터는 무한한 공짜 데이터다"라는 오해입니다. 거르지 않은 합성 데이터는 모델 붕괴와 편향 증폭을 부릅니다. 본체는 생성이 아니라 거르기입니다.

"합성 데이터는 항상 위험하니 피해야 한다"는 오해입니다. 검증 가능한 영역(수학·코드)에서는 매우 효과적입니다. 위험은 무분별한 사용에서 오지 합성 자체에서 오는 것이 아닙니다.

"합성 데이터로 학습하면 교사 모델을 넘을 수 없다"는 오해입니다. 거르기와 검증을 거치면, 특히 검증 가능한 영역에서는 학생이 교사를 능가하는 경우도 있습니다. 정답이라는 외부 신호가 교사의 한계를 넘게 해 주기 때문입니다.

## 정리

합성 데이터는 사람 데이터의 한계를 넘는 후학습 연료이지만(특히 검증 가능한 수학·코드), 본체는 생성이 아니라 거르기입니다. 거르지 않으면 모델 붕괴, 편향 증폭, 거짓 자신감, 평가 오염이 따라옵니다. 섞고, 거르고, 검증해야 합니다. 이로써 S5의 다섯 글 — SFT·보상모델링, RLHF, DPO, RLAIF/CAI, 합성 데이터 — 가 하나의 후학습 그림으로 모였습니다. 다음 모듈에서는 정렬된 모델이 답할 때 더 깊이 생각하게 만드는 추론과 테스트타임으로 넘어갑니다.
