# 3. DPO와 그 변형들 — 보상 모델을 지우다

RLHF(PPO)는 강력하지만 복잡합니다. 정책·보상·기준이라는 세 모델을 동시에 돌려야 하고, 학습이 불안정합니다. DPO는 도발적인 질문을 던집니다. "보상 모델과 강화학습을 통째로 건너뛸 수 없을까?" 그리고 그 답이 GRPO·KTO·IPO로 갈래를 칩니다. 이 글은 DPO가 왜 그런 단순화가 가능한지의 직관을 잡고, 주요 변형들이 각각 무엇을 노리는지를 정리합니다.

## 같은 목적지, 더 단순한 길

기초편에서 DPO를 "보상 모델 없이 선호를 직접 최적화"라고 한 줄로 짚었습니다. 이 글은 왜 그게 가능한가의 직관을 잡습니다. 핵심 통찰은 이렇습니다. RLHF가 결국 도달하려는 최적해는, 선호 데이터만으로 직접 풀 수 있는 형태로 다시 쓸 수 있습니다. 다시 말해 보상 모델을 따로 학습하고 PPO를 돌리는 우회로를, 하나의 지도학습 손실로 압축할 수 있습니다.

이것이 가능한 이유는 수학적 관계에 있습니다. 앞 글 RLHF의 최적해는 "보상이 현재 정책과 기준 정책의 비율의 로그에 비례한다"는 관계를 만족합니다. 그렇다면 보상 모델을 명시적으로 만들 필요 없이, 정책 자신을 보상으로 해석(암묵적 보상, implicit reward)해 선호 데이터에 바로 맞추면 됩니다[1].

```
RLHF:  선호 → [보상 모델 학습] → [PPO로 정책 학습]   (2단계, 불안정)
DPO :  선호 → [정책을 직접 학습]                      (1단계, 안정)
```

결과적으로 DPO는 분류 문제처럼 단순한 손실로 정렬을 합니다. 강화학습 루프도, 별도 보상 모델도, 보상 해킹 걱정도 크게 줄어듭니다. 논문 제목이 말하듯 "당신의 언어 모델은 사실 보상 모델"이었던 셈입니다.

## DPO 손실의 의미

DPO 손실의 형태만 의미로 읽어 보겠습니다(유도는 원문으로).

```
L_DPO = −log σ( β·[ (선호답의 로그비) − (거부답의 로그비) ] )
여기서 로그비 = log( π현재(답) / π기준(답) )
```

이 식이 말하는 것은 선호된 답(chosen)은 기준 모델 대비 확률을 높이고, 거부된 답(rejected)은 낮추라는 것입니다. β는 기준에서 벗어나는 강도로, 앞 글 KL의 역할이 여기 녹아 있습니다. 시그모이드와 로그우도는 기초편 2단계 그대로입니다. 새 수학이 아니라 익숙한 도구의 재배치입니다. 한 줄로 말하면 "좋은 답엔 확률을 더, 나쁜 답엔 덜, 단 원본에서 과하게 벗어나지 않게"입니다.

## 변형들 한눈에

DPO 이후 "더 단순하게, 더 잘, 다른 데이터로"를 노린 변형이 쏟아졌습니다(2026-06 기준). 차이의 핵심 한 줄만 잡으면 됩니다.

| 방법 | 한 줄 차이 | 무엇을 노리나 |
|---|---|---|
| DPO[1] | 선호쌍(A≻B)을 직접 최적화 | 보상 모델·RL 제거(단순·안정) |
| GRPO[2] | RL은 하되 가치함수(critic)를 없애고 그룹 내 상대비교로 이점 추정 | RLHF의 메모리·복잡도 절감(추론 학습에 강함) |
| KTO[3] | 쌍 비교 대신 답 하나당 좋다/나쁘다 라벨만으로 학습 | 선호쌍을 모으기 어려운 현실 데이터 대응 |
| IPO[4] | DPO가 선호에 과적합하는 경향을 이론적으로 교정 | 과최적화 완화(더 안정적인 선호 학습) |

여기서 GRPO는 짚어 둘 만합니다. DPO와 달리 GRPO는 여전히 RL입니다. 다만 PPO의 무거운 부품인 가치함수를 빼고, 한 질문에 답 여러 개를 뽑아 그 그룹 안에서 누가 나은지로 학습합니다. 이 방식은 추론 모델(다음 S6)에서 다시 만납니다. KTO는 "A가 B보다 낫다"는 비교쌍이 없어도 "이 답은 좋아" 또는 "별로야"라는 단독 신호만으로 학습합니다. 현실의 로그(좋아요/싫어요)에 잘 맞아 실용적입니다.

## 채점기를 머릿속에 내장하기

RLHF는 외부 코치(보상 모델)를 두고 점수를 받아 고쳤습니다. DPO는 그 코치를 학생의 머릿속에 흡수시킵니다. 더 이상 외부 채점기를 부르지 않고, "이 답이 저 답보다 낫다"는 비교 경험만으로 스스로 확률을 조정합니다. GRPO는 "코치는 없애되, 같은 시험을 여러 번 봐서 그중 잘 본 답을 표준으로 삼는" 방식이라고 보면 됩니다. 외부 채점기 대신 자기 그룹 안의 상대 평가를 쓰는 것입니다.

## 흔한 오해

"DPO가 RLHF를 항상 대체한다"는 오해입니다. 더 단순하고 안정적이지만, 세밀한 제어나 온라인 탐색에서는 RL 계열(PPO·GRPO)이 유리한 경우가 있습니다. 같은 목표의 다른 길입니다.

"GRPO는 DPO의 개선판이다"라는 오해입니다. DPO는 RL을 없앤 길이고, GRPO는 RL을 가볍게 만든 길입니다. 방향이 다릅니다.

"DPO엔 KL 개념이 없다"는 오해입니다. KL의 역할이 β와 기준 모델 항에 녹아 들어가 있습니다. 사라진 게 아니라 식 안에 숨었습니다.

## DPO의 약점: 오프라인이라는 그늘

DPO가 단순하고 안정적이라는 건 분명하지만, 공짜는 아닙니다. 가장 본질적인 약점은 DPO가 오프라인 방식이라는 점입니다. DPO는 미리 모아 둔 고정된 선호쌍 데이터로 학습합니다. 반면 PPO는 학습 중인 모델이 직접 답을 생성하고 그것을 보상 모델이 채점하는 온라인 방식이라, 모델이 변해 가는 과정을 따라가며 새로운 답을 탐색합니다. 이 차이가 결과를 가릅니다.

오프라인의 한계는 이렇습니다. 선호 데이터는 보통 학습 시작 전의 모델이 만든 답들로 구성되는데, 학습이 진행되면 모델이 그 데이터와 점점 멀어집니다. 그러면 데이터가 현재 모델의 약점을 더 이상 짚어 주지 못합니다. 시험 범위가 고정돼 있어 학생이 이미 아는 것만 반복 학습하는 셈입니다. 그래서 DPO는 데이터에 없는 새로운 실수를 교정하는 데 약하고, 선호 데이터의 품질과 다양성에 성능이 크게 좌우됩니다.

또 하나 자주 보고되는 문제가 길이 편향입니다. DPO는 종종 긴 답을 선호하도록 흘러가는 경향이 있는데, 선호 데이터에서 좋은 답이 우연히 더 길었던 패턴을 모델이 "길수록 좋다"로 잘못 학습하기 때문입니다. 이런 한계 때문에, 학습 중간중간 새 답을 생성하고 다시 선호를 매겨 데이터를 갱신하는 반복적(iterative) DPO나 온라인 변형들이 제안됐습니다. DPO가 RLHF를 완전히 대체하지 못하고 둘이 공존하는 이유가 여기 있습니다.

## 정리

DPO는 RLHF의 최적해를 다시 써서 보상 모델과 RL을 없애고, 선호쌍을 분류처럼 직접 학습합니다("좋은 답 확률 높이고, 나쁜 답 낮추되 원본에서 과하게 벗어나지 말기"). GRPO(가치함수 없는 가벼운 RL), KTO(단독 좋다/나쁘다), IPO(과최적화 교정)는 각각 다른 약점을 노린 갈래입니다. 다음 글에서는 정렬의 피드백 자체를 사람에서 AI로 옮기는 RLAIF와 Constitutional AI로 넘어갑니다.
