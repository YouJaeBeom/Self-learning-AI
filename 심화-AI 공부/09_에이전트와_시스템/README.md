# S9 · 에이전트·시스템 — 말하는 모델을 행동하는 시스템으로

기초편 6단계의 에이전트를 구조 수준으로 확장하는 모듈입니다. "모델이 추론하고, 도구를 쓰고, 계획하고, 반성하고, 여럿이 협업하게 하려면 어떤 구조가 필요한가?"에 답합니다. 이 모듈의 핵심은 더 큰 모델이 아니라, 모델을 둘러싼 구조 — 루프·도구·메모리·역할 — 가 에이전트의 능력을 만든다는 것입니다.

기초편 6단계 5절(에이전트와 멀티모달)에서 "[생각→행동→관찰] 루프"를 잡고 오면 좋습니다. 이 모듈에는 사실상 수학이 없습니다. 수학이 아니라 구조(아키텍처)의 이야기입니다.

## 이 모듈에서 답하는 질문

- 추론과 행동을 엮는 ReAct 루프는 무엇이고, 도구 호출은 어떻게 일어나나
- 에이전트가 여러 단계 과제를 풀려면 왜 계획·메모리·반성이 필요한가
- 멀티에이전트 협업은 무엇을 더 잘하나, 그 비용은 무엇인가
- LLM으로 LLM을 채점하는 LLM-as-judge는 무엇을 풀고, 무엇이 함정인가

한마디로 "말하는 모델을 행동하고 협업하는 시스템으로 어떻게 키우는가?"입니다.
