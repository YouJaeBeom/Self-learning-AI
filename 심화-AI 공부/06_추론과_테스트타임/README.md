# S6 · 추론·테스트타임 컴퓨트

학습이 아니라 추론(답할 때) 시점에 계산을 더 써서 어려운 문제를 푸는 원리를 이해하는 모듈입니다. o1·DeepSeek-R1 같은 추론 모델이 어떻게 더 오래 생각하는가를 파이프라인으로 잡습니다. 지금까지의 스케일링이 모델과 데이터를 키우는 학습타임의 이야기였다면, 이 모듈은 모델을 그대로 두고 생각의 양을 키우는 새로운 축을 다룹니다.

기초편 6단계 2절(추론 모델)에서 "테스트타임 연산 확장"의 직관을 잡고 오면 좋습니다. 이 모듈은 그것을 학습법까지 풀어냅니다. 새 수학은 거의 없습니다. 핵심은 어디에 계산을 더 붓는가라는 발상의 전환입니다.

## 이 모듈에서 답하는 질문

- 생각의 사슬(CoT)은 왜 정답률을 올리고, self-consistency는 거기에 무엇을 더하나
- "답할 때 계산을 더 쓰면 똑똑해진다"는 테스트타임 스케일링은 무엇인가(2026-06 기준)
- 결과만 채점하는 게 아니라 풀이 과정을 채점하는 PRM·검증자는 왜 필요한가
- o1·R1은 어떻게 긴 추론을 스스로 하도록 학습됐나

한마디로 "모델을 더 키우는 대신, 더 오래·더 잘 생각하게 만들 수 있는가?"입니다.

## 이 모듈에 필요한 수학 (의미 수준)

다수결(majority vote)과 표본 추출의 직관(self-consistency), "성능 ∝ 테스트타임 연산"의 경향(정확한 곡선은 원문), 과정 보상(PRM)의 채점 단위가 토큰열 전체가 아니라 단계라는 점이면 충분합니다. 새 수학은 거의 없으며, 핵심은 어디에 계산을 더 붓는가라는 발상의 전환입니다.
