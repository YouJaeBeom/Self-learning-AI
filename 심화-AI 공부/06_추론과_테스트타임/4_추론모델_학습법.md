# 4. 추론 모델 학습법 — STaR에서 R1까지

지금까지는 추론을 잘 끌어내는 법(CoT·다수결·검증자)이었습니다. 모두 모델은 그대로 두고 쓰는 방식만 바꿨습니다. 이 글은 한 걸음 더 갑니다. 모델이 애초에 긴 추론을 스스로 하도록 학습시키는 법입니다. o1·DeepSeek-R1의 비결입니다. 놀라운 점은, 긴 추론과 자기 교정 같은 정교한 행동을 사람이 일일이 설계하지 않고 정답 보상만으로 창발시켰다는 것입니다.

## 추론 능력을 주입하기

CoT는 프롬프트로 추론을 유도했습니다. 하지만 진짜 추론 모델은 시키지 않아도 길고 정교한 풀이를 펼치고, 막히면 스스로 점검하고 되돌아갑니다. 여기서 핵심 질문이 나옵니다. "좋은 추론 과정"을 어떻게 모델 안에 학습시킬까요? 정답 풀이를 사람이 일일이 써 주기는 어렵습니다. 수십만 개의 수학 문제에 모범 풀이를 다는 일은 비현실적입니다.

답의 실마리는 정답 여부를 자동으로 확인할 수 있다는 데 있습니다(수학·코드). 그러면 옳은 답에 이른 풀이만 골라 학습하면 됩니다. 풀이의 품질을 사람이 판단하지 않아도, 정답이라는 객관적 신호가 좋은 풀이를 걸러 줍니다.

## STaR — 자기 학습 추론

STaR(Self-Taught Reasoner)의 발상은 단순하고 강력합니다[1].

```
① 모델이 문제에 CoT 풀이를 생성
② 정답을 맞힌 풀이만 남김 (자동 채점)
③ 그 '성공한 풀이'로 모델을 다시 학습
④ 더 똑똑해진 모델로 ①부터 반복  (부트스트랩)
```

핵심은 사람이 풀이를 주지 않아도, 모델이 스스로 만든 성공 풀이로 추론을 길러 낸다는 것입니다. S5의 합성 데이터와 거르기의 추론판이라고 볼 수 있습니다. 보강 트릭(rationalization)도 있습니다. 못 푼 문제는 정답을 힌트로 줘서 풀이를 만들게 한 뒤, 그 풀이로 학습합니다. 이것이 "스스로 가르치는 추론"의 원형입니다.

## o-시리즈·R1 — RL로 추론을 창발시키기

2024–2025년의 추론 모델(OpenAI o-시리즈, DeepSeek-R1)은 STaR의 정신을 대규모 강화학습으로 확장했습니다(2026-06 기준). 공개된 R1을 기준으로 핵심 레시피를 보면 이렇습니다[2].

```
보상 = "최종 답이 맞았는가?" (+ 형식 규칙)   ← 검증 가능한 정답 신호
        │
        ▼
RL로 모델을 학습 (S5의 GRPO 사용)
        │
        ▼
모델이 보상을 높이려고 '스스로' 더 길게 생각하기 시작
 → 자기 점검·되돌아가기·다른 방법 시도 같은 행동이 창발
```

놀라운 점은 과정을 일일이 가르치지 않아도, "정답에 보상"만으로 모델이 긴 추론과 자기 교정을 스스로 발달시킨다는 것입니다. 생각의 길이가 학습 중 저절로 늘어납니다. 여기서 S5의 GRPO가 다시 등장합니다[3]. 한 문제에 풀이 여러 개를 뽑아 그룹 내 상대 비교로 학습하니, 가치함수 없이 가볍게 RL을 돌릴 수 있어 추론 학습에 잘 맞습니다. DeepSeek-R1은 순수 RL만으로도 추론이 창발함을 보였고, 약간의 SFT(읽기 좋은 풀이)로 다듬어 완성했습니다. 정리하면, o1과 R1이 더 오래 생각하는 비결은 긴 추론을 사람이 시킨 것이 아니라 정답 보상 RL로 모델이 스스로 익혔다는 데 있습니다.

## 검증 가능한 보상과 그룹 RL

```
보상 ≈ 정답 일치 여부(자동 검증) ,  학습 = GRPO(그룹 내 상대 이점으로 정책 갱신)
```

의미는 이렇습니다. 보상이 사람 취향(S5)이 아니라 객관적 정답이라 깨끗하고 보상 해킹이 어렵습니다. 사람 취향 보상은 채점기의 허점을 파고드는 보상 해킹에 취약했지만, 정답 일치는 속이기 어렵습니다. GRPO는 "한 문제의 여러 풀이 중 상대적으로 나은 것을 강화"하므로, 정답을 자주 내는 추론 습관이 길러집니다.

## 창발하는 자기 교정, 그리고 작은 모델로의 전수

R1식 학습에서 특히 주목받은 현상이 있습니다. 학습이 진행되면서 모델이 누가 시키지 않았는데도 "잠깐, 다시 확인해 보자" 같은 자기 점검 행동을 스스로 보이기 시작한다는 것입니다. 막다른 길에 다다르면 되돌아가 다른 방법을 시도하고, 자기 답을 의심하고 검산합니다. 이런 행동은 사람이 설계해 넣은 것이 아니라, "정답에 보상"이라는 단순한 목표를 최대화하는 과정에서 저절로 나타난 것입니다. 정답에 이르는 데 자기 교정이 유리하니, 모델이 그 습관을 스스로 발견한 셈입니다. 생각의 길이도 마찬가지로 학습 중에 저절로 늘어납니다. 어려운 문제일수록 더 오래 생각하는 것이 보상에 유리하기 때문입니다.

또 하나 실용적으로 중요한 발견은, 이렇게 길러 낸 추론 능력을 작은 모델로 전수할 수 있다는 것입니다. 큰 추론 모델이 만든 양질의 풀이(추론 흔적)를 모아, 그것으로 작은 모델을 지도학습(SFT)시키면, 작은 모델도 꽤 긴 추론을 흉내 낼 수 있습니다. 흥미롭게도 작은 모델을 처음부터 강화학습으로 추론시키는 것보다, 큰 모델의 풀이를 증류하는 쪽이 더 효율적인 경우가 많다고 보고됐습니다[2]. 추론을 처음 발견하는 일은 비싸지만, 발견된 추론을 베끼는 일은 훨씬 쌉니다. S5의 합성 데이터와 증류가 여기서 추론 능력의 전파 수단으로 다시 등장합니다.

다만 주의할 점은, 모델이 보여 주는 긴 추론이 반드시 모델의 실제 내부 계산과 일치하지는 않는다는 것입니다(앞 글들의 충실성 논의와 같은 맥락). 보이는 추론이 정답률을 높이는 데 유용한 것은 분명하지만, 그것을 모델의 투명한 사고 과정으로 해석하는 것은 또 다른 문제입니다.

## 채점만으로 스스로 느는 학생

선생님이 풀이법을 안 가르치고 답안만 채점해 줍니다. 단, 학생은 맞힐 때까지 여러 방법을 시도할 수 있습니다. 처음엔 짧게 풀다 자주 틀리던 학생이, 채점 피드백만으로 점점 스스로 검산하고, 다른 길로 다시 풀고, 더 신중해집니다. "정답이라는 보상"만으로 공부 습관 자체가 길러진 것입니다. 이것이 R1식 추론 학습의 직관입니다. 행동을 가르친 것이 아니라 목표를 주었더니 행동이 따라온 것입니다.

## 흔한 오해

"추론 모델은 완전히 새로운 구조다"라는 오해입니다. 토대는 같은 트랜스포머 디코더입니다. 달라진 것은 학습 방식(정답 보상 RL)과 답할 때 생각을 길게 하는 습관입니다.

"긴 생각의 사슬은 항상 사람이 설계한다"는 오해입니다. R1처럼 RL로 저절로 길어질 수 있습니다. 길이는 학습의 결과이지 손으로 짠 것이 아닙니다.

"정답 보상 RL이면 어디서나 통한다"는 오해입니다. 정답을 자동 검증할 수 있는 영역(수학·코드·논리)에서 특히 강합니다. 주관적이거나 열린 과제는 보상 정의가 어렵습니다.

## 정리

STaR는 스스로 만든 성공 풀이로 추론을 부트스트랩하고, o1·R1은 그것을 대규모 RL로 확장합니다. "정답에 보상"만으로 긴 추론과 자기 교정이 창발합니다(2026-06 기준). 핵심 도구는 검증 가능한 보상과 S5의 GRPO입니다. 단, 정답을 자동 검증할 수 있는 영역에서 가장 잘 통합니다. 이로써 S6의 네 글 — CoT·다수결, 테스트타임 컴퓨트, 검증자, 추론 학습 — 이 하나로 이어졌습니다. 다음 모듈에서는 이렇게 똑똑해진 거대 모델을 현실적 비용으로 굴리는 효율과 서빙으로 넘어갑니다.
