# 3. 과정 보상과 검증자 — 답이 아니라 풀이를 채점하기

self-consistency는 최종 답만 봤습니다. 그런데 어려운 추론에서는 답은 맞았는데 과정이 틀린 경우가 흔합니다. 운 좋게 맞은 것이죠. 반대로 좋은 풀이가 사소한 실수로 오답이 되기도 합니다. 그래서 결과가 아니라 과정(단계)을 채점하는 도구가 등장합니다. 바로 과정 보상 모델(PRM)과 검증자입니다. 이 글은 결과 채점과 과정 채점의 차이, 그리고 그 차이가 왜 어려운 추론에서 결정적인지를 다룹니다.

## 운 좋게 맞은 답을 가려내기

S5의 보상 모델은 보통 답 전체에 하나의 점수를 줬습니다(결과 보상). 추론에서는 이게 부족합니다. 답만 맞으면 점수를 주면, 모델은 엉터리 과정으로 정답을 찍는 요령을 배웁니다. 그러면 조금만 어려워져도 무너집니다. 진짜로 추론한 게 아니기 때문입니다.

여기서 발상이 나옵니다. 최종 답이 아니라 추론의 각 단계를 채점하자는 것입니다. 그래야 "옳은 과정으로 옳은 답"을 길러 낼 수 있습니다. 찍어서 맞힌 학생과 제대로 푼 학생을 구분하려면 풀이를 봐야 하는 것과 같습니다.

## ORM 대 PRM

채점 단위가 다른 두 보상 모델을 구분해야 합니다. 결과 보상 모델(ORM, Outcome Reward Model)은 최종 답이 맞았는지에 점수를 줍니다(S5의 보상 모델에 가까움). 과정 보상 모델(PRM, Process Reward Model)은 풀이의 각 단계가 올바른지에 점수를 줍니다[1].

```
문제: ... 풀이를 단계별로

단계 1: 식 세우기      → PRM: ✓ 0.9
단계 2: 양변 정리      → PRM: ✓ 0.8
단계 3: 부호 실수       → PRM: ✗ 0.1   ← 여기서 틀렸다!
단계 4: (틀린 채 진행)  → 최종답: 오답

ORM은 "최종답 오답"만 알지만,
PRM은 "3단계에서 틀렸다"를 콕 짚는다.
```

PRM은 어디서 틀렸는지까지 알려 줘서, 더 촘촘하고 정확한 학습·선택 신호가 됩니다. 과정 보상이 결과 보상보다 어려운 수학 추론에서 더 잘 작동한다는 것이 연구로 확인됐습니다[1].

## 검증자로 고르기

PRM(또는 다른 검증자)은 두 가지로 쓰입니다. 첫째는 선택(테스트타임)입니다. 모델이 여러 풀이를 생성하면 PRM이 각 풀이를 채점해 가장 과정이 탄탄한 풀이를 고릅니다. 이는 self-consistency의 다수결을 품질 가중 선택으로 업그레이드한 것으로, 흔히 best-of-N이라 부릅니다. 둘째는 학습 신호입니다. PRM의 단계별 점수를 보상으로 써서 모델이 좋은 과정을 내도록 학습합니다(다음 글 RL과 연결).

검증자는 생성보다 검증이 쉽다는 비대칭을 활용합니다. "푸는 것보다 채점이 쉽다"는 것입니다. 수학·코드에서 특히 그렇습니다. 정답과 대조하거나 코드를 실행하면 되기 때문입니다[2]. 이 비대칭이 테스트타임 연산을 효율적으로 쓰는 열쇠입니다. 여러 후보를 생성한 뒤 싼 검증으로 거르면, 모델 자체를 키우지 않고도 정확도를 높일 수 있습니다.

## 단계 단위 채점

```
풀이 점수 = 각 단계 점수들의 집계(예: 최소값 또는 곱)
```

이 식이 말하는 것은 풀이의 신뢰도가 가장 약한 단계에 좌우된다는 것입니다. 한 단계라도 틀리면 전체가 무너지므로 최소값이나 곱으로 집계합니다. 그래서 PRM은 결과 한 점이 아니라 과정의 모든 마디를 봅니다. 핵심 대비를 하면, ORM은 "정답인가(예/아니오)"라는 한 점이고, PRM은 단계별 점수의 열입니다. 담는 정보량이 다릅니다.

## 수학 시험의 부분점수

선생님이 답만 보고 채점하면(ORM), 찍어서 맞힌 학생과 제대로 푼 학생을 구분하지 못합니다. 풀이 과정에 부분점수를 주면(PRM), "3번째 줄에서 부호 실수"를 콕 짚어 어디를 고쳐야 하는지 알려 줍니다. PRM은 바로 이 부분점수 채점관입니다. 그리고 좋은 부분점수는 진짜 실력을 길러 냅니다. 결과만 보는 채점은 요령을 키우지만, 과정을 보는 채점은 실력을 키웁니다.

## 흔한 오해

"PRM이 ORM보다 항상 낫다"는 오해입니다. PRM은 단계별 라벨이 필요해 만들기 비쌉니다. 결과만 검증 가능한 과제에서는 ORM이 현실적입니다.

"검증자가 있으면 추론이 완벽해진다"는 오해입니다. 검증자도 모델이라 틀립니다. 검증자의 허점을 노리는 풀이가 나올 수 있습니다(보상 해킹의 추론판).

"best-of-N은 그냥 다수결이다"라는 오해입니다. 다수결(self-consistency)은 빈도로 고르고, best-of-N은 검증자 점수로 고릅니다. 후자가 품질을 직접 봅니다.

## PRM의 진짜 비용: 단계 라벨을 어떻게 만드나

PRM이 ORM보다 정보가 많고 강력하다면, 왜 모두가 PRM을 쓰지 않을까요? 답은 라벨링 비용에 있습니다. ORM은 최종 답이 맞았는지만 알면 되므로 라벨이 싸지만, PRM은 풀이의 각 단계가 옳은지를 일일이 표시해야 합니다. 사람이 수만 개 풀이의 모든 단계에 옳고 그름을 다는 일은 느리고 비쌉니다. 이 라벨링 병목이 PRM의 가장 큰 현실적 장벽입니다.

그래서 단계 라벨을 자동으로 만드는 영리한 방법이 제안됐습니다. 핵심 아이디어는 이렇습니다. 어떤 중간 단계가 좋은 단계인지 판단하려면, 그 단계에서 출발해 끝까지 여러 번 풀어 본 뒤 정답에 이르는 비율을 보면 됩니다. 좋은 단계에서 출발하면 정답에 자주 도달하고, 나쁜 단계에서 출발하면 좀처럼 도달하지 못합니다. 즉 "이 단계 이후로 풀었을 때 정답률"을 그 단계의 자동 라벨로 삼는 것입니다. 사람 없이 정답 검증만으로 과정 라벨을 만드는 셈입니다.

이 방식은 앞서 본 "생성보다 검증이 쉽다"는 비대칭과, 정답을 자동 확인할 수 있는 영역(수학·코드)이라는 조건 위에서 작동합니다. 검증 가능한 영역에서는 최종 정답이라는 단 하나의 신호로부터 과정 전체의 라벨을 끌어낼 수 있고, 그 라벨로 PRM을 학습하면 다시 더 나은 과정 선택과 학습 신호를 얻습니다. 다음 글에서 다룰 추론 모델 학습이, 바로 이 "정답이라는 외부 신호로 과정을 길러 내는" 발상을 대규모 강화학습으로 밀어붙인 결과입니다.

## 정리

결과 보상(ORM)은 최종 답만, 과정 보상(PRM)은 풀이의 각 단계를 채점합니다. 운 좋게 맞은 답을 걸러 내기 위함입니다. 검증자는 "푸는 것보다 채점이 쉽다"는 비대칭을 활용해, 여러 풀이 중 과정이 탄탄한 것을 고르고(best-of-N) 학습 신호로도 씁니다. 다음 글에서는 이 과정 신호와 정답 검증을 학습에 녹여, 모델이 애초에 긴 추론을 스스로 하도록 만드는 추론 모델 학습법으로 넘어갑니다.
