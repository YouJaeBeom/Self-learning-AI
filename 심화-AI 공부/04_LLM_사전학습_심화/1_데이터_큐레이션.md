# 1. 데이터 큐레이션 — 무엇을 먹이느냐가 모델을 만든다

기초편 5단계에서 LLM은 "방대한 텍스트로 다음 토큰 예측을 배운다"고 했습니다. 그런데 어떤 텍스트냐가 결과를 가릅니다. 오래된 격언 그대로 "쓰레기를 넣으면 쓰레기가 나온다"입니다. 사전학습의 절반은 모델 구조가 아니라 데이터를 요리하는 일, 곧 큐레이션입니다. 이 글은 그 요리법을 다룹니다. 의외라고 느낄 수 있지만, 2026-06 기준 모델 구조는 서로 비슷해진 반면 데이터 큐레이션이 모델 간 차이를 만드는 핵심 경쟁력이 됐습니다.

## 모델의 먹거리를 설계하기

수조 개의 토큰을 모은다고 다 좋은 것이 아닙니다. 인터넷은 보석과 쓰레기가 섞인 광산입니다. 큐레이션의 목적은 방대한 원천에서 품질 높고, 다양하고, 중복 없는 학습 데이터를 정제하는 것입니다. 데이터의 질과 구성이 모델의 지식·편향·능력의 상한을 정합니다. 모델이 모르는 것은 대개 데이터에 없던 것이고, 모델이 편향된 것은 대개 데이터가 편향됐던 것입니다.

## 수집과 다양성

첫 단계는 수집입니다. 웹 크롤(예: Common Crawl), 책, 코드, 위키, 논문, 대화 등 다양한 원천에서 텍스트를 모읍니다. 여기서 다양성이 결정적입니다. 한쪽에 치우치면 모델도 치우칩니다. 코드를 넣으면 추론과 구조화 능력이, 다국어를 넣으면 언어 능력이 넓어지는 식입니다. 어떤 능력을 원한다면 그 능력의 씨앗이 되는 데이터가 식단에 들어 있어야 합니다.

## 품질 필터링 — 옥석 가리기

다음은 품질 필터링입니다. 스팸·광고·자동생성 쓰레기·유해 콘텐츠·개인정보 등을 걸러 냅니다. 방법은 크게 세 가지를 섞어 씁니다. 문장 길이나 기호 비율 같은 단순 규칙(휴리스틱), 좋은 글과 나쁜 글을 구분하도록 학습한 분류기, 그리고 유해 콘텐츠를 잡는 안전 필터입니다. 여기서 자주 확인되는 사실 하나는, 적지만 깨끗한 데이터가 많지만 더러운 데이터를 이기는 경우가 많다는 것입니다. 잘 정제된 웹 데이터가 원본 웹 데이터를 크게 앞선다는 보고가 이를 뒷받침합니다[4].

## 중복 제거 — 같은 걸 또 외우지 않게

웹에는 같은 문서가 수없이 복제돼 있습니다. 같은 기사가 수백 사이트에 퍼지고, 같은 문구가 반복됩니다. 이를 그대로 두면 모델이 그 복제된 내용만 과하게 외웁니다(암기·과적합). 중복을 제거하면 같은 연산으로 더 다양한 것을 배우고, 학습 데이터를 그대로 베껴 내뱉는 위험(암기)도 줄어듭니다. 학습 데이터의 중복 제거가 암기를 줄이고 품질을 실제로 끌어올린다는 것이 연구로 확인됐습니다[1]. 중복된 토큰은 새로 배울 것이 없는 토큰이므로, 중복 제거는 곧 유효 데이터를 늘리는 일입니다.

## 토크나이저 — 텍스트를 토큰으로, 잘

기초편 5단계의 BPE를 한 걸음 더 들어가 봅니다. 토크나이저는 텍스트를 토큰으로 쪼개는 장치인데, 그 방식 선택이 다국어·코드·숫자 처리에 영향을 줍니다.

| 방식 | 특징 | 강점 |
|---|---|---|
| BPE | 자주 나오는 쌍을 병합 | 단순, 널리 쓰임 |
| 바이트 수준 BPE | 모든 문자를 바이트 조합으로 | 미등록 단어 없음(이모지·희귀문자도) |
| Unigram (SentencePiece) | 확률적으로 어휘 선택 | 언어 독립적, 다국어 유리 |

바이트 수준 방식은 어떤 문자든 바이트 조합으로 표현하므로 미등록 단어 문제가 사라집니다. SentencePiece 같은 도구는 언어에 독립적이라 다국어·코드 처리의 표준이 됐습니다[3]. 어휘집 크기와 구성은 압축률과 드문 언어 처리 사이의 트레이드오프를 만듭니다. LLM이 "글자 수 세기"를 종종 틀리는 현상도 이 토큰화 방식에 뿌리가 있습니다. 모델은 글자가 아니라 토큰 단위로 세상을 보기 때문입니다(기초편 5단계와 연결).

## 운동선수의 식단

거대 모델을 국가대표 선수라고 하면, 사전학습은 식단 관리입니다. 수집은 다양한 식재료 확보입니다. 탄수화물만 먹으면 균형이 깨지듯, 코드·다국어·추론 자료를 고루 넣어야 합니다. 품질 필터링은 상한 재료와 불량식품을 골라내는 일입니다. 양보다 질입니다. 중복 제거는 매일 같은 메뉴만 주지 않는 것입니다. 같은 것을 반복하면 그것만 잘하고 편식하게 됩니다. 토크나이저는 음식을 소화하기 좋은 크기로 손질하는 일입니다. 너무 잘게도, 너무 크게도 안 됩니다. 같은 체급(모델 크기)이라도 식단(데이터)이 다르면 전혀 다른 선수가 됩니다.

## 데이터 파이프라인

```
원천(웹·책·코드…) ─→ 품질 필터 ─→ 중복 제거 ─→ 혼합 비율 결정 ─→ 토크나이즈 ─→ 학습용 토큰
   다양성 확보        옥석 가리기   같은 것 제거   (코드 몇 %? 다국어 몇 %?)   소화 크기로
```

여기서 혼합 비율(data mixture)이 중요합니다. 어떤 출처를 몇 퍼센트 섞느냐가 모델 성격을 좌우합니다. 코드 비중을 높이면 추론·구조화 능력이 올라가는 식입니다. 이 파이프라인은 한 번으로 끝나지 않고 반복 개선됩니다. 데이터 품질을 1퍼센트 올리는 것이 모델 크기를 키우는 것보다 쌀 때가 많기 때문입니다. 마지막으로 데이터 오염(contamination)에 주의해야 합니다. 평가용 벤치마크 문제가 학습 데이터에 새어 들면 점수가 부풀려집니다. 정제 단계에서 이를 걸러야 공정한 평가가 됩니다(S0의 교훈).

## 수식으로 보는 균형

```
모델 성능  ↔  데이터의 (양 × 질 × 다양성),   단 중복은 유효 양을 깎는다
```

성능은 단순한 토큰 개수가 아니라 질·다양성과 함께 정해집니다. 중복된 토큰은 새로 배울 것이 없는 토큰이라 유효 데이터 양을 깎습니다. 그래서 중복 제거가 곧 유효 데이터를 늘리는 일입니다. 정량적인 '데이터 대 모델' 균형(스케일링·Chinchilla)은 3번 글에서 본격적으로 다룹니다.

## 흔한 오해

"데이터는 많을수록 좋다"는 오해입니다. 질·다양성·중복 제거가 양만큼 중요합니다. 더럽고 중복 많은 대량 데이터보다, 깨끗하고 다양한 적정량이 낫습니다.

"토크나이저는 한 번 정하면 끝인 부품"이라는 오해입니다. 어휘 구성이 다국어·코드·숫자 처리와 압축률을 좌우합니다. 모델 능력의 숨은 변수입니다.

"벤치마크 점수는 순수 실력이다"라는 오해입니다. 학습 데이터에 시험 문제가 섞이면(데이터 오염) 점수가 부풀려집니다. 큐레이션 단계의 오염 제거가 공정성의 전제입니다.

## 정리

사전학습의 절반은 데이터 요리입니다. 다양하게 수집하고, 품질을 필터링하고, 중복을 제거하고, 혼합 비율을 정하고, 토크나이즈합니다. "쓰레기를 넣으면 쓰레기가 나온다"는 격언처럼 양보다 질·다양성·중복 제거가 중요하고, 데이터 오염 제거가 공정한 평가의 전제입니다. 2026-06 기준 데이터 큐레이션이 핵심 경쟁력이 된 이유입니다. 다음 글에서는 이렇게 준비한 데이터를 수천 GPU로 학습시키는 분산학습으로 넘어갑니다.
