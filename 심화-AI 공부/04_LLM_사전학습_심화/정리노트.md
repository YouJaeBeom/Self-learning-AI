# S4 정리노트 — 한눈에 보는 LLM 사전학습

이 모듈의 본문을 한 장으로 압축했습니다. 네 글의 핵심을 다시 짚고, 큰 그림과 대표 논문, 다음 모듈로 가는 다리를 정리합니다.

## 네 글의 요지

| 주제 | 왜 필요한가 | 핵심 한 문장 |
|---|---|---|
| 데이터 큐레이션 | 모델의 '먹거리'를 설계하려고 | 다양 수집→품질 필터→중복 제거→혼합 비율→토크나이즈. 양보다 질·다양성 |
| 분산학습·병렬화 | 한 GPU에 안 들어가서 | 데이터(데이터 분담)·텐서(층 내부)·파이프라인(층 분담) 병렬 + ZeRO(중복 제거) |
| 스케일링·Chinchilla | 예산을 최적 배분하려고 | '가장 큰 모델'이 아니라 N·D 균형 모델이 이긴다. 데이터도 함께 키워라 |
| 사전학습 레시피 | 마라톤을 완주하려고 | 혼합정밀·체크포인팅·클리핑·체크포인트 저장 — 안 터지게 하는 받침대 |

## 이 모듈의 핵심 수식

| 수식 | 한 줄 의미 |
|---|---|
| `성능 ↔ 데이터(양×질×다양성)`, 중복은 유효량↓ | 중복 제거가 곧 유효 데이터 늘리기 |
| `GPU 부담 ≈ (모델+기울기+옵티마이저+활성값)/나눈 정도` | 병렬·ZeRO가 분모를 키움(대가는 통신) |
| `C ≈ N × D`, N·D를 함께 키워라 | Chinchilla: 균형이 답, 데이터가 부족하기 쉬움 |
| `혼합정밀·체크포인팅 = 거래(trade-off)` | 정밀도·연산을 내주고 메모리·속도를 삼 |

큰 그림은 이렇습니다. 기초편 5단계의 '사전학습 규모'가 실제 엔지니어링으로 구체화됩니다. 데이터(1), 인프라(2), 균형(3), 레시피(4)가 합쳐져 다음 토큰 예측을 거대 규모로 실현합니다. 목표는 그대로이고 규모만 커진 것입니다.

## 이 장의 대표 논문

| 주제 | 논문 | 핵심 한 줄 |
|---|---|---|
| 데이터 중복 | Deduplicating Training Data Makes LMs Better (Lee et al., 2021)[1] | 중복 제거가 품질을 실제로 올림 |
| 데이터셋 | The Pile (Gao et al., 2020)[2] | 다양한 출처를 섞은 대표 공개 코퍼스 |
| 토크나이저 | SentencePiece (Kudo & Richardson, 2018)[3] | 언어 독립 서브워드 토크나이저 |
| 병렬화 | Megatron-LM (Shoeybi et al., 2019)[4] | 텐서 병렬로 거대 트랜스포머 학습 |
| 메모리 | ZeRO (Rajbhandari et al., 2020)[5] | 옵티마이저 상태 중복 제거로 메모리 절약 |
| 스케일링 | Scaling Laws (Kaplan et al., 2020)[6] | 손실이 멱법칙으로 매끄럽게 향상 |
| 균형 | Chinchilla (Hoffmann et al., 2022)[7] | N·D 균형 — 모델만 키우지 말고 데이터도 |
| 정밀도 | Mixed Precision Training (Micikevicius et al., 2017)[8] | 16/32비트 혼합으로 메모리·속도 |

2026-06 기준 데이터 큐레이션과 균형(데이터 쪽으로의) 비중이 커지는 추세이고, 분산·정밀도 기법은 대형 학습의 표준 부품입니다.

## 다음 모듈 예고 — S5 · 정렬·후학습 심화

S4에서 만든 기반 모델(base model)은 아직 인터넷 글을 흉내 낼 뿐, 대화 상대가 아닙니다(기초편 6단계의 '정렬' 예고를 떠올려 보세요). S5는 그 기반 모델을 쓸모 있고 안전하게 다듬는 후학습 전 과정을 다룹니다. SFT → 보상 모델링 → RLHF(PPO) → DPO, 그리고 GRPO·Constitutional AI·합성 데이터입니다. 기초편 6단계에서 맛본 RLHF·DPO가 실제 파이프라인으로 펼쳐집니다. 대표 논문은 InstructGPT/RLHF(Ouyang et al., 2022), DPO(Rafailov et al., 2023)입니다. 사전학습이 '날것의 지능'을 빚었다면, 후학습은 그것을 '대화할 수 있는 조력자'로 길들이는 과정입니다.
