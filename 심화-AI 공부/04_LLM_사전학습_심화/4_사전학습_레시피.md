# 4. 사전학습 레시피 — 거대 학습을 실제로 굴리는 장치들

데이터를 정제하고(1번), 일을 GPU에 나누고(2번), 예산을 균형 있게 배분했습니다(3번). 마지막으로, 거대 학습을 실제로 몇 달간 안 터지고 돌리는 실전 레시피가 필요합니다. 이 글은 그 자잘하지만 결정적인 장치들 — 혼합정밀·체크포인팅·안정화 — 을 한눈에 모읍니다. 화려하지 않지만, 이것이 없으면 거대 학습은 메모리 부족이나 발산으로 멈춥니다. 수개월짜리 마라톤을 완주시키는 것은 결국 이런 받침대들입니다.

## 메모리와 시간을 쥐어짜고, 안 터지게 하기

거대 학습은 수개월에 수천 GPU가 걸린 마라톤입니다. 두 가지가 늘 부족하고, 한 가지가 늘 위협합니다. 부족한 것은 메모리(모델이 겨우 들어감)와 시간(빠를수록 비용이 줄어듦)입니다. 위협하는 것은 불안정으로, 한 번의 발산이 몇 주의 학습을 날립니다. 목적은 정밀도·메모리·시간을 쥐어짜 더 큰 모델을 더 빠르게 학습하면서, 마라톤이 중간에 터지지 않게 지키는 것입니다.

## 혼합정밀 — 숫자를 가볍게

보통 숫자는 32비트(FP32)로 다룹니다. 그런데 16비트(FP16/BF16)로 다루면 메모리가 절반이고 계산도 빠릅니다. 핵심은 대부분의 계산은 16비트로 하되, 정밀도가 중요한 일부(누적이나 가중치 사본)는 32비트로 유지하는 것입니다. 둘을 섞어(mixed) 쓴다고 해서 혼합정밀입니다[1]. 특히 BF16은 표현 범위가 넓어 발산에 강해서 2026-06 기준 대형 학습의 표준입니다. 효과는 같은 GPU로 더 큰 모델을 더 빨리 학습하는 것입니다.

## 그래디언트 체크포인팅 — 메모리와 연산의 거래

역전파(기초편 3단계)는 순전파의 중간 활성값을 저장해 둬야 합니다. 이것이 메모리를 많이 먹습니다. 그래디언트 체크포인팅은 중간값을 다 저장하지 않고, 역전파 때 필요하면 다시 계산합니다[2]. 즉 메모리를 아끼는 대신 연산을 조금 더 하는 거래입니다. 메모리가 부족할 때 더 큰 모델을 끼워 넣는 카드로 쓰입니다.

## 학습 안정화 — 마라톤을 지키기

거대 학습은 손실이 갑자기 튀는 일(loss spike)이 잦습니다. 한 번 발산하면 큰 손해라, 여러 안전장치를 둡니다. 그래디언트 클리핑(gradient clipping)은 기울기가 너무 크면 잘라 발산을 막습니다(S1의 기울기 폭발과 연결). 워밍업과 학습률 스케줄(S1)은 초반엔 살살, 이후 부드럽게 감쇠시킵니다. 레이어 정규화와 잔차연결(S1·기초편 4단계)은 신호와 기울기가 안정적으로 흐르게 합니다. 체크포인트 저장·재개는 모델 상태를 주기적으로 저장해, 터지면 직전 지점부터 재시작합니다. 귀중한 학습을 잃지 않기 위함입니다. 한 가지 주의할 점은, 이 '체크포인트 저장'은 앞의 '그래디언트 체크포인팅'과 이름만 비슷할 뿐 전혀 다른 것이라는 점입니다. 이건 학습 상태 백업입니다.

여기에 모니터링이 더해집니다. 손실 곡선·기울기 크기·학습률을 실시간으로 관찰하다, 이상 징후(스파이크)가 보이면 개입합니다. 거대 학습은 자동화돼도 결국 사람이 지켜보는 마라톤입니다.

## 장거리 비행 운영

거대 사전학습을 대서양 횡단 비행으로 그려 보면 직관이 잡힙니다. 혼합정밀은 연료를 가볍게 하는 것입니다. 꼭 필요한 곳만 고급 연료(32비트), 나머지는 경량 연료(16비트)를 써서 더 멀리 더 빨리 갑니다. 그래디언트 체크포인팅은 짐을 덜고 필요하면 현지 조달하는 것입니다. 다 싣지 않고(메모리 절약), 필요할 때 다시 만듭니다(연산 증가). 안정화와 클리핑은 난기류 대비입니다. 갑작스러운 흔들림(loss spike)에 안전벨트(클리핑)와 자동 보정(워밍업·정규화)을 둡니다. 체크포인트 저장은 블랙박스이자 중간 기착지입니다. 문제가 생기면 처음이 아니라 마지막 기착지에서 다시 뜹니다. 거대 학습은 화려한 한 방이 아니라, 수개월을 안 터지고 완주하게 하는 수많은 작은 장치들의 합입니다.

## 레시피 한눈에

```
[준비]   정제된 데이터(1) + 병렬 구성(2) + 균형 잡힌 N·D(3)
[정밀도] BF16 혼합정밀로 메모리·속도 확보
[메모리] 그래디언트 체크포인팅으로 활성값 메모리 절약
[안정성] 워밍업→감쇠 LR + 그래디언트 클리핑 + 정규화
[복원력] 주기적 체크포인트 저장 → 스파이크/장애 시 재개
[감시]   손실·기울기 모니터링, 이상 시 개입
   └──────── 수조 토큰을 수개월간 반복 ────────┘
```

이 레시피의 모든 항목은 새 능력을 더하는 것이 아니라, 거대 규모를 현실적으로 가능하게 하는 받침대입니다. S1부터 S4까지가 여기서 만납니다. S1(옵티마이저·정규화·워밍업), S3(어텐션 효율), S4(데이터·병렬·균형)가 합쳐져야 하나의 사전학습이 완성됩니다. 그리고 이렇게 만든 기반 모델(base model)은 아직 대화하도록 다듬어지지 않았습니다. 그 다음 단계가 후학습이고, S5에서 다룹니다. 대규모 학습의 손실 스파이크와 그 대응을 상세히 기록한 보고로 PaLM 논문이 좋은 참고서입니다[3].

## 수식으로 보는 두 가지 거래

```
혼합정밀:       메모리·시간 ↓   (16비트 사용),  정밀도 일부는 32비트로 보존
체크포인팅:     메모리 ↓  ↔  연산 ↑   (중간값을 저장 대신 재계산)
```

두 기법 모두 거래(trade-off)입니다. 혼합정밀은 정밀도를 약간 양보해 메모리·속도를 얻고(중요 부분은 보존), 체크포인팅은 연산을 더 해서 메모리를 삽니다. 거대 학습은 이런 거래로 부족한 자원을 쥐어짜는 일입니다.

## 흔한 오해

"이런 건 디테일이라 안 중요하다"는 오해입니다. 혼합정밀·체크포인팅·안정화가 없으면 거대 학습은 메모리 부족이나 발산으로 멈춥니다. 모델을 가능하게 하는 필수 받침대입니다.

"두 종류의 '체크포인팅'은 같은 것"이라는 오해입니다. 그래디언트 체크포인팅(활성값 재계산으로 메모리 절약)과 체크포인트 저장(학습 상태 백업·재개)은 이름만 닮은 다른 장치입니다.

"혼합정밀은 정확도를 해친다"는 오해입니다. 중요한 부분(누적·가중치 사본)은 32비트로 지키므로, 잘 쓰면 품질을 거의 유지하면서 메모리·속도를 크게 얻습니다(BF16이 표준이 된 이유).

## 정리

사전학습 레시피는 거대 마라톤을 안 터지고 완주하게 하는 작은 장치들의 합입니다. 혼합정밀(BF16, 메모리·속도), 그래디언트 체크포인팅(메모리와 연산의 거래), 안정화(클리핑·워밍업·정규화), 체크포인트 저장(장애 재개)이 그것입니다. 화려하지 않지만 없으면 학습이 멈춥니다. 이로써 S4의 네 글 — 데이터, 병렬, 균형, 레시피 — 이 하나의 사전학습으로 모였습니다. 이렇게 만든 기반 모델을 쓸모 있고 안전하게 다듬는 후학습이 다음 모듈 S5입니다.
